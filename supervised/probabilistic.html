11 · Probabilistic Perspectives
MLE · MAP · Bayesian Inference · Regularisation as Priors · Generative vs Discriminative · Naïve Bayes · Calibration
Every loss function we have used — MSE for regression, cross-entropy for classification — looks like an engineering choice. But each is actually a consequence of a principled probabilistic assumption about the data. Once you see this, a great deal of supervised learning unifies: regularization is a prior, training is inference, and the loss function is the log-likelihood of a specific generative model for the labels. This lesson traces that unification, then opens the generative direction — models that explicitly represent how data is generated — and discusses what a model's output probabilities actually mean.
Maximum likelihood estimation
The setup. We observe data $(x_1, y_1), \ldots, (x_n, y_n)$ drawn i.i.d. from some distribution. We model the conditional distribution of labels given inputs as $p(y \mid x; \theta)$, parametrized by $\theta$ (the weights of a model). We do not know $\theta$ and want to find it from the data.
Maximum likelihood estimation (MLE) finds the $\theta$ that makes the observed data most probable under the model: $$\hat{\theta}_{\text{MLE}} = \argmax_\theta \prod_{i=1}^n p(y_i \mid x_i; \theta)$$ Taking the log (which is monotone and turns the product into a sum): $$\hat{\theta}_{\text{MLE}} = \argmax_\theta \sum_{i=1}^n \log p(y_i \mid x_i; \theta)$$ Equivalently, minimizing the negative log-likelihood: $$\hat{\theta}_{\text{MLE}} = \argmin_\theta -\frac{1}{n}\sum_{i=1}^n \log p(y_i \mid x_i; \theta)$$ This is the canonical form. Every standard loss function is a negative log-likelihood under a specific conditional model $p(y \mid x; \theta)$.
Connection to empirical risk minimization. Empirical risk minimization (Lesson 1) minimizes $\hat{R}(h) = \frac{1}{n}\sum_i \mathcal{L}(h(x_i), y_i)$. MLE minimizes the negative log-likelihood $-\frac{1}{n}\sum_i \log p(y_i \mid x_i; \theta)$. These are the same when the loss function is $\mathcal{L}(h(x), y) = -\log p(y \mid x; \theta_h)$. ERM is MLE. The loss function implicitly specifies the probability model.
Standard losses derived from MLE
MSE from Gaussian noise. Assume $y = h_\theta(x) + \varepsilon$ where $\varepsilon \sim \mathcal{N}(0, \sigma^2)$. Then $p(y \mid x; \theta) = \mathcal{N}(y; h_\theta(x), \sigma^2)$: $$\log p(y \mid x; \theta) = -\frac{(y - h_\theta(x))^2}{2\sigma^2} - \frac{1}{2}\log(2\pi\sigma^2)$$ Maximizing this over $\theta$ is equivalent to minimizing $(y - h_\theta(x))^2$ — the mean squared error. MSE is MLE under Gaussian label noise.
Cross-entropy from Bernoulli / Categorical. For binary classification, model $p(y=1 \mid x; \theta) = \sigma(h_\theta(x))$. Then: $$\log p(y \mid x; \theta) = y \log \hat{p} + (1-y) \log(1-\hat{p})$$ The negative log-likelihood is the binary cross-entropy. For $K$-class classification with softmax $\hat{p}_k = \text{softmax}(h_\theta(x))_k$: $$\log p(y \mid x; \theta) = \sum_{k=1}^K y_k \log \hat{p}_k = \log \hat{p}_{y}$$ The negative log-likelihood is the categorical cross-entropy (Lesson 3). Cross-entropy is MLE under a Bernoulli / Categorical label model.
MAE from Laplacian noise. If $\varepsilon \sim \text{Laplace}(0, b)$, the log-likelihood is $-|y - h_\theta(x)|/b$ plus a constant. Maximizing gives the mean absolute error (MAE) objective. Laplacian noise is heavier-tailed than Gaussian, so MAE is more robust to outliers — it corresponds to a model that assigns non-negligible probability to large deviations. This is the probabilistic explanation for why MAE is preferred over MSE when labels contain outliers.
MAP estimation
MLE treats $\theta$ as a fixed unknown and maximizes the likelihood. But we often have prior knowledge about what $\theta$ should look like — weights should be small, for instance. Bayesian inference treats $\theta$ as a random variable with a prior distribution $p(\theta)$. By Bayes' theorem: $$p(\theta \mid \mathcal{D}) = \frac{p(\mathcal{D} \mid \theta) \, p(\theta)}{p(\mathcal{D})}$$ The posterior $p(\theta \mid \mathcal{D})$ encodes our belief about $\theta$ after seeing the data.
Maximum a posteriori (MAP) estimation finds the mode of the posterior — the single $\theta$ that maximizes $p(\theta \mid \mathcal{D})$: $$\hat{\theta}_{\text{MAP}} = \argmax_\theta p(\theta \mid \mathcal{D}) = \argmax_\theta \left[\log p(\mathcal{D} \mid \theta) + \log p(\theta)\right]$$ The $p(\mathcal{D})$ denominator is constant in $\theta$ and drops out. MAP is MLE with an extra term $\log p(\theta)$ — the log-prior over weights.
Bayesian update: prior × likelihood → posterior
Regularisation as priors
Derivation — L2 regularisation is a Gaussian prior

Suppose weights $\theta_j \overset{\text{i.i.d.}}{\sim} \mathcal{N}(0, \tau^2)$ independently. Then: $$\log p(\theta) = \sum_j \log \mathcal{N}(\theta_j; 0, \tau^2) = -\frac{1}{2\tau^2}\|\theta\|^2 + \text{const}$$

The MAP objective: $$\hat{\theta}_{\text{MAP}} = \argmin_\theta \left[-\sum_i \log p(y_i \mid x_i; \theta) + \frac{1}{2\tau^2}\|\theta\|^2\right]$$

This is exactly ERM with L2 regularization $\lambda = \frac{1}{2\tau^2}$. Larger $\tau$ (broader prior, less confident weights are small) → smaller $\lambda$ (less regularization). Smaller $\tau$ (stronger prior that weights are near zero) → larger $\lambda$ (more regularization). $\square$

L1 regularisation is a Laplace prior. If $\theta_j \sim \text{Laplace}(0, b)$, then $\log p(\theta) = -\frac{1}{b}\|\theta\|_1 + \text{const}$. MAP gives L1-regularized ERM with $\lambda = 1/b$. The Laplace distribution has a sharp peak at zero — it strongly "expects" most weights to be exactly zero, inducing sparsity. This is the probabilistic reason L1 produces sparse solutions while L2 does not.
The prior encodes inductive bias. Choosing the regularization type and strength is equivalent to choosing the prior distribution over model parameters. This is not arbitrary — the right regularization depends on what kind of function you believe the true relationship to be. A Gaussian prior says "weights should be small and symmetrically distributed." A Laplace prior says "most weights should be zero, but a few may be large." Domain knowledge should inform the choice.
The Bayesian view
MAP is still a point estimate — it finds one $\theta$. Full Bayesian inference maintains the entire posterior $p(\theta \mid \mathcal{D})$ and makes predictions by integrating over it: $$p(y^* \mid x^*, \mathcal{D}) = \int p(y^* \mid x^*, \theta) \, p(\theta \mid \mathcal{D}) \, d\theta$$ This is the posterior predictive distribution. Instead of one model's prediction, it averages over all models weighted by their posterior probability — a continuous ensemble.
The integral is intractable for neural networks: the posterior over millions of parameters is a high-dimensional distribution with no closed form. Practical approximations include:
  • Laplace approximation: approximate the posterior as a Gaussian centered at the MAP solution, with covariance given by the inverse Hessian of the loss.
  • Variational inference: find the simplest distribution $q(\theta)$ that minimizes the KL divergence to the true posterior. MC Dropout (Lesson 10) is an instance of this.
  • Deep ensembles: train $M$ networks from different random initialisations, use their prediction spread as the uncertainty estimate. Surprisingly competitive with explicit Bayesian methods.
The Bayesian view does not always improve predictive accuracy over MAP — but it provides principled uncertainty quantification, which matters when the cost of a confident-but-wrong prediction is high.
Generative vs discriminative models
Supervised learning models $p(y \mid x)$ — the conditional distribution of labels given inputs. There are two fundamentally different strategies:
Discriminative models directly parametrize $p(y \mid x; \theta)$. Logistic regression, neural network classifiers, SVMs — all discriminative. They learn the boundary between classes without modeling the input distribution. Since they directly model what we want (the label given the input), discriminative models generally achieve lower classification error when there is enough data.
Generative models model the joint distribution $p(x, y) = p(x \mid y) p(y)$ and recover the conditional by Bayes' theorem: $$p(y \mid x) = \frac{p(x \mid y) p(y)}{p(x)} = \frac{p(x \mid y) p(y)}{\sum_{y'} p(x \mid y') p(y')}$$ Generative models explicitly represent how data is generated — what inputs look like given a class label. This costs extra: we have to model the input distribution $p(x \mid y)$, which is harder than modeling the boundary. But it gives capabilities discriminative models lack: (1) generating new examples from a class; (2) detecting out-of-distribution inputs ($p(x)$ is low for inputs unlike training data); (3) learning from unlabeled data (semi-supervised learning, since the generative model can use unlabeled $x$ to improve $p(x)$).

Naïve Bayes classifier
Naïve Bayes is a generative classifier that makes one strong simplifying assumption: features $x_1, \ldots, x_d$ are conditionally independent given the class $y$: $$p(x_1, \ldots, x_d \mid y) = \prod_{j=1}^d p(x_j \mid y)$$ This is "naïve" because it ignores all correlations between features. Despite being wrong for nearly every real dataset, Naïve Bayes often works surprisingly well.
Classification uses Bayes' theorem: $$p(y \mid \mathbf{x}) \propto p(y) \prod_{j=1}^d p(x_j \mid y)$$ Taking logs and predicting the class with the highest log-posterior: $$\hat{y} = \argmax_y \left[\log p(y) + \sum_{j=1}^d \log p(x_j \mid y)\right]$$ The components $p(y)$ (class prior) and $p(x_j \mid y)$ (per-feature likelihoods) are estimated from training data by counting: $\hat{p}(y=k) = n_k / n$, and $\hat{p}(x_j \mid y=k)$ estimated by the empirical distribution of feature $j$ among class-$k$ examples (Gaussian for continuous features, categorical counts for discrete).
Gaussian Naïve Bayes models each $p(x_j \mid y)$ as $\mathcal{N}(\mu_{jk}, \sigma_{jk}^2)$ — a Gaussian per feature per class. The parameters $\mu_{jk}$ and $\sigma_{jk}^2$ are estimated by the mean and variance of feature $j$ among class-$k$ training examples.
Multinomial Naïve Bayes models each $p(x_j \mid y)$ as a categorical distribution — useful for text classification where $x_j$ is a word count. With Laplace smoothing (add 1 to every count to avoid zero probabilities): $$\hat{p}(x_j = v \mid y=k) = \frac{N_{jvk} + 1}{\sum_{v'} (N_{jv'k} + 1)}$$ This is the classic spam filter model.
Why does Naïve Bayes work despite the independence assumption? Even if the feature distribution is wrong, the posterior log-odds $\log p(y=1 \mid \mathbf{x}) - \log p(y=0 \mid \mathbf{x})$ may still be approximately correct in sign — correct classification even with a miscalibrated probability. Naïve Bayes consistently misestimates probabilities (they cluster near 0 and 1) but often gets the argmax right. Its great strength is efficiency: parameter estimation is a single scan through the data, and prediction is a sum of $d$ terms. This makes it practical for $d = 10^6$ (text data).
Calibration
A model that outputs $\hat{p}(y=1 \mid x) = 0.8$ is claiming 80% confidence. Is it right? Calibration asks whether a model's probability outputs match empirical frequencies: among all inputs where the model says 80%, what fraction actually belong to class 1?
A perfectly calibrated model satisfies: $$P(Y = 1 \mid \hat{p}(Y=1 \mid X) = p) = p \quad \forall\, p \in [0, 1]$$ Real models are often poorly calibrated:
  • Naïve Bayes is systematically overconfident — it pushes probabilities toward 0 and 1.
  • Neural networks trained with cross-entropy tend to be overconfident (Guo et al. 2017) — they assign high probability to correct classes even when training on a small dataset.
  • Neural networks trained with label smoothing (replacing one-hot targets $[0, 1]$ with $[0.1, 0.9]$) are better calibrated, because they are penalized for assigning extreme probabilities.
Reliability diagrams visualize calibration: bin predictions by their confidence, plot mean confidence vs. mean accuracy per bin. A perfectly calibrated model's points fall on the diagonal. Overconfident models cluster above the diagonal.
Platt scaling: fit a logistic regression on the model's raw output scores to recalibrate its probabilities. Temperature scaling: divide the final logit by a scalar $T > 1$ (soften the distribution) to reduce overconfidence. Both are post-hoc calibration methods.
When calibration matters. For classification decisions with equal cost per error (e.g., spam/not-spam), calibration doesn't matter — you just pick the argmax. For decisions where probabilities inform downstream choices (medical diagnosis, weather forecasting, financial risk), calibration matters a great deal. A model that says "95% probability of malignancy" should actually be right 95% of the time for that claim to be meaningful.
Worked examples
Example 1 — MLE for a Gaussian

Derive the MLE for $\mu$ and $\sigma^2$ given observations $x_1, \ldots, x_n \overset{\text{i.i.d.}}{\sim} \mathcal{N}(\mu, \sigma^2)$.

Log-likelihood: $$\ell(\mu, \sigma^2) = \sum_{i=1}^n \log \mathcal{N}(x_i; \mu, \sigma^2) = -\frac{n}{2}\log(2\pi\sigma^2) - \frac{1}{2\sigma^2}\sum_i (x_i - \mu)^2$$

Setting $\frac{\partial \ell}{\partial \mu} = 0$: $$\frac{\partial \ell}{\partial \mu} = \frac{1}{\sigma^2}\sum_i (x_i - \mu) = 0 \implies \hat{\mu}_{\text{MLE}} = \frac{1}{n}\sum_i x_i = \bar{x}$$ The MLE for the mean is the sample mean.

Setting $\frac{\partial \ell}{\partial \sigma^2} = 0$: $$\frac{\partial \ell}{\partial \sigma^2} = -\frac{n}{2\sigma^2} + \frac{1}{2(\sigma^2)^2}\sum_i (x_i - \hat{\mu})^2 = 0 \implies \hat{\sigma}^2_{\text{MLE}} = \frac{1}{n}\sum_i (x_i - \bar{x})^2$$ This is the biased sample variance (divides by $n$, not $n-1$). The unbiased estimator $s^2 = \frac{1}{n-1}\sum (x_i - \bar{x})^2$ is the Bayesian MAP under a weak improper prior, not the MLE.

Example 2 — MAP with a Gaussian prior gives Ridge regression

For linear regression with $y = \mathbf{w}^\top \mathbf{x} + \varepsilon$, $\varepsilon \sim \mathcal{N}(0, \sigma^2)$, and prior $w_j \overset{\text{i.i.d.}}{\sim} \mathcal{N}(0, \tau^2)$. Show the MAP estimate is the Ridge solution.

Log-likelihood: $\log p(\mathcal{D} \mid \mathbf{w}) = -\frac{1}{2\sigma^2}\sum_i (y_i - \mathbf{w}^\top \mathbf{x}_i)^2 + \text{const}$

Log-prior: $\log p(\mathbf{w}) = -\frac{1}{2\tau^2}\|\mathbf{w}\|^2 + \text{const}$

MAP objective (minimise negative log-posterior): $$\hat{\mathbf{w}}_{\text{MAP}} = \argmin_\mathbf{w} \left[\frac{1}{2\sigma^2}\sum_i (y_i - \mathbf{w}^\top \mathbf{x}_i)^2 + \frac{1}{2\tau^2}\|\mathbf{w}\|^2\right]$$

This is Ridge regression with $\lambda = \sigma^2 / \tau^2$. Setting the gradient to zero: $$\frac{1}{\sigma^2}X^\top(X\mathbf{w} - \mathbf{y}) + \frac{1}{\tau^2}\mathbf{w} = 0 \implies \hat{\mathbf{w}}_{\text{MAP}} = \left(X^\top X + \frac{\sigma^2}{\tau^2}I\right)^{-1} X^\top \mathbf{y}$$ which is exactly the Ridge solution $(X^\top X + n\lambda I)^{-1} X^\top \mathbf{y}$ (with $n\lambda = \sigma^2/\tau^2$). Ridge regression is Gaussian MLE under a Gaussian prior. $\square$

Example 3 — Naïve Bayes spam classifier

Training data: 3 spam emails and 2 ham emails. Word "free" appears: in 2/3 spam, 0/2 ham. Word "meeting" appears: in 0/3 spam, 2/2 ham. New email contains both words. Classify it (spam or ham)? Use Laplace smoothing with $\alpha = 1$.

Class priors: $\hat{p}(\text{spam}) = 3/5 = 0.6$, $\hat{p}(\text{ham}) = 2/5 = 0.4$

With Laplace smoothing ($\alpha=1$, vocabulary size = 2 words for simplicity): $$\hat{p}(\text{``free''} \mid \text{spam}) = \frac{2+1}{3+2} = \frac{3}{5} = 0.6$$ $$\hat{p}(\text{``free''} \mid \text{ham}) = \frac{0+1}{2+2} = \frac{1}{4} = 0.25$$ $$\hat{p}(\text{``meeting''} \mid \text{spam}) = \frac{0+1}{3+2} = \frac{1}{5} = 0.2$$ $$\hat{p}(\text{``meeting''} \mid \text{ham}) = \frac{2+1}{2+2} = \frac{3}{4} = 0.75$$

Log-posteriors (using log for numerical stability): $$\log \hat{p}(\text{spam} \mid \text{email}) \propto \log 0.6 + \log 0.6 + \log 0.2 = -0.511 - 0.511 - 1.609 = -2.631$$ $$\log \hat{p}(\text{ham} \mid \text{email}) \propto \log 0.4 + \log 0.25 + \log 0.75 = -0.916 - 1.386 - 0.288 = -2.590$$

Ham has a slightly higher log-posterior. Classification: ham (but only marginally). The "meeting" signal outweighs the "free" signal in this tiny dataset. In practice, spam classifiers use thousands of word features and the decisions become more decisive.

Food for thought
  • MLE is consistent — with infinite data, it converges to the true parameters. But with finite data, MLE can overfit. MAP adds a prior to prevent this. Doesn't adding a prior introduce bias? Isn't that cheating?
    Yes, a prior introduces bias — the MAP estimate is biased toward the prior mean. But bias-variance decomposition (Lesson 4) shows that a small amount of bias is a good trade when it substantially reduces variance. With finite data, MLE has high variance (the estimator is sensitive to which $n$ examples you happened to see). A Gaussian prior shrinks weights toward zero, reducing variance at the cost of a small bias toward zero. The same tradeoff as Ridge regression: biased but lower MSE. As $n \to \infty$, the likelihood dominates the prior and MAP converges to MLE — the bias shrinks to zero. This is why regularization strength $\lambda$ should decrease as more data is available: more data means less variance, so less regularization is needed. The prior is not "cheating" — it is a formalization of the inductive bias every model needs (no free lunch).
  • A language model like GPT outputs $p(x_t \mid x_{ Both, depending on the framing. As a standalone model, GPT is generative: it models $p(x) = \prod_t p(x_t \mid x_{can be used discriminatively — classifying a text example $x$ involves computing $\log p(x \mid \text{class } k \text{ prompt})$ for each class $k$ and picking the highest. This is in-context learning: the generative model implicitly becomes a discriminative classifier by conditioning on examples. The generative/discriminative distinction is not binary for modern LLMs — they are fundamentally generative models that acquire discriminative capabilities through conditioning.
  • Naïve Bayes assumes features are conditionally independent given the class. Logistic regression makes no such assumption. Yet on many small datasets they perform comparably. Why?
    Naïve Bayes is a generative classifier with a strong (usually wrong) prior that features are independent. Logistic regression is discriminative, estimating $p(y \mid x)$ directly. Ng & Jordan (2001) proved a formal result: in the limit of infinite data, logistic regression has lower asymptotic error. But with finite data, Naïve Bayes converges to its (biased) asymptote faster — it has fewer effective parameters because its independence assumption kills all the feature-feature interaction terms. With small data, fast convergence beats asymptotic correctness. The practical upshot: Naïve Bayes wins on small datasets with many features (like text classification with a small labelled corpus); logistic regression wins with enough data to estimate the feature correlations properly. This is a general pattern: stronger assumptions (more bias, less variance) help with small data; weaker assumptions (less bias, more variance) need large data to shine.
    • Common pitfalls
    • Treating model outputs as calibrated probabilities without checking. Neural network softmax outputs are not probabilities in the sense of "if the model says 95%, it's right 95% of the time." Modern networks trained with cross-entropy tend to be overconfident. Always evaluate calibration (reliability diagrams, expected calibration error) before using probabilities for downstream decisions.
    • Using log-probability naively with very small values. In Naïve Bayes with discrete features, a feature that never appeared with a given class gets probability 0 — a single unseen word zeros out the entire posterior. Always use Laplace (add-one) or add-$\alpha$ smoothing. In general, work in log-space and never compute $\log(0)$.
    • Equating "training loss is NLL" with "the model is well-calibrated." Minimizing cross-entropy encourages the model to assign high probability to correct labels, but it does not constrain how wrong probabilities are assigned. A model can have low cross-entropy loss while still assigning 99% probability to incorrect classes for adversarial examples. NLL is a proper scoring rule (calibration is rewarded in expectation) but requires infinite data to achieve perfect calibration.
    • Using the wrong noise model. MSE is MLE under Gaussian noise, which means outliers are very unlikely under the model — a single outlier at distance $5\sigma$ contributes $25$ to the MSE but only $5$ to the MAE. If your labels contain outliers (corrupted measurements, mislabelling), MSE will distort the learned function toward the outliers. Use MAE, Huber loss, or explicit outlier-robust regression for contaminated data.
