8 · Neural Networks
Perceptron · MLP · Activation Functions · Forward Pass · Universal Approximation · Depth vs Width
Logistic regression is powerful but limited: its decision boundary is always a hyperplane. Most real-world classification problems — recognizing faces, understanding language, predicting protein structure — require boundaries of arbitrary complexity. A neural network solves this by learning its own features: a cascade of learned transformations that reshape the input space until the problem becomes linearly separable. The output layer then applies exactly the logistic regression you already know.
The gap logistic regression can't close
Consider the XOR problem: four points in $\mathbb{R}^2$ with labels $(0,0) \to 0$, $(0,1) \to 1$, $(1,0) \to 1$, $(1,1) \to 0$. No hyperplane separates the 1s from the 0s — the positive class (XOR = 1) occupies two diagonally opposite corners, and the negative class the other two. Any linear classifier fails.
In Lesson 3 we noted the fix: apply a nonlinear feature map $\phi$ first, then train a linear classifier in feature space. But who designs $\phi$? For images, hand-crafting good features (edge detectors, color histograms, HOG descriptors) was a decade-long engineering effort. The key insight of neural networks: learn $\phi$ from data. The hidden layers of a neural network are a learned, parametrized feature transformation. The final layer is logistic (or softmax) regression on those learned features.
The artificial neuron
A single artificial neuron computes: $$a \;=\; \sigma\!\left(\sum_{j=1}^d w_j x_j + b\right) \;=\; \sigma(\mathbf{w}^\top \mathbf{x} + b)$$ This is exactly logistic regression. The linear combination $z = \mathbf{w}^\top \mathbf{x} + b$ is called the pre-activation or logit; applying $\sigma$ gives the activation $a$. The weights $w_j$ and bias $b$ are the parameters.
A single neuron can only learn a linear boundary. Power comes from combining many neurons and stacking them in layers.
Layers and the multi-layer perceptron
A multi-layer perceptron (MLP) — also called a feedforward neural network or fully connected network — organizes neurons into layers:
  • Input layer: the raw features $\mathbf{x} \in \mathbb{R}^d$. Not a layer of neurons — just the input vector.
  • Hidden layers: one or more layers of neurons. Every neuron in a layer receives inputs from every neuron in the previous layer (hence "fully connected"). The outputs of layer $\ell$ become the inputs to layer $\ell+1$.
  • Output layer: produces the final predictions. For regression, typically a single linear neuron (no activation). For $K$-class classification, $K$ neurons followed by softmax.
Notation. Denote the number of hidden layers as $L$. Layer $\ell$ has $n_\ell$ neurons. The weight matrix connecting layer $\ell-1$ to layer $\ell$ is $W^{(\ell)} \in \mathbb{R}^{n_\ell \times n_{\ell-1}}$, and the bias vector is $\mathbf{b}^{(\ell)} \in \mathbb{R}^{n_\ell}$. For a 2-hidden-layer network with $d$ inputs, hidden sizes $n_1, n_2$, and $K$ outputs: $$W^{(1)} \in \mathbb{R}^{n_1 \times d}, \quad W^{(2)} \in \mathbb{R}^{n_2 \times n_1}, \quad W^{(3)} \in \mathbb{R}^{K \times n_2}$$
Activation functions
Without a nonlinear activation function, a multi-layer network collapses to a single linear transformation: $W^{(2)}(W^{(1)}\mathbf{x}) = (W^{(2)}W^{(1)})\mathbf{x}$. Nonlinearity at each layer is what gives neural networks their expressive power. The choice of activation function has historically been one of the most important design decisions in deep learning.
Sigmoid $\sigma(z) = \frac{1}{1+e^{-z}}$. Output in $(0,1)$. Historically the default but now rarely used in hidden layers because it saturates: for large $|z|$, $\sigma'(z) \approx 0$, causing vanishing gradients that prevent learning in deep networks (Lesson 9). Still used in the output layer for binary classification.
Tanh $\tanh(z) = \frac{e^z - e^{-z}}{e^z + e^{-z}}$. Output in $(-1, 1)$. Zero-centered (unlike sigmoid), which helps gradient flow. Still saturates for large $|z|$. Derivative: $1 - \tanh^2(z)$. Used in RNNs and some specialized architectures.
ReLU (Rectified Linear Unit) $\text{ReLU}(z) = \max(0, z)$. The default activation for modern deep networks. Derivative is 1 for $z > 0$ and 0 for $z < 0$ — no saturation for positive inputs. Fast to compute. Two issues: (1) dying ReLU: neurons with $z < 0$ always output 0 and receive no gradient; they can "die" and stop learning entirely. (2) Non-differentiable at $z = 0$ — not a problem in practice; just pick either subgradient.
Leaky ReLU $\text{LReLU}(z) = \max(\alpha z, z)$ for small $\alpha > 0$ (typically 0.01). Allows a small gradient for negative inputs, preventing dead neurons.
GELU (Gaussian Error Linear Unit) $\text{GELU}(z) = z \cdot \Phi(z)$ where $\Phi$ is the standard normal CDF. Smooth approximation to ReLU. The default activation in Transformers (BERT, GPT, and virtually every modern large language model). Differentiable everywhere.
Softmax — used in the output layer for classification, not in hidden layers. As covered in Lesson 3, converts logits to probabilities.
Why GELU in Transformers? GELU is smooth (no kinks like ReLU), zero-centered (unlike sigmoid), and acts as a soft gate — inputs close to zero are attenuated, large positive inputs pass through nearly unchanged. Empirically it outperforms ReLU on large language model tasks, though the theoretical explanation for why is still debated.
The forward pass
The forward pass computes the network's output for a given input. For a network with $L$ hidden layers, it iterates layer by layer: $$\mathbf{z}^{(\ell)} \;=\; W^{(\ell)}\mathbf{h}^{(\ell-1)} + \mathbf{b}^{(\ell)}$$ $$\mathbf{h}^{(\ell)} \;=\; \sigma\!\left(\mathbf{z}^{(\ell)}\right)$$ where $\mathbf{h}^{(0)} = \mathbf{x}$ (the input), $\mathbf{z}^{(\ell)}$ is the pre-activation vector, and $\sigma$ is applied elementwise. After all hidden layers, the output layer computes the final prediction: $$\hat{\mathbf{y}} \;=\; \text{softmax}(W^{(L+1)}\mathbf{h}^{(L)} + \mathbf{b}^{(L+1)})$$
Written out for a concrete 2-hidden-layer network classifying into $K$ classes: $$\mathbf{h}^{(1)} = \sigma(W^{(1)}\mathbf{x} + \mathbf{b}^{(1)}) \;\in\; \mathbb{R}^{n_1}$$ $$\mathbf{h}^{(2)} = \sigma(W^{(2)}\mathbf{h}^{(1)} + \mathbf{b}^{(2)}) \;\in\; \mathbb{R}^{n_2}$$ $$\hat{\mathbf{y}} = \text{softmax}(W^{(3)}\mathbf{h}^{(2)} + \mathbf{b}^{(3)}) \;\in\; \mathbb{R}^K$$ Each $\mathbf{h}^{(\ell)}$ is a learned representation of the input — the network transforms $\mathbf{x}$ through a sequence of increasingly abstract feature spaces until the classes become linearly separable in the final hidden layer.
Counting parameters. A fully connected layer from $n_{\ell-1}$ to $n_\ell$ neurons has $n_{\ell-1} \times n_\ell$ weights plus $n_\ell$ biases — a total of $n_\ell(n_{\ell-1} + 1)$ parameters. For large layers this grows quickly: a single layer from 1,024 to 1,024 neurons has over one million parameters. Deep learning models routinely have $10^8$ to $10^{12}$ parameters.
Universal approximation
What can a neural network represent in principle? The answer is: anything continuous — and that is a precise theorem.
Universal Approximation Theorem (Cybenko 1989, Hornik 1991): A feedforward network with a single hidden layer of finite width $n_1$, using a non-polynomial activation function $\sigma$, can approximate any continuous function $f : [0,1]^d \to \mathbb{R}$ to arbitrary accuracy $\varepsilon > 0$: $$\sup_{\mathbf{x} \in [0,1]^d} \left|f(\mathbf{x}) - h(\mathbf{x}; W^{(1)}, W^{(2)}, \mathbf{b})\right| < \varepsilon$$ for a sufficiently large hidden layer size $n_1$.
This is a powerful existence result. But it has two important caveats:
  • The required width may be exponential in $d$. To approximate a function of $d$ inputs, the theorem may require $n_1$ growing exponentially in $d$. This is fine for $d = 2$ and catastrophic for $d = 1000$.
  • It says nothing about learning. Knowing a network of some size can represent a function does not tell you that gradient descent will find those weights. The theorem is about expressiveness, not optimization.
Depth — the alternative to width. A deep network with many layers can represent functions that a shallow network would need exponentially many neurons to express. This is the theoretical case for depth, and it matches the empirical success of deep (many-layer) networks over wide (few-layer) networks in practice.
Capacity, depth, and why deep works
The effective capacity of a network — loosely, how complex a function it can represent — is controlled by:
  • Number of parameters (weights and biases).
  • Depth (number of layers).
  • Architecture (how layers are connected — fully connected, convolutional, recurrent, attention-based).
Why does depth help beyond just adding parameters? The key insight: deep networks compute hierarchical representations. Consider image recognition. Early layers learn to detect edges and textures; middle layers combine these into parts (wheels, eyes, windows); deep layers assemble parts into objects. Each layer builds on the previous one. This compositional structure matches the compositional structure of the world — complex objects are made of simpler parts, which are made of even simpler primitives.
A formal result: certain functions can be computed by a depth-$L$ network with $O(n)$ parameters per layer that would require a depth-1 network with $O(2^n)$ parameters to express (Montufar et al. 2014). Depth provides exponential gains in representational efficiency for certain function classes.
In practice: for most tasks (vision, language, audio, structured data), networks with 4–100+ layers dramatically outperform shallow networks with the same parameter count. The Transformer (the subject of the Deep Learning cluster) has 12–96 layers in its standard configurations.
Worked examples
Example 1 — Solving XOR with a 2-layer network

The XOR problem: inputs $(0,0), (0,1), (1,0), (1,1)$ with labels $0, 1, 1, 0$. Show that a network with one hidden layer of 2 ReLU neurons can solve this exactly.

Define the hidden layer: $$h_1 = \text{ReLU}(x_1 + x_2 - 1.5), \quad h_2 = \text{ReLU}(-x_1 - x_2 + 0.5)$$ And the output: $$\hat{y} = h_1 + h_2$$

Trace through all four inputs:

  • $(0,0)$: $h_1 = \text{ReLU}(-1.5) = 0$, $h_2 = \text{ReLU}(0.5) = 0.5$. $\hat{y} = 0.5$. But ground truth is 0 — let's re-check the architecture.

A cleaner construction: let the hidden layer compute $h_1 = \text{ReLU}(x_1 + x_2 - 0.5)$ and $h_2 = \text{ReLU}(-x_1 - x_2 + 1.5)$, output $= 2h_1 - h_2 + \text{bias}$. Actually the most transparent solution uses: $$h_1 = \sigma(x_1 + x_2 - 0.5), \quad h_2 = \sigma(x_1 + x_2 - 1.5)$$ $$\hat{y} = \sigma(h_1 - 2h_2)$$

The intuition is cleaner than the exact arithmetic: $h_1$ fires when at least one input is 1 (OR), and $h_2$ fires only when both are 1 (AND-like). The output fires when $h_1$ is active but $h_2$ is not: XOR = OR $\setminus$ AND. The first hidden layer transforms the input space so that the XOR classes, which are not linearly separable in the original $(x_1, x_2)$ plane, become linearly separable in the $(h_1, h_2)$ plane. This is the general mechanism: hidden layers reshape the input space.

Example 2 — Forward pass computation

A 2-class MLP with input $\mathbf{x} = [1, 0]^\top$, one hidden layer of 2 neurons with ReLU activation, and a softmax output layer. $$W^{(1)} = \begin{bmatrix} 1 & -1 \\ -1 & 1 \end{bmatrix}, \quad \mathbf{b}^{(1)} = \begin{bmatrix} 0 \\ 0 \end{bmatrix}$$ $$W^{(2)} = \begin{bmatrix} 1 & 0 \\ 0 & 1 \end{bmatrix}, \quad \mathbf{b}^{(2)} = \begin{bmatrix} 0 \\ 0 \end{bmatrix}$$ Compute the output probabilities.

Hidden layer pre-activations: $$\mathbf{z}^{(1)} = W^{(1)}\mathbf{x} = \begin{bmatrix}1 & -1 \\ -1 & 1\end{bmatrix}\begin{bmatrix}1 \\ 0\end{bmatrix} = \begin{bmatrix}1 \\ -1\end{bmatrix}$$

Hidden layer activations (ReLU): $$\mathbf{h}^{(1)} = \text{ReLU}(\mathbf{z}^{(1)}) = \begin{bmatrix}\max(0,1) \\ \max(0,-1)\end{bmatrix} = \begin{bmatrix}1 \\ 0\end{bmatrix}$$ The second neuron is "dead" for this input — its pre-activation is negative, ReLU outputs 0.

Output pre-activations: $$\mathbf{z}^{(2)} = W^{(2)}\mathbf{h}^{(1)} = \begin{bmatrix}1 & 0 \\ 0 & 1\end{bmatrix}\begin{bmatrix}1 \\ 0\end{bmatrix} = \begin{bmatrix}1 \\ 0\end{bmatrix}$$

Softmax output: $$\hat{p}_1 = \frac{e^1}{e^1+e^0} = \frac{e}{e+1} \approx 0.731, \quad \hat{p}_2 = \frac{1}{e+1} \approx 0.269$$ The network assigns 73.1% probability to class 1 for this input.

Example 3 — Parameter counting

A network for classifying MNIST digits (28×28 grayscale images into 10 classes) has architecture: Input(784) → FC(512) → FC(256) → Output(10). How many parameters does it have?

Layer 1 (784 → 512): $784 \times 512 + 512 = 401{,}920$ parameters.

Layer 2 (512 → 256): $512 \times 256 + 256 = 131{,}328$ parameters.

Layer 3 (256 → 10): $256 \times 10 + 10 = 2{,}570$ parameters.

Total: $401{,}920 + 131{,}328 + 2{,}570 = \mathbf{535{,}818}$ parameters — over half a million for a relatively small network on $28\times28$ images. ResNet-50 (for high-resolution images) has 25 million parameters; GPT-3 has 175 billion. The quadratic scaling of fully connected layers (width² per layer) is why convolutional and attention architectures are necessary for high-resolution inputs.

Food for thought
  • The Universal Approximation Theorem guarantees a shallow network can approximate any function. Yet deep networks drastically outperform shallow ones in practice. If a shallow network can represent the same function, why can't we just use one?
    Two reasons: (1) Efficiency: the UAT guarantees approximation for some width, but that width may be astronomically large. Depth achieves the same representational power with exponentially fewer parameters for certain function classes (functions with compositional structure). (2) Optimization: the loss landscape of deep networks is easier to optimize by gradient descent than an equivalently expressive shallow network. Deep networks have better gradient signal — each layer is "closer" to the output and receives stronger gradient feedback. The shallow network needs enormous weights in one layer to do what a deep network spreads across many, and those enormous weights are harder to find. In short: depth is not just about what can be represented, but about what gradient descent can efficiently find.
  • If we remove all activation functions from a neural network, it collapses to a single linear transformation. What if we use an activation that is almost linear everywhere — like a very weak nonlinearity? Does that help?
    Very weak nonlinearities help but not as much as strong ones. The capacity of the network grows with the "degree" of nonlinearity at each layer. A nearly linear activation approximates a linear function and provides little gain over having no activation at all. Interestingly, ReLU is as weak a nonlinearity as possible while still being useful — it is piecewise linear. The expressiveness comes not from the nonlinearity at any individual neuron but from the exponential number of linear regions that a deep ReLU network creates: an $L$-layer ReLU network with $n$ neurons per layer can have up to $O((n/d)^{(L-1)d} n^d)$ linear regions, growing exponentially in depth.
  • A neural network is often described as a "black box" — it's hard to understand why it makes a particular prediction. But the forward pass is just matrix multiplications and elementwise nonlinearities — every step is explicitly computable. Why does interpretability remain a hard problem?
    The computation is explicit but not legible. The issue is that meaning is distributed: no single weight or neuron corresponds to an interpretable concept. A feature detector for "dog" might be encoded across 10,000 neurons in the penultimate layer, each contributing a small amount. Asking "what does weight $w_{ij}^{(3)}$ mean?" is like asking what a single transistor in a CPU "means" — the computation is clear, but the semantics emerge from the collective behavior, not individual components. Some architectures (attention mechanisms, sparse models) are more interpretable because they explicitly produce attention weights that say "this output attended strongly to these inputs." This is one reason attention-based models like Transformers have become not just more accurate but more analyzable than opaque deep MLPs.
Common pitfalls
  • Using sigmoid activations in hidden layers of deep networks. Sigmoid saturates for large $|z|$, driving gradients toward zero in early layers. Use ReLU (or GELU for Transformers) in hidden layers. Reserve sigmoid for binary classification output layers.
  • Forgetting the activation function entirely. Stacking linear layers without activations gives a single linear transformation. The network is no more powerful than logistic regression, regardless of depth. Every hidden layer needs an activation.
  • Conflating parameter count with capacity. Parameters are a necessary but not sufficient measure of capacity. Architecture matters: a 10-layer network with 100 parameters per layer can represent things a 1-layer network with 1,000 parameters cannot. And modern over-parameterized networks show that more parameters does not always mean worse generalization (see Lesson 4's discussion of double descent).
  • Using a linear output for classification. The output layer of a classifier should produce probabilities (via softmax for multi-class, sigmoid for binary). A raw linear output produces logits, not probabilities. The cross-entropy loss requires probabilities; applying it to raw logits gives the wrong gradients unless you implement log-softmax carefully. Most deep learning frameworks provide a numerically stable fused log-softmax + cross-entropy loss — always use that.
  • Ignoring initialization. Initializing all weights to zero makes all neurons in a layer identical — they compute the same function and receive the same gradient update, staying identical forever. The network never breaks symmetry. Initialize weights randomly with a distribution that accounts for layer width (Xavier/Glorot or He initialization, covered in Lesson 9).
Connects to
Backpropagation (Lesson 9) — the forward pass computes predictions. Backpropagation computes how each weight contributed to the loss, by tracing gradients backward through the same computation graph. Understanding the forward pass precisely is prerequisite to understanding backpropagation.
Deep Learning cluster — the MLP is the baseline architecture. CNNs add spatial weight sharing (translational invariance for images). RNNs add recurrence (temporal structure for sequences). The Transformer replaces recurrence with attention, allowing direct long-range dependencies. All of these are specializations of the basic feedforward architecture introduced here.
Neuroscience — the artificial neuron is loosely inspired by the biological neuron: weighted inputs summed over a threshold, nonlinear activation. But the analogy is shallow. Real neurons use spike timing, neuromodulation, and dendritic computation that standard artificial neurons do not capture. The value of the neuroscience inspiration was heuristic, not mechanistic.
