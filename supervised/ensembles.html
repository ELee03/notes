7 · Ensemble Methods
Bagging · Random Forests · AdaBoost · Gradient Boosting · XGBoost · Stacking
Ensemble methods combine many weak models into one strong model. The idea is simple: individual predictors make different errors, and if those errors are uncorrelated, averaging cancels them out. The same principle explains why asking 100 independent people to estimate a quantity is better than asking one. This lesson derives the variance-reduction guarantee of bagging from first principles, explains Random Forests as decorrelated bagging, derives AdaBoost as an exponential loss minimiser, and develops gradient boosting as functional gradient descent in prediction space. Together, these methods — especially gradient-boosted decision trees (XGBoost, LightGBM, CatBoost) — are the most consistently strong performers on structured tabular data in practical ML, routinely winning Kaggle competitions and industrial deployments.
Why combine models?
A single complex model faces a fundamental tension: high capacity models fit the training data well (low bias) but vary wildly across different training sets (high variance). The ensemble approach breaks this tradeoff. If you can train many models that are collectively diverse (their errors are not all the same), their average is more stable and more accurate than any individual.
Bias-variance decomposition for ensembles
Variance reduction by averaging

Suppose we train $M$ independent models $f_1, \ldots, f_M$, each with bias $B$, variance $\sigma^2$, and pairwise correlation $\rho = \text{Corr}(f_i(\mathbf{x}), f_j(\mathbf{x}))$ for $i \neq j$. The ensemble prediction is $\bar{f} = \frac{1}{M}\sum_m f_m$.

Bias of ensemble: $\text{Bias}(\bar{f}) = B$ (averaging does not change bias).

Variance of ensemble: $$\text{Var}(\bar{f}) = \frac{1}{M^2}\text{Var}\!\left(\sum_m f_m\right) = \frac{1}{M^2}\left[M\sigma^2 + M(M-1)\rho\sigma^2\right] = \frac{\sigma^2}{M} + \frac{(M-1)}{M}\rho\sigma^2$$

As $M \to \infty$: $$\text{Var}(\bar{f}) \to \rho\sigma^2$$ If models are uncorrelated ($\rho = 0$): variance $\to 0$ as $M \to \infty$ — perfect variance elimination. If models are perfectly correlated ($\rho = 1$): variance $= \sigma^2$ — averaging does nothing.

The fundamental requirement: ensemble members must be diverse (low $\rho$). High-variance, diverse models are the best candidates for ensembling. Bagging achieves diversity through training set randomness; Random Forests add feature randomness; boosting achieves a different kind of diversity by sequentially correcting residuals.

Bagging (Bootstrap Aggregating)
Bagging (Breiman 1994) achieves diversity via bootstrap sampling: draw $M$ training sets of size $N$ by sampling with replacement from the original $N$ training points. Each bootstrap set omits (on average) $1 - (1-1/N)^N \approx 1 - e^{-1} \approx 36.8\%$ of original training points and includes duplicates of others. Train one model on each bootstrap set. The ensemble prediction is the average (regression) or majority vote (classification).
Out-of-bag (OOB) error. Each model was trained without seeing the $\approx 36.8\%$ of examples that were not included in its bootstrap set. These out-of-bag examples act as a built-in validation set. For each training point $i$, average the predictions only of the models whose bootstrap set did not include point $i$. This gives an OOB prediction for every training point without any held-out set — an essentially free cross-validation estimate. OOB error is almost as accurate as 5-fold cross-validation.
Random forests
The key problem with bagging trees: the bootstrap samples are highly correlated (they all come from the same distribution), so the trees are correlated. From the variance formula, $\rho$ is bounded away from 0 — variance reduction is limited. Random Forests (Breiman 2001) add a second source of randomness: at each split in each tree, only a random subset of $m$ features (out of $p$ total) are considered as candidates for the split. This feature subsampling decorrelates the trees, reducing $\rho$ significantly.
Feature subsampling rule of thumb: Classification: $m = \lfloor\sqrt{p}\rfloor$. Regression: $m = \lfloor p/3 \rfloor$. These are starting points — tune as a hyperparameter. Smaller $m$ = more decorrelation = lower variance, but higher bias (each tree sees less information). The optimal $m$ balances this tradeoff.
Feature importance in Random Forests. Two measures: (1) Mean decrease in impurity (MDI): sum the impurity reductions for splits using each feature, averaged over all trees. Fast to compute (already available from training), but biased toward high-cardinality continuous features. (2) Permutation importance: for each feature, randomly permute its values in the OOB set, measure the decrease in OOB accuracy, and average over trees. Model-agnostic and unbiased, but requires one OOB evaluation per feature.
Random Forests are robust and easy to tune. They require minimal preprocessing (no feature scaling, handle mixed types, robust to outliers), rarely overfit with more trees, and are embarrassingly parallelisable (each tree is independent). The main hyperparameter to tune is $m$ (feature subset size); $M$ (number of trees) is essentially "more is better" up to computational limits.
Boosting — the idea
Bagging reduces variance by averaging parallel models. Boosting reduces both bias and variance by training models sequentially, each one correcting the residual errors of the ensemble so far. Where bagging works by averaging, boosting works by addition: the final model is an additive combination $F(\mathbf{x}) = \sum_{m=1}^M \beta_m h_m(\mathbf{x})$ where each $h_m$ is a "weak learner" (typically a shallow tree) fitted to the current errors.
AdaBoost
AdaBoost derivation (Freund & Schapire 1997)

Binary classification: $y_i \in \{-1, +1\}$. AdaBoost builds a model $F_M(\mathbf{x}) = \sum_{m=1}^M \alpha_m h_m(\mathbf{x})$ where each $h_m$ is a binary classifier.

Algorithm: Initialise sample weights $w_i^{(1)} = 1/N$ for all $i$. For $m = 1, \ldots, M$:

  1. Train weak classifier $h_m$ on training data with weights $w_i^{(m)}$ (minimise weighted 0-1 loss: $\sum_i w_i^{(m)} \mathbf{1}[h_m(\mathbf{x}_i) \neq y_i]$).
  2. Compute weighted error rate: $\varepsilon_m = \sum_{i: h_m(\mathbf{x}_i) \neq y_i} w_i^{(m)}$
  3. Compute classifier weight: $\alpha_m = \frac{1}{2}\ln\frac{1-\varepsilon_m}{\varepsilon_m}$
  4. Update sample weights: $w_i^{(m+1)} \propto w_i^{(m)} \exp(-\alpha_m y_i h_m(\mathbf{x}_i))$ (normalise so they sum to 1)

Weights increase for misclassified examples ($y_i h_m = -1 \to e^{\alpha_m}$); decrease for correctly classified ($y_i h_m = +1 \to e^{-\alpha_m}$). Subsequent weak learners must focus on the hard examples — those misclassified by earlier learners.

Loss function interpretation. AdaBoost minimises the exponential loss: $$\mathcal{L}(F) = \frac{1}{N}\sum_i \exp(-y_i F(\mathbf{x}_i))$$ The greedy forward stagewise algorithm exactly minimises this loss at each step. Proof: at stage $m$, adding $\alpha h$ to $F_{m-1}$ minimises: $$\sum_i e^{-y_i F_{m-1}(\mathbf{x}_i)} e^{-\alpha y_i h(\mathbf{x}_i)} = \sum_{y_i h = +1} w_i e^{-\alpha} + \sum_{y_i h = -1} w_i e^{\alpha}$$ where $w_i = e^{-y_i F_{m-1}(\mathbf{x}_i)}$. Setting the derivative to zero gives $\alpha_m = \frac{1}{2}\ln\frac{1-\varepsilon}{\varepsilon}$ and the minimum $h$ minimises the weighted error $\varepsilon = \sum_{y_i \neq h_i} w_i / \sum w_i$ — exactly the AdaBoost update.

Gradient boosting
Gradient boosting — functional gradient descent (Friedman 2001)

Gradient boosting generalises AdaBoost to arbitrary differentiable loss functions by viewing boosting as gradient descent in the space of functions.

The additive model is $F_M(\mathbf{x}) = \sum_{m=0}^M h_m(\mathbf{x})$ (note: no separate $\alpha_m$ — the step size is absorbed into $h_m$).

At step $m$, we want to add $h_m$ to reduce the empirical risk: $$\mathcal{L}(F) = \sum_{i=1}^N \ell(y_i, F(\mathbf{x}_i))$$ The direction of steepest descent in function space is the negative functional gradient: $$r_i^{(m)} = -\frac{\partial \ell(y_i, F(\mathbf{x}_i))}{\partial F(\mathbf{x}_i)}\bigg|_{F = F_{m-1}}$$ These are called pseudo-residuals.

Since we can't follow the functional gradient directly (it's only defined at training points, not everywhere), we fit a regression tree $T_m$ to the pseudo-residuals: $$T_m = \arg\min_{T} \sum_i (r_i^{(m)} - T(\mathbf{x}_i))^2$$ Update: $F_m = F_{m-1} + \eta T_m$ where $\eta$ is the learning rate (shrinkage).

Loss function → pseudo-residuals:

  • MSE ($\ell = (y-F)^2/2$): $r_i = y_i - F(\mathbf{x}_i)$ — actual residuals.
  • MAE ($\ell = |y-F|$): $r_i = \text{sign}(y_i - F(\mathbf{x}_i))$
  • Log-loss ($\ell = \log(1+e^{-yF})$): $r_i = y_i - \sigma(F(\mathbf{x}_i))$ — probability residuals.
  • Huber (robust): interpolates between MSE for small residuals and MAE for large ones.

Algorithm: Initialise $F_0 = \arg\min_c \sum_i \ell(y_i, c)$ (constant predictor: mean for MSE, log-odds for log-loss). For $m = 1, \ldots, M$:

  1. Compute pseudo-residuals $r_i^{(m)}$ for each training point.
  2. Fit a regression tree $T_m$ with $J$ leaves to $\{(\mathbf{x}_i, r_i^{(m)})\}$.
  3. Compute optimal leaf values: for each leaf $j$, set $\gamma_{jm} = \arg\min_\gamma \sum_{i \in \text{leaf}_j} \ell(y_i, F_{m-1}(\mathbf{x}_i) + \gamma)$.
  4. Update: $F_m(\mathbf{x}) = F_{m-1}(\mathbf{x}) + \eta \sum_j \gamma_{jm} \mathbf{1}[\mathbf{x} \in \text{leaf}_j]$.

Regularisation in gradient boosting:
  • Shrinkage (learning rate $\eta$): smaller $\eta$ requires more trees but better generalises. Typical: $\eta \in [0.01, 0.3]$.
  • Subsampling (stochastic GBT): fit each tree on a random subsample of training points (fraction $s \in (0.5, 1)$). Reduces variance and speeds training.
  • Tree depth: shallow trees ($J = 3$–$8$ leaves) are standard. Depth controls the order of feature interactions captured.
XGBoost
XGBoost (Chen & Guestrin 2016) extends gradient boosting with a second-order Taylor expansion of the loss, which gives a closed-form optimal leaf value and a principled regularised structure score: $$\mathcal{L}^{(m)} \approx \sum_i \left[g_i f_m(\mathbf{x}_i) + \frac{1}{2}h_i f_m(\mathbf{x}_i)^2\right] + \Omega(f_m)$$ where $g_i = \partial_F \ell(y_i, F(\mathbf{x}_i))$ (first derivative — the pseudoresidual), $h_i = \partial^2_F \ell(y_i, F(\mathbf{x}_i))$ (second derivative — Hessian), and $\Omega(f) = \gamma |T| + \frac{\lambda}{2}\sum_j w_j^2$ is the regularisation term (penalising tree size and leaf weights).
The optimal leaf weight for leaf $j$ is $w_j^* = -G_j / (H_j + \lambda)$ where $G_j = \sum_{i \in \text{leaf}_j} g_i$ and $H_j = \sum_{i \in \text{leaf}_j} h_i$. The gain for a particular split is: $$\text{Gain} = \frac{1}{2}\left[\frac{G_L^2}{H_L+\lambda} + \frac{G_R^2}{H_R+\lambda} - \frac{(G_L+G_R)^2}{H_L+H_R+\lambda}\right] - \gamma$$ This analytical formula makes split finding much faster (no line search needed per split). XGBoost also uses approximate quantile-based sketching for even faster split finding on large datasets, column subsampling (like Random Forests), and sparsity-aware algorithms for missing values. These engineering improvements made it 10–50× faster than vanilla gradient boosting and the dominant method on tabular Kaggle competitions from 2015–2022.
Stacking (Stacked Generalisation)
Bagging and boosting combine models of the same type. Stacking (Wolpert 1992) combines models of different types using a learned meta-model. The procedure:
  1. Train level-0 models (e.g., Random Forest, SVM, GBT, logistic regression) on the training set using cross-validation to generate out-of-fold predictions.
  2. Build a meta-feature matrix: each column is the out-of-fold predictions of one level-0 model. This has the same number of rows as the training set, with one column per level-0 model.
  3. Train a meta-model (level-1) on the meta-feature matrix to predict the true targets. The meta-model learns how to best combine the level-0 predictions.
Stacking typically outperforms any individual model because different models make different errors, and the meta-model learns to exploit this. The cross-validation step is essential: without it, level-0 models have seen their own training targets and the meta-features are overfit, leading to a degenerate meta-model.
Worked examples
Example 1 — Variance reduction by averaging independent trees

You train $M = 100$ decision trees on bootstrap samples. Each tree has bias $B = 0.2$ and variance $\sigma^2 = 4.0$. After bootstrap sampling, the inter-tree correlation is $\rho = 0.5$. Compute the ensemble's bias, variance, and approximate MSE. Compare to a single tree and to hypothetically uncorrelated trees.

Single tree: MSE $\approx B^2 + \sigma^2 = 0.04 + 4.0 = 4.04$.

Ensemble ($M=100$, $\rho=0.5$): Bias unchanged: $B = 0.2$. Variance: $\frac{\sigma^2}{M} + \frac{M-1}{M}\rho\sigma^2 = \frac{4.0}{100} + \frac{99}{100}(0.5)(4.0) = 0.04 + 1.98 = 2.02$. MSE $\approx B^2 + 2.02 = 0.04 + 2.02 = 2.06$ — roughly halved.

Hypothetically uncorrelated trees ($\rho=0$): Variance: $\sigma^2/M = 4.0/100 = 0.04$. MSE $\approx 0.04 + 0.04 = 0.08$ — 50× reduction vs. single tree. This is the ceiling that Random Forests approach by decorrelating trees via feature subsampling.

The gap between $\rho=0.5$ (plain bagging) and $\rho=0$ shows why feature subsampling matters: reducing tree correlation from 0.5 to near 0 reduces variance from 2.02 to 0.04 — a 50× reduction in the variance contribution.

Example 2 — AdaBoost iteration by hand

4 training points: $(\mathbf{x}_1, +1), (\mathbf{x}_2, +1), (\mathbf{x}_3, -1), (\mathbf{x}_4, -1)$. Weak learner $h_1$ classifies: $h_1(\mathbf{x}_1)=+1$ ✓, $h_1(\mathbf{x}_2)=+1$ ✓, $h_1(\mathbf{x}_3)=-1$ ✓, $h_1(\mathbf{x}_4)=+1$ ✗ (misclassified). Run one AdaBoost iteration.

Initial weights: $w_i^{(1)} = 1/4$ for all $i$.

Weighted error: Only $\mathbf{x}_4$ is misclassified. $\varepsilon_1 = w_4^{(1)} = 1/4 = 0.25$.

Classifier weight: $\alpha_1 = \frac{1}{2}\ln\frac{1-0.25}{0.25} = \frac{1}{2}\ln(3) \approx 0.549$.

Update weights: $w_i^{(2)} \propto w_i^{(1)} \exp(-\alpha_1 y_i h_1(\mathbf{x}_i))$

  • $\mathbf{x}_1$: $y_1 h_1 = +1$. $w_1^{(2)} \propto (1/4)e^{-0.549} \propto 0.144$
  • $\mathbf{x}_2$: $y_2 h_1 = +1$. $w_2^{(2)} \propto 0.144$
  • $\mathbf{x}_3$: $y_3 h_1 = +1$. $w_3^{(2)} \propto 0.144$
  • $\mathbf{x}_4$: $y_4 h_1 = -1$. $w_4^{(2)} \propto (1/4)e^{+0.549} \propto 0.433$
Normalise: total $= 0.144 \times 3 + 0.433 = 0.865$. Normalised: $w_{1,2,3}^{(2)} = 0.167$, $w_4^{(2)} = 0.5$.

The misclassified point $\mathbf{x}_4$ now has weight 0.5 (half the total weight, up from 0.25). The next weak learner trained on these weights must focus on $\mathbf{x}_4$ — it will make a large error if it ignores this heavy point. This is AdaBoost's sequential correction mechanism.

Example 3 — Gradient boosting step for regression

Regression with MSE loss. Current model $F_0(\mathbf{x}) = 5$ (constant, the mean of $y = [3, 5, 7, 9]$). $\eta = 0.5$. Show the first boosting step and compute the new predictions.

Targets: $y = [3, 5, 7, 9]$. Initial model: $F_0 = \bar{y} = 6$. (mean of all 4 targets = 24/4 = 6)

Pseudo-residuals (for MSE: $r_i = y_i - F_0$): $r = [3-6, 5-6, 7-6, 9-6] = [-3, -1, +1, +3]$

Fit regression tree $T_1$ to residuals: Suppose $T_1$ perfectly fits the residuals: $T_1(\mathbf{x}_i) = r_i = [-3, -1, +1, +3]$ (or in a real problem, finds the best split on the feature values).

Update: $F_1(\mathbf{x}_i) = F_0 + \eta \cdot T_1(\mathbf{x}_i)$ $= 6 + 0.5 \cdot [-3, -1, +1, +3] = [4.5, 5.5, 6.5, 7.5]$

New residuals for step 2: $r^{(2)} = y - F_1 = [3-4.5, 5-5.5, 7-6.5, 9-7.5] = [-1.5, -0.5, +0.5, +1.5]$

The residuals halved after one step (due to $\eta = 0.5$). After $M$ steps with $\eta = 0.5$ and a perfect tree, residuals are $(1-\eta)^M = 0.5^M$ of the original — exponential decay. Smaller $\eta$ requires more steps but generalises better (the slow decay regularises).

Food for thought
  • Gradient boosting with MSE loss fits each tree to the residuals $y_i - F_{m-1}(\mathbf{x}_i)$. This is the same as what a neural network does when trained with gradient descent on MSE. What's the key difference in how they represent and update the model?
    In gradient boosting, the model is an additive tree ensemble: each step adds a whole new tree $T_m$. The model grows in discrete units (trees). Each tree has $O(2^J)$ parameters (leaf values and split structure), and once added, a tree's split structure is fixed — only the leaf values could be further tuned. In a neural network, gradient descent updates all parameters $\theta$ simultaneously via $\theta \leftarrow \theta - \eta \nabla_\theta \mathcal{L}$. Every weight changes at every step. The model is differentiable everywhere (with smooth activations), enabling fine-grained, coordinated updates across all layers. Key differences: (1) Gradient boosting parameters are discrete trees — you can't differentiate through a tree to compute the globally optimal next tree, only through individual leaves. (2) Boosting is a coordinate-wise approach in function space (each tree is one coordinate); neural network SGD is a gradient step in weight space. (3) Trees are piece-wise constant functions; neural networks with smooth activations approximate smooth functions. Neural networks excel on unstructured data (images, text) where smoothness is natural; GBDTs excel on structured tabular data where individual features may have non-smooth effects.
  • Random Forests are trained in parallel (each tree independently) and are easy to parallelise. Gradient Boosting is sequential (tree $m$ depends on the residuals of tree $m-1$). Is there any way to parallelise gradient boosting?
    The sequential dependency in gradient boosting is between trees (each tree depends on the previous ones to compute residuals). You cannot train tree $m+1$ until tree $m$ is done. However, two types of parallelism are still available: (1) Within-tree parallelism: Building each tree is embarrassingly parallel — each candidate split can be evaluated independently across features and thresholds. XGBoost, LightGBM, and CatBoost all parallelise the split-finding inner loop across CPU/GPU cores. Building a single tree of depth $J$ with $N$ points can scale nearly linearly with cores. (2) Approximate methods: Some algorithms (DART — Dropouts meet Multiple Additive Regression Trees) randomly drop trees during training, which reduces their mutual dependence and can be partially parallelised. Feature-parallel and data-parallel implementations of gradient boosting (LightGBM uses histogram-based approximate splits in a distributed setting) scale to very large datasets. But fundamentally, the sequential tree dependency limits total parallelism more than Random Forests. For very large training jobs (100M+ examples), Random Forests scale more cleanly; gradient boosting requires careful distributed implementation.
  • Neural networks have largely supplanted tree ensembles on unstructured data (images, audio, text). Yet tree ensembles (especially XGBoost and LightGBM) still dominate on structured tabular data, even where deep learning is applied. Why do trees have such a persistent advantage on tabular data?
    This is an empirically observed fact confirmed across many benchmarks (Grinsztajn et al. 2022, "Why do tree-based models still outperform deep learning on tabular data?"). Several explanations are offered: (1) Tabular data often contains uninformative features: GBDTs naturally ignore useless features via their feature importance mechanism (irrelevant features rarely get split on). Neural networks have no such natural selection and must learn to ignore irrelevant inputs via regularisation, which requires more data and careful tuning. (2) Rotational invariance: neural networks are typically invariant to rotations of the input space. Tabular features are often meaningful as individual variables (age, income), not as linear combinations. Trees are not rotationally invariant — they respect individual feature thresholds natively. (3) Sample efficiency: GBDT convergence is typically rapid in the number of boosting rounds. Neural networks require more data and epochs to reach comparable performance on small datasets. (4) Robustness: GBDTs handle missing values, mixed data types, and class imbalance robustly. Neural networks require preprocessing (encoding, normalisation, imputation) that introduces additional choices and potential errors. Deep learning approaches that add table-specific inductive biases (TabNet, SAINT, FT-Transformer) have narrowed the gap but not consistently closed it.
Common pitfalls
  • Not using early stopping in gradient boosting. Unlike Random Forests (more trees = better until compute runs out), gradient boosting can overfit if the number of trees is too large. Always use early stopping: monitor validation loss and stop when it stops improving. XGBoost and LightGBM have native early stopping support.
  • Treating feature importances as causal. Both MDI (mean decrease in impurity) and permutation importances measure predictive contribution, not causal effect. A feature that is correlated with the target but not causal (e.g., hospital ward predicts mortality because sicker patients are routed there) will have high importance. Use causal inference methods (instrumental variables, DoWhy) for causal questions.
  • Using the same data for OOB evaluation and final performance reporting. OOB error is a valid cross-validation estimate for Random Forests. But it uses the training set — if you select hyperparameters based on OOB error, the OOB estimate is no longer unbiased. For final performance reporting, always hold out a separate test set that was not used for any part of model selection.
  • Not tuning learning rate and number of trees jointly in gradient boosting. The optimal number of trees $M$ depends heavily on $\eta$ (learning rate). Small $\eta$ requires large $M$ to achieve the same fit; large $\eta$ converges in fewer trees but may overfit. The right procedure: set $\eta$ small (e.g., 0.05–0.1), use early stopping to find $M$, then tune other hyperparameters. Do not tune $M$ as a standalone parameter.
Connects to
Decision Trees (SL6) — this lesson is the direct extension. Every ensemble method here (bagging, random forests, gradient boosting) uses trees as the base learner. The CART algorithm, Gini impurity, and tree variance are prerequisites.
Training Neural Networks (SL10) — gradient boosting uses the same loss functions, pseudo-residual concepts, and regularisation principles (learning rate, early stopping) as neural network training. The functional gradient descent view of boosting and parameter-space gradient descent in neural networks are parallel ideas.
Deep Learning (DL cluster) — the distinction between ensemble methods and deep learning is the primary fork in modern ML practice: tabular data → GBDTs; unstructured data (images, audio, text) → deep networks. Understanding both is necessary for choosing the right tool and for understanding the ongoing research into tabular deep learning.
