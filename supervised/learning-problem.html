1 · The Learning Problem
Hypothesis Spaces · Empirical Risk · True Risk · Generalisation · Inductive Bias · No Free Lunch
Every supervised learning algorithm is trying to do the same thing: given examples of inputs paired with outputs, infer a function that generalizes to unseen inputs. The challenge is not fitting the training data — any algorithm can do that. The challenge is building a function that captures the actual underlying relationship, not the noise in your particular sample.

What is supervised learning?
Supervised learning is the problem of learning a mapping from inputs to outputs, given a dataset of examples of that mapping. You observe $n$ pairs $$\{(x_1, y_1),\; (x_2, y_2),\; \ldots,\; (x_n, y_n)\}$$ where $x_i \in \mathcal{X}$ are the inputs (also called features or covariates) and $y_i \in \mathcal{Y}$ are the outputs (also called labels or targets). The goal is to find a function $h : \mathcal{X} \to \mathcal{Y}$ that accurately predicts $y$ from $x$ for new, unseen examples — examples not in the training set.
The word supervised refers to the labels. During training, a "supervisor" provides the correct output for each input — in contrast to unsupervised learning, where no labels exist, and reinforcement learning, where a reward signal replaces explicit labels.
Two families of tasks fall under supervised learning:
  • Regression — the output is continuous: $\mathcal{Y} \subseteq \mathbb{R}$ (or $\mathbb{R}^k$ for multiple outputs). Predicting house prices, tomorrow's temperature, a patient's blood glucose level.
  • Classification — the output is discrete: $\mathcal{Y} = \{1, 2, \ldots, K\}$ for $K$ classes. Spam detection ($K = 2$), handwritten digit recognition ($K = 10$), species identification ($K = $ hundreds).
Under the hood, both tasks share the same formal structure. The difference lies only in the choice of output space and loss function.
The three machine learning paradigms
Machine learning is divided into three broad paradigms, distinguished by the kind of feedback the algorithm receives during training.
Supervised learning — the algorithm sees labeled input-output pairs. A supervisor tells it the correct answer for each training example. This is the richest form of feedback and produces the most reliable results when labeled data is available. The challenge is that labeling data is expensive: a radiologist must read each X-ray, a linguist must annotate each sentence.
Unsupervised learning — the algorithm sees only unlabeled inputs $\{x_1, \ldots, x_n\}$, with no $y$. The goal shifts from prediction to structure discovery: find clusters in the data (k-means), find low-dimensional representations (PCA, autoencoders), or learn the distribution $P(X)$ itself (generative models). The challenge is that "success" is harder to define without labels.
Reinforcement learning — there is no dataset at all. An agent interacts with an environment, takes actions, and receives scalar rewards. The goal is to learn a policy that maximizes cumulative reward. The challenge is the credit assignment problem: which actions in a long sequence were responsible for a reward that arrived much later? RL is covered in its own cluster.
This cluster focuses exclusively on supervised learning. Much of the mathematical machinery — loss functions, gradient descent, generalization theory — carries over to the other paradigms, but supervised learning is the cleanest setting in which to develop the ideas.
The formal framework
To reason carefully about what an algorithm can and cannot learn, we need a precise formulation. Every supervised learning problem has five ingredients:
1. Input space $\mathcal{X}$. The set of all possible inputs. For tabular data with $d$ numerical features, $\mathcal{X} = \mathbb{R}^d$. For images, it might be the space of $28 \times 28$ pixel grids. For text, the space of all sequences of tokens. The structure of $\mathcal{X}$ constrains which algorithms are natural to use.
2. Output space $\mathcal{Y}$. For regression, $\mathcal{Y} = \mathbb{R}$. For binary classification, $\mathcal{Y} = \{0, 1\}$. For multi-class classification, $\mathcal{Y} = \{1, \ldots, K\}$.
3. The unknown distribution $P(X, Y)$. This is the heart of the problem. We assume the training data was generated by sampling $n$ pairs independently and identically (i.i.d.) from some fixed but unknown distribution $P(X, Y)$ over $\mathcal{X} \times \mathcal{Y}$. The distribution $P$ encodes the true relationship between inputs and outputs — including the irreducible noise. We never observe $P$ directly; we see only a finite sample from it.
4. Hypothesis class $\mathcal{H}$. Not all functions $h : \mathcal{X} \to \mathcal{Y}$ are equally useful or equally tractable. In practice we restrict attention to some family $\mathcal{H}$: perhaps all linear functions, all decision trees of depth at most 5, or all neural networks with a given architecture. This choice encodes our prior beliefs about what kind of relationship we expect between $X$ and $Y$. It is arguably the most important design decision in any machine learning problem.
5. Loss function $\mathcal{L} : \mathcal{Y} \times \mathcal{Y} \to \mathbb{R}_{\geq 0}$. Given a prediction $\hat{y} = h(x)$ and the true label $y$, the loss $\mathcal{L}(y, \hat{y})$ measures how bad the prediction was. Common choices:
  • Squared error (regression): $\mathcal{L}(y, \hat{y}) = (y - \hat{y})^2$. Penalizes large errors severely — outliers have disproportionate influence.
  • Absolute error (regression): $\mathcal{L}(y, \hat{y}) = |y - \hat{y}|$. More robust to outliers than squared error, but not differentiable at zero.
  • 0-1 loss (classification): $\mathcal{L}(y, \hat{y}) = \mathbf{1}[y \neq \hat{y}]$. Simply counts mistakes, but is not differentiable — hard to optimize directly.
  • Cross-entropy loss (classification): $\mathcal{L}(y, \hat{p}) = -\log \hat{p}_y$, where $\hat{p}_y$ is the predicted probability assigned to the true class. Differentiable and well-suited to gradient-based optimization. More on this in Lesson 3.
With these five ingredients, the true risk (or expected loss) of a hypothesis $h$ is defined as the expected loss over the distribution $P$: $$R(h) \;=\; \mathbb{E}_{(X,\,Y) \sim P}\bigl[\mathcal{L}(h(X),\, Y)\bigr]$$ The ideal goal is to find $$h^* \;=\; \operatorname{arg\,min}_{h \,\in\, \mathcal{H}}\; R(h)$$ That is, the function in $\mathcal{H}$ with the smallest expected loss under the true distribution.
The problem, of course, is that we cannot compute $R(h)$ — we don't know $P$. This is the fundamental obstacle that makes supervised learning non-trivial.
Empirical risk minimization
The standard response to not knowing $P$ is to replace the expectation over $P$ with an average over the training data. Define the empirical risk: $$\hat{R}(h) \;=\; \frac{1}{n} \sum_{i=1}^n \mathcal{L}(h(x_i),\, y_i)$$ This is simply the average loss on the training set — what practitioners call training error or training loss. The Empirical Risk Minimization (ERM) principle says: find the hypothesis in $\mathcal{H}$ that minimizes the empirical risk: $$\hat{h} \;=\; \operatorname{arg\,min}_{h \,\in\, \mathcal{H}}\; \hat{R}(h)$$
Why does this work? By the Law of Large Numbers, for any fixed $h$, the empirical average converges to the true expectation as the sample size grows: $$\hat{R}(h) \;\xrightarrow{n \to \infty}\; R(h) \quad \text{almost surely}$$ If this convergence were uniform across all $h \in \mathcal{H}$ simultaneously — meaning $\hat{R}(h) \approx R(h)$ for every $h$ at once — then minimizing $\hat{R}$ would approximately minimize $R$. Whether this uniform convergence holds, and how many samples it requires, depends critically on the complexity of $\mathcal{H}$.
For small hypothesis classes, uniform convergence comes cheaply. For large or infinitely complex classes — like all possible neural networks — it requires much more data. Generalization theory (VC dimension, Rademacher complexity) quantifies exactly this tradeoff, though those tools are beyond our scope here.
ERM in practice: Even finding $\hat{h}$ exactly is often computationally intractable. For neural networks, gradient descent finds a local minimum of $\hat{R}$ — not the global minimum. The distinction between "minimizer of empirical risk" (a theoretical object) and "what gradient descent finds" (a practical one) matters. We return to this in Lesson 9.
The generalization question
The central question of supervised learning: when does minimizing training error lead to low test error?
Write the test error of $\hat{h}$ as: $$\underbrace{R(\hat{h})}_{\text{test error}} \;=\; \underbrace{\bigl[R(\hat{h}) - \hat{R}(\hat{h})\bigr]}_{\text{generalization gap}} \;+\; \underbrace{\hat{R}(\hat{h})}_{\text{training error}}$$ The test error decomposes into two parts. The generalization gap measures how much the training and test errors differ — it is always non-negative, since $\hat{h}$ was chosen to minimize training error (so it can only look artificially good on training data). The training error measures how well we fit the training set.
Two failure modes:
  • Overfitting. Low training error, large generalization gap. The model has memorized the training data, including its noise, and does not generalize. Classic symptom: near-perfect training accuracy, poor test accuracy. Occurs when $\mathcal{H}$ is too expressive relative to the amount of data.
  • Underfitting. High training error. The model cannot capture the true relationship even in the training data — $\mathcal{H}$ is not expressive enough. Adding more data does not help; the model is limited by its own capacity.
A good model sits between these extremes: low training error and a small generalization gap, so the test error is also low.
This gives rise to the fundamental tension in supervised learning. There are two ways to think about it, and they are equivalent:
  • Model complexity view. A complex model (large $\mathcal{H}$) can fit the training data well but tends to overfit. A simple model (small $\mathcal{H}$) underfits but generalizes stably.
  • Bias-variance view. A complex model has low bias (can approximate almost anything) but high variance (its output changes drastically with different training sets). A simple model has high bias (systematically wrong) but low variance (stable). We will formalize this decomposition precisely in Lesson 4.
The art of supervised learning is choosing a model complex enough to capture the true structure, but simple enough to generalize from the available data.
Hypothesis class and inductive bias
The choice of $\mathcal{H}$ encodes assumptions about the problem. This set of assumptions is called the algorithm's inductive bias.
  • Linear models assume the output is a linear function of the input features. Strong assumption — often wrong — but makes the problem tractable and generalizes stably with limited data.
  • Decision trees assume the function is piecewise constant with axis-aligned boundaries. No continuity assumption — handles categorical features naturally but can overfit badly.
  • SVMs (Lesson 5) assume the classes are separable by a large-margin boundary. The margin assumption acts as a regularizer, controlling generalization.
  • Neural networks make very weak structural assumptions — the architecture (number of layers, type of activations) is the main constraint. This makes them highly expressive and widely applicable, but they require large amounts of data to use their flexibility well.
A common misconception is that neural networks "make no assumptions." They do — the architecture encodes translation invariance (CNNs), sequential structure (RNNs), or general function approximation (MLPs). The optimizer, initialization, and regularization all implicitly further constrain which functions are found. The difference is not that neural networks are assumption-free; it is that their assumptions are weaker and more general than classical models.
Crucially, inductive bias is not a weakness. It is the mechanism by which algorithms generalize. A learner with no prior assumptions about the problem cannot generalize at all from a finite sample — any unseen input is equally consistent with the training data under all possible functions. Inductive bias is what makes generalization possible.
The No Free Lunch theorem
The previous discussion raises a natural question: is there an optimal inductive bias? Is there an algorithm that is universally best?
The answer is no. The No Free Lunch (NFL) theorem (Wolpert, 1996) states, informally, that averaged over all possible data-generating distributions $P$, every learning algorithm performs identically — no better than random guessing. Any algorithm that does well on some distributions must do poorly on others.
The proof sketch is elegant. For any algorithm $A$ that performs well on some set of distributions $\mathcal{P}_{\text{good}}$ (where its inductive bias is well-matched), one can construct a complementary set $\mathcal{P}_{\text{bad}}$ where $A$ performs poorly — and the two sets are symmetric enough that the average performance is the same for every algorithm.
This might seem discouraging. But it is not, once you read the fine print: averaged over all possible problems. In practice, the problems we encounter are not drawn uniformly from all conceivable distributions. They are structured — images are spatially local, text has grammatical regularities, physical systems are smooth. Algorithms succeed because their inductive biases are calibrated to this structure.
The NFL theorem is not a license for nihilism about algorithm selection. It is a precise statement that success requires assumptions — and the assumptions must match the problem. The question "which algorithm should I use?" is inseparable from "what do I know about this problem's structure?"
NFL and neural networks: Deep neural networks have dominated across extraordinarily diverse domains — vision, speech, language, protein structure, board games. This does not contradict NFL. It says that neural networks' implicit inductive biases (smoothness under gradient descent, local composition via deep layers, data augmentation symmetries) happen to align with the structure of many real-world problems. On a problem with genuinely random structure, a neural network trained by gradient descent would fail just as badly as any other method.
Worked examples
Example 1 — Choosing the hypothesis class

You are predicting whether a loan applicant will default (yes/no) based on three numerical features: credit score, annual income, and loan amount. You have 10,000 labeled examples.

Compare the following choices of $\mathcal{H}$: (a) all linear functions $\mathcal{X} \to \mathbb{R}$, thresholded at 0; (b) all decision trees of depth at most 20; (c) all neural networks with 10 hidden layers and 1,000 units each.

For each, describe the inductive bias and predict whether overfitting or underfitting is more likely on this dataset.

(a) Linear threshold. Assumes the default boundary is a hyperplane in 3D feature space. Likely to underfit if the true boundary is nonlinear (e.g., defaults cluster in a region with both low income and high loan amount, not captured by a single plane). But with 10,000 examples and only 3 features, this model is unlikely to overfit. Low variance, potentially high bias.

(b) Depth-20 decision tree. Can represent up to $2^{20} \approx 10^6$ distinct leaf regions — far more than the 10,000 training examples. With that many parameters relative to data, ERM will typically find a tree that memorizes the training labels. High risk of overfitting. In practice, trees are limited to much smaller depths (5–10) or pruned after fitting.

(c) Large neural network. Potentially millions of parameters on 10,000 examples. With standard training (gradient descent, dropout, regularization), modern neural networks can generalize despite massive overparameterization — this is a surprising empirical fact called benign overfitting, currently an active area of theory. But without regularization, this network will almost certainly overfit on 10,000 examples.

Takeaway: With 10,000 examples and 3 features, the linear model is probably too simple, the depth-20 tree is too complex (without pruning), and the neural network sits in between — powerful enough but needing regularization to generalize.

Example 2 — ERM and the memorization failure mode

You have $n = 5$ training points $\{(x_i, y_i)\}_{i=1}^5$ on the real line. You fit a degree-4 polynomial (which has 5 free parameters: the coefficients $a_0, a_1, a_2, a_3, a_4$). The ERM solution achieves $\hat{R}(\hat{h}) = 0$.

Is this good? What does zero training error tell you about test error?

Zero training error is not good here — it is a warning sign. By the fundamental theorem of algebra, any $n$ points in the plane can be exactly interpolated by a polynomial of degree $n - 1$. With 5 points and a degree-4 polynomial, the ERM solution is the unique polynomial passing through all 5 training points exactly.

But this polynomial oscillates wildly between the training points. If the true relationship is smooth and the $y_i$ contain measurement noise, the interpolating polynomial has memorized the noise, not the signal.

The generalization gap is large: $R(\hat{h}) - \hat{R}(\hat{h}) \approx R(\hat{h})$ since $\hat{R}(\hat{h}) = 0$ but $R(\hat{h})$ is large. Training error of zero tells you nothing about test error when the number of parameters equals the number of training points.

This is the overfitting failure mode in its starkest form. The fix: use a simpler model (lower-degree polynomial), or add regularization to penalize oscillatory solutions (Lesson 4).

Example 3 — Identifying the loss function

A hospital wants a model to diagnose whether a chest X-ray shows pneumonia. A software engineer proposes minimizing the 0-1 loss. A statistician suggests minimizing cross-entropy loss. Who is right, and why?

Both capture the right goal (classify correctly) but the statistician's choice is far more practical.

The 0-1 loss counts mistakes: $\mathcal{L}(y, \hat{y}) = \mathbf{1}[y \neq \hat{y}]$. This is the correct measure of clinical accuracy. But it is discontinuous: a small change in model parameters either changes a prediction (loss jumps by 1) or doesn't (loss stays the same). No gradient exists. Gradient descent — the algorithm that trains almost every modern ML model — cannot optimize 0-1 loss directly.

Cross-entropy loss $\mathcal{L}(y, \hat{p}) = -\log \hat{p}_y$ treats the model output as a probability $\hat{p}_y$ for the true class. It is smooth and differentiable — gradient descent can optimize it continuously. When minimized, it pushes the model toward correct and confident predictions.

The resolution: we train by minimizing cross-entropy (a differentiable surrogate) and evaluate by measuring 0-1 loss (the actual clinical metric). The surrogate loss approximates the true objective closely enough that minimizing it in training usually minimizes 0-1 loss on the test set. This surrogate-loss pattern appears throughout supervised learning — the training objective and the evaluation metric are frequently different.

Food for thought
  • The No Free Lunch theorem says no algorithm is universally best. Yet in practice, neural networks have dominated across vision, language, audio, and scientific domains. Does this contradict NFL? If not, what is the reconciliation?
    NFL says averaged over all possible distributions, every algorithm ties. But real-world problems are not drawn from this uniform distribution over all conceivable functions. They are structured: natural images are locally smooth, language has grammar, physical processes are governed by differential equations. Neural networks' inductive biases — composition of local features, smoothness under gradient descent, invariance from data augmentation — happen to match this structure. NFL applies to the average over all problems; in practice we only care about the structured problems that actually arise. If you constructed a dataset whose labels were truly random (i.i.d. uniform over $\{0,1\}$ regardless of the input), gradient-descent-trained neural networks would fail just as badly as any other method.
  • We said the training data is sampled i.i.d. from $P(X, Y)$. What goes wrong if this assumption is violated — for example, if the training data was collected in 2020 and the model is deployed in 2025?
    When the distribution at deployment (test time) differs from the distribution at training time, we face distribution shift. The theoretical guarantees of ERM — that minimizing $\hat{R}$ minimizes $R$ — assume the training and test examples are drawn from the same distribution. If $P$ changes between training and deployment, then even a model with tiny training error and tiny generalization gap (by the training distribution's standard) can have large test error. In the medical context: a disease prevalence predictor trained on 2020 pandemic-era data may fail badly in 2025 when disease patterns differ. Distribution shift is one of the hardest practical challenges in ML deployment. Techniques like domain adaptation and importance weighting partially address it, but there is no general solution.
  • You train a model to 99% accuracy on the training set. A colleague says "great, that should give us around 95% test accuracy." What assumption is implicit in that estimate, and when would it be badly wrong?
    The implicit assumption is that the generalization gap is small — perhaps 4–5 percentage points. This is reasonable if: (1) the model is not heavily overparameterized relative to the data, (2) regularization was used, and (3) the train/test data are genuinely i.i.d. from the same distribution. It would be badly wrong if: the model memorized the training set (zero training error on a small dataset with a complex model), in which case training accuracy tells you almost nothing about test accuracy; or if the training set was not representative of the test distribution (e.g., all training images from one camera, test images from a different one). In the extreme, a model can achieve 99% training accuracy and near-chance test accuracy.
Common pitfalls
  • Equating training error with test error. Training error measures fit to the data you have. Test error measures generalization to data you don't have. These are different quantities that can diverge dramatically. Always report both.
  • Choosing the hypothesis class after seeing the data. If you fit 20 different model types, pick the one with the best test set performance, and report that performance — you have used your test set as a validation signal. The test set no longer measures generalization to truly unseen data; it measures fit to the test set. This is called "data snooping" and inflates reported accuracy. Use a held-out test set that you look at exactly once.
  • Assuming more data always helps. More data reduces the generalization gap but does not fix underfitting. If your model's hypothesis class cannot represent the true function, adding training examples will not improve matters — the model is limited by its capacity, not by the amount of data. Conversely, no amount of model complexity fixes a dataset that is too small for the problem.
  • Ignoring the i.i.d. assumption. ERM's guarantees require training and test data to come from the same distribution. Real datasets frequently violate this: time-series data has autocorrelation, data from different hospitals has covariate shift, scraped web data has selection bias. Treating distribution shift as a detail to address later is one of the most common sources of real-world ML failures.
  • Treating the loss function as fixed. The loss function is a design choice that encodes what "good" means for your problem. Cross-entropy and squared error are defaults, not the only options. In medical diagnosis, false negatives (missing disease) and false positives (alarming healthy patients) have very different costs. Using an equal-cost loss in an asymmetric problem gives a model optimized for the wrong objective.
Connects to
Probability Theory — the formal foundation. True risk is an expected value over $P$; the Law of Large Numbers is why empirical risk concentrates around true risk; generalization bounds (VC dimension, Rademacher complexity) are probabilistic guarantees on the generalization gap. Understanding those bounds requires measure-theoretic probability.
Information Theory — cross-entropy loss $-\log \hat{p}_y$ is the negative log-likelihood, and minimizing it is equivalent to minimizing the KL divergence from the model distribution to the true one. The connection between information theory and learning is deep: the bits of information in a training set formally bound how much can be learned.
Reinforcement Learning — the RL "policy improvement" loop is a form of ERM over trajectories. The reward signal plays the role of the loss function. The exploration-exploitation dilemma is a variant of the bias-variance tradeoff — exploiting known good actions (low variance) versus exploring for potentially better ones (potentially lower bias).
