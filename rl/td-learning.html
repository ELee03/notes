7 · Temporal Difference Learning
TD(0) · TD Error · SARSA · Q-Learning · n-Step TD · Online Learning
Monte Carlo methods are model-free but require complete episodes. Dynamic programming is exact but requires a complete model. Temporal Difference learning is model-free and bootstraps from incomplete episodes — it is the union of the best properties of both. TD learning is arguably the most central idea in modern RL: SARSA, Q-learning, and the TD component of actor-critic methods all rest on the same core update rule introduced here.
Notation used in this lesson
$\delta_t$ TD error at time $t$: $\delta_t = R_{t+1} + \gamma V(S_{t+1}) - V(S_t)$
$\alpha$ Step size (learning rate) — how strongly each update moves the estimate
$G_t^{(n)}$ $n$-step return: $R_{t+1} + \gamma R_{t+2} + \cdots + \gamma^{n-1}R_{t+n} + \gamma^n V(S_{t+n})$
SARSA On-policy TD control; name from the quintuple $(S_t, A_t, R_{t+1}, S_{t+1}, A_{t+1})$
Q-learning Off-policy TD control; directly approximates $q_*$ using $\max_{a'} Q(S_{t+1}, a')$
The TD idea: bootstrap without a model
The Bellman expectation equation tells us that the value of a state equals the expected immediate reward plus the discounted value of the next state: $$v_\pi(S_t) = \mathbb{E}_\pi[R_{t+1} + \gamma v_\pi(S_{t+1}) \mid S_t]$$ DP turns this into an update by substituting the model and averaging over all possible next states. MC turns it into an update by running to the end of the episode and observing the actual return $G_t$.
TD takes a different route: observe just the next single step $(R_{t+1}, S_{t+1})$, and substitute the current estimate $V(S_{t+1})$ for the unknown $v_\pi(S_{t+1})$:
$$V(S_t) \leftarrow V(S_t) + \alpha\bigl[\underbrace{R_{t+1} + \gamma V(S_{t+1})}_{\text{TD target}} - V(S_t)\bigr]$$
This is the TD(0) update — the simplest TD method. The quantity $R_{t+1} + \gamma V(S_{t+1})$ is the TD target: a one-step estimate of what $V(S_t)$ should be. The difference between the target and current estimate is the TD error $\delta_t$. The update nudges $V(S_t)$ toward the target by step size $\alpha$.
Key observations:
  • No model. The update only uses observed $(R_{t+1}, S_{t+1})$ — actual samples from the environment, not $p(s', r \mid s, a)$.
  • No complete episode. The update happens after every single step. TD can learn in continuing tasks with no terminal state.
  • Bootstrapping. The target uses $V(S_{t+1})$, which is itself an estimate. We update one estimate using another — this is bootstrapping. It introduces some bias (if $V$ is wrong, the target is wrong), but dramatically reduces variance compared to waiting for a full episode return.
TD(0) prediction
TD(0) for policy evaluation (estimating $v_\pi$) works as follows:
  1. Initialize $V(s)$ arbitrarily (e.g. all zeros). Set $V(\text{terminal}) = 0$.
  2. Observe $S_t$. Take action $A_t \sim \pi(\cdot \mid S_t)$. Observe $R_{t+1}$ and $S_{t+1}$.
  3. Compute the TD error: $\delta_t = R_{t+1} + \gamma V(S_{t+1}) - V(S_t)$.
  4. Update: $V(S_t) \leftarrow V(S_t) + \alpha \delta_t$.
  5. Move to $S_{t+1}$ and repeat. At end of episode (if episodic), reset.
That is the entire algorithm. One step of experience → one update → move on. Compare to MC, which must buffer the entire episode before updating anything.
Proof sketch — TD(0) convergence (on-policy, tabular)

TD(0) is a form of stochastic approximation (Robbins-Monro 1951). The update $V(s) \leftarrow V(s) + \alpha_t \delta_t$ can be written as $V \leftarrow V + \alpha_t(b_t - A_t V)$ where $A_t$ and $b_t$ are random quantities. For TD(0) to converge, we require:

Robbins-Monro conditions on step sizes: $\sum_t \alpha_t = \infty$ and $\sum_t \alpha_t^2 < \infty$. The first condition ensures learning is not swamped by initialization; the second ensures noise averages out.

On-policy convergence argument. Under these conditions and assuming all states are visited infinitely often (guaranteed by the on-policy ε-soft behavior), one can show that the expected update $\mathbb{E}[\delta_t \mid V]$ points toward $v_\pi$. Formally, define $\bar{A} = \mathbb{E}[A_t]$ and $\bar{b} = \mathbb{E}[b_t]$; the fixed point of the mean ODE $\dot{V} = \bar{b} - \bar{A}V$ is $v_\pi$. The matrix $\bar{A}$ is positive definite under $\gamma < 1$ (a consequence of the discount — the "shifted" Bellman operator shrinks norms), so $v_\pi$ is a stable equilibrium. By the ODE method (Borkar & Meyn 2000), the stochastic iterates track this ODE and converge to $v_\pi$ w.p.1. $\blacksquare$

With a constant step size $\alpha$, TD(0) does not converge in the strict sense — it cycles around $v_\pi$ with variance $O(\alpha)$. Decaying step sizes are required for exact convergence.

TD vs MC on the same data: Consider an episode $S_0, R_1, S_1, R_2, S_2, \ldots$. MC waits for the episode to end, then updates $V(S_0)$ using $G_0 = R_1 + \gamma R_2 + \cdots$. TD updates $V(S_0)$ using $R_1 + \gamma V(S_1)$ immediately after observing $S_1$. If $V(S_1)$ is a reasonable estimate, this is a much lower-variance update — we're not propagating all the noise from the entire episode. The trade-off is that the TD target is biased if $V$ is wrong — but as learning proceeds and $V$ improves, this bias decreases.
The TD error $\delta_t$
The TD error deserves its own section because it appears everywhere in RL — not just in TD prediction, but in actor-critic methods, reward shaping, and neuroscience models of learning.
$$\delta_t = R_{t+1} + \gamma V(S_{t+1}) - V(S_t)$$
Intuitively: $\delta_t$ measures the prediction error at time $t$. Before seeing $R_{t+1}$, the agent's "prediction" of what it will observe from $S_t$ onward is $V(S_t)$. After seeing one step, the revised estimate is $R_{t+1} + \gamma V(S_{t+1})$. The TD error is the discrepancy between the prediction and the revised estimate.
A positive $\delta_t$ means the outcome was better than predicted ($V(S_t)$ was too low). A negative $\delta_t$ means it was worse. The TD update moves $V(S_t)$ in the direction that would have made the prediction correct.
There is a beautiful relationship between the TD error and the Monte Carlo return. If you run an episode and collect $\delta_t$ at each step, the sum telescopes: $$G_t - V(S_t) = \sum_{k=t}^{T-1} \gamma^{k-t} \delta_k$$ This identity means the MC update ($V(S_t) \leftarrow V(S_t) + \alpha[G_t - V(S_t)]$) is equivalent to summing all future TD errors along the trajectory. MC is TD, applied retroactively over the entire episode. This is the formal bridge between the two methods.
⟶ Worth verifying yourself: Expand the sum $\sum_{k=t}^{T-1} \gamma^{k-t}\delta_k$ using the definition $\delta_k = R_{k+1} + \gamma V(S_{k+1}) - V(S_k)$. Write out the first three terms and watch the cancellation. The telescoping property is not obvious until you write it out term by term — and once you see it, the MC/TD relationship is completely clear.
TD, MC, and DP: a unified view
These three methods are not competitors — they are points in a continuum. The key axes are how far ahead do you look and do you use a model or samples:
The bias–variance tradeoff runs left to right in this diagram. DP and TD bootstrap — they use estimates as targets, introducing bias but keeping variance low (a one-step TD error is much less noisy than a full episode return). MC uses the true return, which is unbiased but has high variance because it accumulates noise over the entire episode. In practice TD methods typically converge faster and more stably than MC, especially in long-episode environments.
There is also a subtler advantage of TD on Markov environments. Because TD exploits the Markov structure (propagating values step-by-step through the state space), it can make better use of limited data than MC. If you observe an episode where state $s$ transitions to $s'$, TD immediately updates $V(s)$ toward $V(s')$. MC waits for the episode to end and uses the full return — the information about the $s \to s'$ relationship is mixed into a single number and the Markov structure is not explicitly used.
SARSA: on-policy TD control
For control (finding an optimal policy), we switch from $V(s)$ to $Q(s,a)$ — just as we did with MC control. This avoids needing a model for policy improvement.
SARSA is the on-policy TD control algorithm. The update rule uses the quintuple $(S_t, A_t, R_{t+1}, S_{t+1}, A_{t+1})$ — hence the name:
$$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha\bigl[R_{t+1} + \gamma Q(S_{t+1}, A_{t+1}) - Q(S_t, A_t)\bigr]$$
The TD target uses the Q-value of the actual next action $A_{t+1}$ — the one selected by the current ε-greedy policy. Since the target depends on the policy being followed, this is on-policy: SARSA evaluates and improves the same policy it acts with.
The algorithm:
  1. Initialize $Q(s,a) = 0$ for all $s, a$.
  2. In state $S_t$: choose $A_t$ using ε-greedy w.r.t. $Q$.
  3. Take $A_t$, observe $R_{t+1}, S_{t+1}$.
  4. Choose $A_{t+1}$ using ε-greedy w.r.t. $Q$ from $S_{t+1}$.
  5. Update: $Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha[R_{t+1} + \gamma Q(S_{t+1}, A_{t+1}) - Q(S_t, A_t)]$.
  6. Set $S_t \leftarrow S_{t+1}$, $A_t \leftarrow A_{t+1}$; repeat.
SARSA converges to $q_*$ (the optimal action-value function) provided all state-action pairs continue to be visited (i.e. $\varepsilon > 0$ throughout or decaying appropriately) and the step sizes satisfy the usual stochastic approximation conditions.
Q-learning: off-policy TD control
Q-learning modifies the SARSA target in one critical way: instead of using the Q-value of the actual next action $A_{t+1}$, it uses the maximum Q-value over all next actions:
$$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha\bigl[R_{t+1} + \gamma \max_{a'} Q(S_{t+1}, a') - Q(S_t, A_t)\bigr]$$
This seemingly small change has a major consequence. The target $R_{t+1} + \gamma\max_{a'} Q(S_{t+1}, a')$ is the Bellman optimality operator applied to $Q$ — the same operator whose fixed point is $q_*$. So Q-learning directly approximates $q_*$, regardless of which policy is actually being followed to generate data. Q-learning is inherently off-policy: the behavior policy (ε-greedy) is used to collect data, but the target is computed as if a greedy policy were followed.
Q-learning as stochastic value iteration: Recall that DP value iteration updates $V(s) \leftarrow \max_a \sum_{s',r} p(s',r|s,a)[r + \gamma V(s')]$. Q-learning does the same thing, but uses a single sampled $(s', r)$ instead of averaging over the model. Q-learning is value iteration with sampled transitions — no model needed.
The algorithm is simpler than SARSA: you do not need to select $A_{t+1}$ before updating. After taking $A_t$, observing $(R_{t+1}, S_{t+1})$, update and move to $S_{t+1}$. Then select the next action. The update target does not depend on what you will do next.
Proof sketch — Q-learning convergence (Watkins & Dayan 1992)

Q-learning converges to $q_*$ w.p.1 provided: (1) all state-action pairs are visited infinitely often, (2) step sizes satisfy $\sum_t \alpha_t(s,a) = \infty$ and $\sum_t \alpha_t(s,a)^2 < \infty$ for all $(s,a)$, and (3) rewards are bounded.

Key idea. Define the sequence of iterates $Q_t$ and the approximation error $F_t(s,a) = Q_t(s,a) - q_*(s,a)$. The Q-learning update can be rewritten as: $$F_{t+1}(s,a) = (1-\alpha_t)F_t(s,a) + \alpha_t\bigl[\gamma\max_{a'}Q_t(s',a') - \gamma\max_{a'}q_*(s',a') + \text{noise}_t\bigr]$$ where $\text{noise}_t$ has zero conditional mean (since $\mathbb{E}[R|s,a]$ and $\mathbb{E}[\max_{a'}q_*(s',a')|s,a]$ are the deterministic parts). The $\max$ operator is a $\gamma$-contraction: $$|\gamma\max_{a'}Q_t(s',a') - \gamma\max_{a'}q_*(s',a')| \leq \gamma\|F_t\|_\infty$$ So $\|F_{t+1}\|_\infty \leq (1-\alpha_t)\|F_t\|_\infty + \alpha_t\gamma\|F_t\|_\infty + \alpha_t|\text{noise}_t|$. This is a contraction-with-noise iteration. By the stochastic approximation theory (Robbins-Monro), the noise term averages out under the step-size conditions, and the contraction factor $\gamma < 1$ drives $\|F_t\|_\infty \to 0$. $\blacksquare$

Cliff walking: SARSA vs Q-learning
The classic example showing where SARSA and Q-learning differ is the cliff walking problem. Consider a grid with a sheer cliff along the bottom row. Stepping onto the cliff sends the agent back to the start with a reward of $-100$. All other steps give $-1$. The goal is in the bottom-right corner.
Q-learning learns the truly optimal policy: walk along the edge of the cliff, since that is the shortest path to the goal with reward $-1$ per step. Q-learning can afford to learn this because its target is computed using $\max_{a'} Q$, which imagines a greedy agent that never accidentally falls off the cliff. The random exploration doesn't affect what the algorithm is trying to learn.
SARSA learns the optimal ε-soft policy: the best policy given that it must sometimes explore randomly. Since an ε-greedy SARSA agent occasionally takes random actions, walking next to the cliff means sometimes falling off (reward $-100$). SARSA experiences these falls during training and incorporates them into its value estimates. It learns that the safe path (going high above the cliff) is better in practice, even though it is longer.
Neither is universally better. Q-learning finds the truly optimal policy but risks poor performance during training (many $-100$ penalties). SARSA finds a safer policy that accounts for its own exploration behavior. This distinction becomes important in real-world systems where falling off a "cliff" during training is costly or dangerous.
n-step TD: unifying MC and TD
TD(0) looks one step ahead before bootstrapping. MC looks all the way to the end of the episode. The natural generalization is to look $n$ steps ahead, then bootstrap:
$$G_t^{(n)} = R_{t+1} + \gamma R_{t+2} + \cdots + \gamma^{n-1} R_{t+n} + \gamma^n V(S_{t+n})$$
The $n$-step TD update is: $$V(S_t) \leftarrow V(S_t) + \alpha\bigl[G_t^{(n)} - V(S_t)\bigr]$$
At $n=1$: $G_t^{(1)} = R_{t+1} + \gamma V(S_{t+1})$ — ordinary TD(0). At $n=\infty$: $G_t^{(\infty)} = G_t$ — the full Monte Carlo return. Any finite $n$ is a middle ground: accumulate $n$ real reward observations, then bootstrap with the current value estimate.
In practice, $n$ is a hyperparameter. Larger $n$ reduces bias (more real reward observations, less dependence on the current $V$ estimate) at the cost of higher variance and requiring $n$ steps before an update can be made. For many problems, $n = 3$–$10$ outperforms both $n=1$ and $n=\infty$.
TD(λ) and eligibility traces: Rather than picking a fixed $n$, TD(λ) uses a weighted average of all $n$-step returns, with the weight of the $n$-step return decaying geometrically as $\lambda^{n-1}(1-\lambda)$. This is the $\lambda$-return. $\lambda = 0$ gives TD(0); $\lambda = 1$ gives MC. Implemented using eligibility traces (a per-state memory that tracks how recently each state was visited), TD(λ) is an elegant unified framework. This lesson focuses on the $n$-step case; TD(λ) is covered in most textbooks as the natural extension.
Worked examples
Example 1 — TD(0) update, step by step
Current estimates: $V(s_3) = -1.5$, $V(s_7) = -3.0$. An agent in state $s_3$ takes an action, receives reward $R = -1$, and transitions to $s_7$. Use $\alpha = 0.1$, $\gamma = 0.9$. Compute the TD target, TD error, and updated $V(s_3)$.
TD target: $R + \gamma V(s_7) = -1 + 0.9 \times (-3.0) = -1 - 2.7 = -3.7$
TD error: $\delta = -3.7 - (-1.5) = -2.2$
Update: $V(s_3) \leftarrow -1.5 + 0.1 \times (-2.2) = -1.5 - 0.22 = \mathbf{-1.72}$
The negative TD error means the outcome was worse than predicted — $s_3$ is further from a good future than $V$ thought. The update moves $V(s_3)$ downward. Notice this update happens immediately after one step; MC would wait for the whole episode.
Example 2 — Q-learning update
State $s$: $Q(s,\text{N})=-2.0$, $Q(s,\text{E})=-1.5$, $Q(s,\text{S})=-3.0$, $Q(s,\text{W})=-2.5$. Agent takes East, receives $R=-1$, arrives at $s'$: $Q(s',\text{N})=Q(s',\text{E})=-0.5$, $Q(s',\text{S})=Q(s',\text{W})=-1.0$. Use $\alpha=0.2$, $\gamma=1$. Compute the Q-learning update to $Q(s,\text{E})$.
$\max_{a'} Q(s', a') = -0.5$ (North or East from $s'$).
TD target: $R + \gamma \max_{a'} Q(s', a') = -1 + 1.0 \times (-0.5) = -1.5$
TD error: $\delta = -1.5 - (-1.5) = 0$
No update! $Q(s, \text{E})$ is already consistent with the optimal value of $s'$. This is a fixed-point: if the rest of $Q$ is correct, no further update is needed here. In practice this happens near convergence — once $Q$ is close to $q_*$, TD errors become small everywhere.
Example 3 — n-step return calculation
An episode proceeds: $S_3 \xrightarrow{R_4=-1} S_4 \xrightarrow{R_5=-1} S_5 \xrightarrow{R_6=-1} S_6$. Current estimate $V(S_6) = -2.0$. Compute the 3-step return $G_3^{(3)}$ for state $S_3$ with $\gamma=1$, and compare it to the TD(0) target for the same state.
$$G_3^{(3)} = R_4 + \gamma R_5 + \gamma^2 R_6 + \gamma^3 V(S_6) = -1 + (-1) + (-1) + (-2) = -5$$
Compare to TD(0) for $S_3$: $G_3^{(1)} = R_4 + \gamma V(S_4)$. This requires $V(S_4)$ but only one real observation. The 3-step version uses three real reward observations and then bootstraps — lower variance, but needs to wait 3 steps before the update can be applied.

Food for thought
Reflection questions
  1. TD(0) bootstraps from $V(S_{t+1})$, which is itself an estimate. This makes the target a moving target — as $V$ changes, so do all past TD targets. Does this cause a problem for convergence? What assumption about the step size $\alpha$ is needed to guarantee convergence, and why does that assumption address the issue?
    The moving target is a genuine complication — TD is not gradient descent on a fixed loss, so the standard convergence proofs don't apply directly. The classic Robbins-Monro conditions handle it: $\sum_t \alpha_t = \infty$ (enough total learning to overcome any initialization) and $\sum_t \alpha_t^2 < \infty$ (updates shrink fast enough that noise averages out and the algorithm "settles"). As $\alpha_t \to 0$, the target is moving slower and slower relative to the update size, so the algorithm can converge to a fixed point even though the target was never truly fixed. With a constant $\alpha$, TD(0) converges in the mean for linear function approximation but oscillates around the true values — useful in practice but technically non-convergent in the classic sense.
  2. Q-learning converges to $q_*$ regardless of the behavior policy (as long as the behavior policy covers all actions). Does this mean you could use any data — even data from a completely different task — to learn $q_*$ for your task? What would go wrong (or right) in practice?
    No — "any behavior policy" means any policy acting in the same MDP. The reward signal in the data must come from the task you're learning. Data from a different task has different rewards, so Q-learning on it would converge to $q_*$ of that other task, not yours. What you can reuse: transitions $(s,a,s')$ collected under any policy in your MDP — you just replace the reward with the correct reward for your task. This is exactly experience replay in DQN: stored transitions are resampled with your own rewards regardless of when or how they were collected. Cross-task reuse requires transfer learning or meta-learning, not just Q-learning's off-policy property.
  3. In the cliff walking example, SARSA learns a safer policy while Q-learning learns the optimal policy. Which would you prefer in a real robot deployment? Does the answer change depending on whether the robot is in training or in production?
    During training: SARSA is typically safer for physical robots. Its on-policy update accounts for the ε-greedy exploration noise — so the learned policy stays away from catastrophic states even when the robot takes random actions. Q-learning's target policy ignores the exploration; the cliff edge is avoided only once exploration stops.

    In production: With ε=0 (purely greedy), both algorithms' policies converge to the same behaviour in most MDPs. Q-learning's optimal policy (shorter path along the cliff) is strictly better in terms of expected reward. If the deployment environment is safe and well-controlled, Q-learning's policy is preferable.

    The practical answer: Train with SARSA (safe exploration), then at deployment switch to the greedy policy derived from Q-learning's Q-values for optimal performance. This is a common pattern in real robotics.
Common pitfalls
Mistakes worth knowing about in advance
  • Confusing TD target and TD error. The TD target is $R_{t+1} + \gamma V(S_{t+1})$ — it is what you are trying to move $V(S_t)$ toward. The TD error $\delta_t$ is the difference between the target and $V(S_t)$. The update is $V(S_t) \leftarrow V(S_t) + \alpha \delta_t$, not $V(S_t) \leftarrow \text{target}$ (which would be a hard assignment, ignoring $\alpha$).
  • Using $V(S_{T+1})$ after the terminal step. When $S_{t+1}$ is the terminal state, $V(S_{t+1})$ must be treated as 0, not as whatever value $V(\text{terminal})$ happens to store. If you forget this, the terminal state's stored value will bleed into all previous states' estimates, producing wrong values near the end of episodes.
  • Thinking SARSA and Q-learning converge to the same policy. In most textbook examples they do (or nearly so). But as the cliff walking example shows, they can converge to genuinely different policies. SARSA converges to the best policy under ε-greedy exploration; Q-learning converges to $q_*$ regardless. With small $\varepsilon$, the policies are close — but not identical.
  • Treating Q-learning as on-policy. Q-learning is off-policy — the update target uses $\max_{a'} Q$, not the action that was actually taken. This means you can train Q-learning on any data collected from any policy, and re-use the same data multiple times (experience replay). This is a major practical advantage of Q-learning over SARSA.
  • Choosing $n$ in n-step TD without thinking about episode length. If your episodes are typically 10 steps long, $n=50$ step returns are mostly MC returns — you hit the terminal before bootstrapping. Choosing $n$ relative to the episode length matters; a good starting point is $n$ on the order of the average episode length divided by 5–10.
Connects to
  • RL · Dynamic Programming (Lesson 5) — Q-learning is stochastic value iteration: the same Bellman optimality operator, applied using sampled transitions rather than the model. SARSA is stochastic policy iteration (on-policy evaluation + greedy improvement). The DP algorithms are the model-based versions of what TD does here with samples.
  • RL · Function Approximation (Lesson 8) — TD(0) and Q-learning extend naturally to parameterized value functions via semi-gradient methods. The TD error $\delta_t$ becomes the signal for updating neural network weights. Deep Q-Networks (DQN) use Q-learning with a neural network for $Q(s,a;\theta)$ and experience replay to stabilize training.
  • Neuroscience — The TD error $\delta_t$ has a biological interpretation: it matches the firing pattern of dopaminergic neurons during reward learning in the mammalian brain. Animals show positive prediction errors (unexpected rewards → dopamine burst) and negative prediction errors (expected reward omitted → dopamine dip) — exactly what $\delta_t$ captures. TD learning was partly inspired by, and later confirmed to explain, this neural signal.
