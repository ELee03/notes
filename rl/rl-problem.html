1 · The RL Problem
Agent · Environment · Reward · Return · Discounting · Policy
Before any algorithms, before any equations, there is a situation: something is trying to achieve a goal by taking actions in a world it does not fully control. A chess engine choosing moves. A robot learning to walk. A heating system deciding when to turn on. In each case, the actor receives feedback on how well it is doing, and over time — through trial and error — it gets better. This is the problem reinforcement learning is built to solve.
What is reinforcement learning?
Reinforcement learning is a framework for learning from interaction. An agent takes actions in an environment and receives a reward signal in return. The agent's goal is to learn a strategy — a policy — that maximises the total reward it accumulates over time.
Three features distinguish RL from other machine learning paradigms:
  • No supervisor, only a reward signal. In supervised learning, a teacher provides the correct answer for each input. In RL there is no correct answer — only a scalar signal saying "this went well" or "this went badly," usually arriving long after the actions that caused it.
  • Feedback is delayed. In a game of chess, you only know who won at the end. The reward from winning cannot be attributed to any single move — it is the consequence of a long sequence of decisions. This is the credit assignment problem: which actions deserve credit for the final outcome?
  • The agent's actions change what it sees. Each action affects the state of the environment, which determines what actions become available next. An agent that learns to play video games changes the game state with every button press. This feedback loop means the data the agent trains on depends on the policy it is currently running — a circular dependency with no equivalent in supervised learning.
The agent-environment interface
The formal setup is a discrete-time loop. At each time step $t = 0, 1, 2, \ldots$: the agent observes the current state $S_t$, selects an action $A_t$, and the environment responds with a reward $R_{t+1}$ and a new state $S_{t+1}$. This cycle repeats indefinitely (or until a terminal condition).
At each step t: the agent observes $S_t$ and selects $A_t$; the environment responds with reward $R_{t+1}$ and next state $S_{t+1}$.
This loop generates a trajectory:
$$S_0,\; A_0,\; R_1,\; S_1,\; A_1,\; R_2,\; S_2,\; A_2,\; R_3,\; \ldots$$
The reward $R_{t+1}$ is indexed one step ahead of the action $A_t$ that caused it — the environment responds to $A_t$ by producing $R_{t+1}$. This subscript convention is consistent throughout Sutton & Barto and is worth internalising early.
What counts as the "environment"? The boundary between agent and environment is not necessarily the agent's physical body. It is the boundary of what the agent can directly control. A robot's motor controller might be considered part of the agent; the physics of the floor it walks on is the environment. The boundary is a design choice that depends on what problem you want the agent to learn.
Rewards and returns
The reward hypothesis is the central thesis of RL: all goals can be described as the maximisation of the expected cumulative reward signal. This is a bold claim — it says that whatever "success" means in a given problem, you can encode it as a scalar reward function, and an agent that maximises cumulative reward will achieve it.
The agent's goal is therefore to maximise the return $G_t$, defined as the sum of rewards from time $t$ onward. For a task with a natural endpoint (a game, an episode), the return is simply the sum of all future rewards:
$$G_t = R_{t+1} + R_{t+2} + R_{t+3} + \cdots + R_T$$
where $T$ is the terminal time step. For tasks that run indefinitely, this sum might diverge. The solution is discounting: multiply each future reward by a factor $\gamma^k$ where $\gamma \in [0, 1)$ is the discount rate and $k$ is how many steps in the future it arrives.
$$G_t = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots$$
Discounting has two practical effects. First, it ensures the sum converges (as long as rewards are bounded): a geometric series with ratio $\gamma < 1$ always converges. Second, it encodes a preference for reward sooner rather than later — a reward one step away is worth $\gamma$ times a reward zero steps away. At $\gamma = 0$ the agent is purely myopic (only cares about the next reward); as $\gamma \to 1$ the agent becomes more far-sighted.
The return satisfies a crucial recursive relationship that underpins almost every RL algorithm:
$$G_t = R_{t+1} + \gamma G_{t+1}$$
This says: the return from step $t$ is the immediate reward plus the discounted return from step $t+1$. This one-step decomposition is the seed of the Bellman equations — it lets you reason about long-horizon returns without summing infinitely many rewards directly.
Choosing $\gamma$. In practice, $\gamma$ is a hyperparameter you set. Values around 0.95–0.99 are common for tasks where long-term consequences matter (robot locomotion, game playing). For problems with short horizons or where speed of learning matters more than optimality, lower values (0.9 or even 0.5) can help. $\gamma = 1$ is valid for episodic tasks with guaranteed termination but requires careful handling.

Episodic vs continuing tasks
Episodic tasks have a natural terminal state that ends the interaction. A game of chess ends when someone wins or the board is drawn; a robot trying to stand up ends when it falls or reaches some target pose. Each complete interaction from start to terminal state is an episode. Episodes are independent — the agent starts fresh each time. The return formula uses a finite sum up to $T$.
Continuing tasks have no terminal state. A heating system runs indefinitely; a recommendation engine is always on. Here the finite-sum return diverges, making discounting essential. The discounted return formula with $\gamma < 1$ is the standard tool for continuing tasks.
S&B provides a unified notation: a terminal state can be treated as an absorbing state that transitions only to itself with zero reward, allowing the discounted-sum formula to handle both cases without modification.
Policies
A policy $\pi$ is the agent's strategy — a mapping from states to actions (or, more generally, from states to probability distributions over actions). A deterministic policy assigns exactly one action to each state: $\pi(s) = a$. A stochastic policy assigns a probability to each action:
$$\pi(a \mid s) = \Pr\{A_t = a \mid S_t = s\}$$
Stochastic policies are essential for exploration — if the agent always does the same thing in each state, it will never discover that other actions might be better. The goal of RL is to find the optimal policy $\pi^*$: the policy that maximises the expected return from every state.
What makes RL hard
Two fundamental tensions make RL genuinely difficult, and they recur throughout every algorithm in this cluster.
The credit assignment problem. When a reward arrives, it is not immediately clear which past action (or combination of actions) caused it. A reward at step 100 might trace back to a critical decision at step 12. Every RL algorithm is, at its core, a solution to this problem: a method for distributing credit (or blame) backward through time to the actions that mattered.
The exploration-exploitation dilemma. To maximise reward, the agent should take the action it currently believes is best (exploitation). But to learn which actions are best, it must sometimes try actions it hasn't tried before (exploration). These are in direct conflict — every action spent exploring is a missed opportunity to exploit what has already been learned. The next lesson (Multi-Armed Bandits) isolates this tension in the simplest possible setting before adding state and long-horizon credit assignment.
Food for thought
Reflection questions
  1. The reward hypothesis claims any goal can be encoded as a scalar reward signal. Is this actually true? Can you think of a goal that might be difficult or impossible to capture faithfully this way — without creating unintended agent behaviour?
    Answer
    The reward hypothesis is more of a working assumption than a proven theorem. Goals that are genuinely hard to encode: Multi-objective goals: "Be fast AND safe AND fuel-efficient" — any scalar combines them into one number, losing the original structure. The weighting is arbitrary and may not match the designer's true preferences in edge cases. Human preference: "Write a poem I like" — the reward signal (human ratings) is expensive, noisy, and not perfectly aligned with actual preference. RLHF systems face this: reward hacking (generating text that scores highly but is subtly wrong) is a known failure mode. Safety constraints: "Never do X, under any circumstances" — encoding as a large negative reward $-M$ for $X$ still allows the agent to violate $X$ if the positive reward elsewhere is large enough. Hard constraints are difficult to represent as scalar rewards. The hypothesis is useful as a framework, not a guarantee. Many real-world RL systems require careful reward engineering to avoid unintended emergent behaviors.
  2. If you added a constant $c$ to every reward $R_t$, would the optimal policy change? What about multiplying every reward by a positive constant? What if the constant were negative? Think through the effect on $G_t$ and on the ordering of policies.
    Answer
    Adding a constant $c$ to every reward: For an episodic task with $T$ steps, $G_t^\text{new} = G_t + c/(1-\gamma)$ (continuous) or $G_t + c \cdot \sum_{k=0}^{T-t-1}\gamma^k$ (episodic). This adds the same constant to every policy's return, preserving the ordering of policies — the optimal policy is unchanged. This is the basis of reward shaping techniques. Multiplying by a positive constant $\alpha > 0$: $G_t^\text{new} = \alpha G_t$. Every return scales uniformly; the ordering is preserved. Optimal policy unchanged (but learning rates and convergence of value-based methods are affected since value estimates change scale). Multiplying by a negative constant $\alpha < 0$: The sign of every return flips. A policy that was optimal (highest return) becomes the worst policy. The optimal policy reverses. If you accidentally negate your rewards, your agent learns to minimize reward instead of maximize it.
  3. Discounting with $\gamma < 1$ ensures the infinite sum $G_t$ converges (as long as rewards are bounded). But $\gamma = 1$ is also used in practice for episodic tasks. Why doesn't the return diverge in that case — and when might $\gamma = 1$ cause problems even for tasks that do terminate?
    Answer
    For episodic tasks (finite episodes), $G_t = \sum_{k=0}^{T-t-1} R_{t+k+1}$ is a finite sum regardless of $\gamma$ — there are only $T-t$ terms. So $\gamma = 1$ doesn't cause divergence when episodes always terminate in finite time. When $\gamma = 1$ causes problems even for terminating tasks: (1) Very long episodes: If an episode can last thousands of steps, the return $G_0$ can be very large in magnitude, causing numerical instability in value function estimates and gradient updates. (2) Non-termination risk: If the agent's policy can cause the episode never to terminate (e.g., the agent wanders in a cycle), $G_0 = \infty$ is undefined. With $\gamma < 1$, a cycling agent's return is still finite. (3) Variance: $\gamma = 1$ gives no preference for earlier rewards, so the variance of $G_t$ is the sum of variances of all future rewards — potentially very high for long episodes. This slows learning compared to discounted returns.
Common pitfalls
Mistakes worth knowing about in advance
  • The reward subscript convention. Reward $R_{t+1}$ is the reward that arrives after action $A_t$ is taken — it is indexed one step ahead. The environment responds to $(S_t, A_t)$ by producing $(S_{t+1}, R_{t+1})$. Confusing $R_t$ with $R_{t+1}$ scrambles all the bookkeeping and makes the recursion $G_t = R_{t+1} + \gamma G_{t+1}$ look wrong.
  • Return ≠ reward. $G_t$ is the cumulative sum of all future rewards, not just the next one. Maximising $R_{t+1}$ is myopic and often produces bad long-run behaviour (the agent takes a short-term gain that forecloses better future options). The actual goal is always to maximise the expected return.
  • Discounting ≠ ignoring the future. $\gamma = 0.99$ still gives substantial weight to rewards hundreds of steps out. Discounting encodes a relative preference for sooner rewards, not indifference to distant ones. Only $\gamma = 0$ produces a purely myopic agent.
  • Reward shaping can backfire. Adding intermediate "progress" rewards to guide the agent often causes unintended behaviour: the agent finds ways to collect the intermediate rewards without actually achieving the goal (reward hacking). A reward signal should measure what you want achieved, not how you expect it to be achieved.
