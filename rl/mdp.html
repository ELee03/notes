3 · Markov Decision Processes
State · Action · Transition Dynamics · Markov Property · Policy · Value Functions
Notation used in this lesson
$\mathcal{S},\, \mathcal{A}$ Sets of all states and actions (calligraphic notation for sets)
$p(s', r \mid s, a)$ Joint probability of transitioning to state $s'$ and receiving reward $r$, given current state $s$ and action $a$ — the complete dynamics function
$p(s' \mid s, a)$ Marginal probability of next state — equals $\sum_r p(s',r \mid s,a)$
$\Pr\{E\}$ Probability of event $E$
$\Pr\{E \mid F\}$ Conditional probability of $E$ given that $F$ has occurred
$\mathbb{E}_\pi[\cdot \mid \cdot]$ Expected value conditioned on an event, when the agent follows policy $\pi$
$\sum_r$ Sum over all possible reward values (marginalises out the reward)
$v_\pi(s),\; q_\pi(s,a)$ State-value and action-value functions under policy $\pi$ (previewed here, defined fully in Lesson 4)
The bandit problem captures exploration and exploitation but has no notion of state: each decision is independent, and today's choice has no effect on what will be available tomorrow. Most interesting problems are not like this. The move you make in chess changes the board. The acceleration you apply to a car changes its velocity. The treatment you administer to a patient changes their physiology. To reason about these problems, we need the full RL framework: the Markov Decision Process.
Adding state to the bandit
In the bandit problem, the agent always faces the same $k$ arms. There is no context — no information about what happened before or what might happen next. The agent simply picks an arm and gets a reward.
Now imagine a contextual version: the agent sees a state $S_t$ before choosing. In one state the best arm might be arm 3; in another, arm 5. The optimal action depends on the context. But there is still no sequential structure — the choice in state $S_t$ doesn't change what state comes next.
A full MDP adds the final ingredient: actions affect the next state. The choice of $A_t$ in state $S_t$ determines, stochastically, both the reward $R_{t+1}$ and the next state $S_{t+1}$. Now the agent must reason about consequences that unfold over many steps — how today's choice shapes tomorrow's options.
Formal definition
A finite MDP is a tuple $(\mathcal{S},\, \mathcal{A},\, p,\, \gamma)$ where:
  • $\mathcal{S}$ — finite set of states
  • $\mathcal{A}$ — finite set of actions (sometimes $\mathcal{A}(s)$ if the action set depends on state)
  • $p$ — the dynamics function, a probability distribution over next states and rewards given current state and action
  • $\gamma \in [0,1)$ — the discount factor
Everything about the environment's behaviour is encoded in the four-argument dynamics function:
$$p(s', r \mid s, a) = \Pr\{S_t = s',\, R_t = r \mid S_{t-1} = s,\, A_{t-1} = a\}$$
This single function defines the complete rules of the environment. From it, you can derive any quantity you need:
Derived quantity Formula Meaning
State-transition probabilities $p(s' \mid s,a) = \sum_r p(s',r \mid s,a)$ Prob of landing in $s'$ regardless of reward
Expected reward $r(s,a) = \sum_r r \sum_{s'} p(s',r \mid s,a)$ Mean reward from taking $a$ in $s$
Expected reward given transition $r(s,a,s') = \frac{\sum_r r\, p(s',r \mid s,a)}{p(s' \mid s,a)}$ Mean reward given the specific transition $s \to s'$
In practice, most environments are specified through $r(s,a)$ and $p(s'|s,a)$ separately, or defined by a simulation. But mathematically, $p(s',r|s,a)$ contains all of it.
The Markov property
The definition above conditions $p$ on only the most recent state and action: $S_{t-1}$ and $A_{t-1}$. It does not look further back into history. This is the Markov property:
$$\Pr\{S_t, R_t \mid S_{t-1}, A_{t-1}, S_{t-2}, A_{t-2}, \ldots\} = \Pr\{S_t, R_t \mid S_{t-1}, A_{t-1}\}$$
The future is independent of the past, given the present. The current state $S_{t-1}$ is a sufficient statistic for everything that has happened — it captures all information from history that is relevant to predicting the future.
This assumption is what makes MDPs tractable. If the future depended on the entire history, any algorithm would need to keep track of arbitrarily long sequences. The Markov property lets you discard history and act on the current state alone.
When the Markov property fails. Real environments are often not fully Markov — a single observation may not summarise all the relevant history. In a card game, the current hand does not tell you what cards have already been played. In robot locomotion, a position snapshot does not give velocity. These are partially observable MDPs (POMDPs). The standard workaround is to augment the state: track enough history (velocity + position, a count of played cards) until the augmented state is approximately Markov. Deep RL often uses recurrent networks (LSTMs) to learn what to include in the effective state.
Grid world: a concrete MDP
Abstract definitions become easier to reason about with a running example. S&B's standard example is a 4×4 grid world. Each cell is a state. At every non-terminal state, the agent can take one of four actions: up, down, left, right. The two shaded cells (top-left and bottom-right) are terminal states — the episode ends when the agent reaches either one. Every non-terminal step incurs a reward of $-1$; reaching a terminal state gives 0. If an action would move the agent off the grid, it stays in place.
The 4×4 grid world. States are cell positions; actions are the four compass directions. The goal is to reach a terminal state in as few steps as possible (minimising total $-1$ penalties = maximising total reward).
The optimal strategy — the optimal policy $\pi^*$ — is to take the shortest path from any state to the nearest terminal. States near the terminals are valuable (few steps to go); states far away are worth less. Quantifying this intuition precisely is exactly what value functions do, which we will define formally in Lesson 4.
Goals and policies
With state in the picture, a policy must specify what to do in every state, not just "which arm to pull next." A stochastic policy is a conditional distribution over actions given states:
$$\pi(a \mid s) = \Pr\{A_t = a \mid S_t = s\}$$
The agent's goal is to find the policy that maximises the expected discounted return from every state it might start in. We say a policy $\pi'$ is at least as good as $\pi$ if its expected return is greater than or equal to $\pi$'s from every state:
$$\pi' \geq \pi \iff v_{\pi'}(s) \geq v_\pi(s) \quad \text{for all } s \in \mathcal{S}$$
A key theorem: for any finite MDP, at least one optimal policy $\pi^*$ exists that is simultaneously at least as good as all other policies from every state. This is not obvious — it is possible that policy $A$ beats policy $B$ in some states but not others, making it seem like no single "best" policy could exist. The theorem says the partial ordering can always be resolved.
Value functions — a preview
To find $\pi^*$, we need a way to evaluate how good a state (or state-action pair) is under a given policy. The state-value function $v_\pi(s)$ answers: "if I follow policy $\pi$ starting from state $s$, what return should I expect?"
$$v_\pi(s) = \mathbb{E}_\pi\bigl[G_t \;\big|\; S_t = s\bigr] = \mathbb{E}_\pi\!\left[\sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \;\bigg|\; S_t = s\right]$$
The action-value function $q_\pi(s,a)$ is the expected return from taking action $a$ in state $s$, then following $\pi$:
$$q_\pi(s, a) = \mathbb{E}_\pi\bigl[G_t \;\big|\; S_t = s,\; A_t = a\bigr]$$
These two functions are the central objects of RL. If we could compute them exactly, finding $\pi^*$ would be trivial: in each state, take the action with the highest $q_\pi(s,a)$. The challenge is that we usually cannot compute them exactly — we must estimate them from experience. That is what Lessons 4–7 are about.
Notice the recursive structure. Substituting $G_t = R_{t+1} + \gamma G_{t+1}$ into the definition of $v_\pi(s)$ yields: $$v_\pi(s) = \mathbb{E}_\pi\bigl[R_{t+1} + \gamma v_\pi(S_{t+1}) \;\big|\; S_t = s\bigr]$$ This is the Bellman expectation equation: the value of a state equals the expected immediate reward plus the discounted value of the next state. It is a self-consistency condition — $v_\pi$ is the unique function that satisfies it. Lesson 4 derives this equation fully and shows how to use it.

Food for thought
Reflection questions
  1. In blackjack, the state "my current hand" does not capture which cards have already been played. Does this mean blackjack is not an MDP — or is it? How would you augment the state so that the game is Markov? Is there always a way to do this?
    The game with a single deck is technically not Markov under the "current hand" state: knowing which cards remain in the deck affects the transition probabilities, and that information isn't in the state. However, Sutton & Barto treat blackjack as an MDP with the assumption of infinite decks (cards are drawn with replacement), making the transition probabilities depend only on the current hand — restoring the Markov property. To make blackjack truly Markov with a finite deck, you would augment the state with the full card count (a "count" of which cards have been dealt). This is exponentially large but mathematically valid. Is there always a way to do this? In principle, yes: for any system with finite history, augmenting the state with the full history makes it Markov (the "history state"). But this may be exponentially or infinitely large. The practical question is whether a compact Markov representation exists — and often the answer is to use a POMDP (partially observable MDP) with a belief state instead.
  2. If you doubled all rewards in an MDP, would the optimal policy $\pi^*$ change? Would the value functions $v_{\pi^*}(s)$ change? Are these two questions the same or different?
    Would $\pi^*$ change? No. Doubling all rewards multiplies every policy's value function by 2: $v_\pi^\text{new}(s) = 2\, v_\pi(s)$. Since all values scale uniformly, the ordering of policies is preserved — the policy with the highest value at each state is still the same. Would $v_{\pi^*}(s)$ change? Yes — it doubles: $v_{\pi^*}^\text{new}(s) = 2\, v_{\pi^*}(s)$. Same or different questions? Different. The policy is defined by the relative ordering of state values; the value function itself has an absolute magnitude. Transformations that preserve order (positive scaling, shifting) leave the optimal policy unchanged but can change the value function's magnitude. This distinction matters in practice: reward scaling affects the learning rate of value-based algorithms, even when the optimal policy is unchanged.
  3. The Bellman equation says $v_\pi(s) = \mathbb{E}_\pi[R + \gamma v_\pi(S')]$, which defines $v_\pi$ in terms of itself — it looks circular. Why does a unique solution exist? What mathematical condition on $\gamma$ makes this work, and what goes wrong if that condition fails?
    The Bellman equation is a linear system: in matrix form, $v = r + \gamma P v$, where $P$ is the transition probability matrix and $r$ is the expected reward vector. This rearranges to $(I - \gamma P)v = r$. The solution is unique iff $(I - \gamma P)$ is invertible, which requires that $\gamma \rho(P) < 1$, where $\rho(P)$ is the spectral radius of $P$. Since $P$ is a stochastic matrix, $\rho(P) = 1$. So the condition becomes $\gamma < 1$ — the discount factor must be strictly less than 1 for a unique finite solution to exist. What goes wrong at $\gamma = 1$? $(I - P)$ is singular (it always has eigenvalue 0, since $P$ has eigenvalue 1 for the stationary distribution). The system either has no solution or infinitely many solutions. In practice, for episodic tasks, you "escape" this because the terminal state absorbs probability and the system is effectively finite. For continuing tasks, $\gamma = 1$ is undefined unless rewards sum to zero under the stationary distribution (the "ergodic" case, leading to the average-reward formulation).
Common pitfalls
Mistakes worth knowing about in advance
  • Confusing $p(s' \mid s,a)$ with $p(s',r \mid s,a)$. The first is the marginal transition probability (summed over all reward values); the second is the joint distribution over next states and rewards together. They are related by $p(s' \mid s,a) = \sum_r p(s',r \mid s,a)$. Most algorithms work with the marginal for transition logic and the expected reward $r(s,a)$ for value estimation — computing one from the other requires this marginalisation.
  • Thinking the Markov property means the agent must ignore history. The Markov property is a statement about the environment's dynamics, not the agent's memory. The dynamics function $p(s',r \mid s,a)$ depends only on the current state — but the agent is free to maintain a complete history of observations and use it to inform its policy. The property merely says that the environment does not need that history to determine what happens next.
  • Confusing $r(s,a)$ and $r(s,a,s')$. The expected reward $r(s,a) = \sum_{s'} p(s' \mid s,a)\, r(s,a,s')$ is averaged over all possible next states. The reward $r(s,a,s')$ is specific to a particular transition. These differ whenever the reward depends on where you end up (e.g., reaching a terminal state may give a different reward than a non-terminal step from the same action). Using one in place of the other produces incorrect value estimates.
  • Forgetting that terminal states are absorbing. In episodic MDPs, the episode ends the moment the agent reaches a terminal state — no further actions are taken and no more rewards are received. Value functions are defined as $v_\pi(\text{terminal}) = 0$ by convention, not as undefined. The return from a terminal state is zero, not some large negative or positive number.
Connects to
  • Probability Theory — the dynamics function $p(s',r \mid s,a)$ is a joint conditional distribution; computing $r(s,a)$ and $p(s' \mid s,a)$ requires marginalisation. Lesson 4's Bellman equations are expectations over this distribution, and understanding them requires fluency with conditional expectations.
  • Control Theory — the MDP is a stochastic generalisation of the state-space model $x_{t+1} = f(x_t, u_t) + \text{noise}$; $\mathcal{S}$ and $\mathcal{A}$ correspond to the state and input spaces, and $p$ corresponds to the plant dynamics. Optimal control and RL solve the same underlying problem with different mathematical tools.
  • Linear Algebra — for finite state-action spaces, the transition dynamics form a tensor that can be written as a family of matrices. Lesson 5's Dynamic Programming exploits this matrix structure to compute value functions exactly.
