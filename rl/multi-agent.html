12 · Multi-Agent Reinforcement Learning
MARL Formulation · Nash Equilibria · Independent Q-Learning · Cooperative vs Competitive · CTDE
Every lesson so far has assumed a single agent making decisions in an environment that doesn't react strategically to the agent's policy. This fails the moment there's more than one decision-maker — a second robot sharing a workspace, an opponent in a game, other drivers on the road, competing bidders in an auction. In these settings, the "environment" includes other agents whose policies change over time, making the world non-stationary from each agent's perspective. Multi-Agent Reinforcement Learning (MARL) extends RL to handle $n > 1$ learning agents, each with its own objectives, observations, and actions.
Notation recap
$n$Number of agents
$\mathbf{a} = (a^1, \ldots, a^n)$Joint action of all agents
$r^i(s,\mathbf{a})$Reward for agent $i$ (may differ across agents)
$\pi^i(a^i|o^i)$Policy of agent $i$ based on its local observation $o^i$
$\boldsymbol{\pi} = (\pi^1, \ldots, \pi^n)$Joint policy profile
MARL problem formulation
A MARL problem is typically formalised as a Markov game (also called a stochastic game): $$\langle \mathcal{S},\, \{\mathcal{A}^i\}_{i=1}^n,\, p(s'|s,\mathbf{a}),\, \{r^i(s,\mathbf{a})\}_{i=1}^n,\, \gamma \rangle$$ where the key extension over a single-agent MDP is that the transition $p$ and rewards $r^i$ depend on the joint action $\mathbf{a} \in \mathcal{A}^1 \times \cdots \times \mathcal{A}^n$. Each agent $i$ seeks to maximise its own discounted return $J^i = \mathbb{E}[\sum_t \gamma^t r^i_t]$.
The key challenge introduced by multiple agents is non-stationarity: from agent $i$'s perspective, the environment is non-stationary because the other agents' policies $\pi^{-i}$ are changing during training. This breaks the Markov property from any single agent's viewpoint — the same state $s$ and action $a^i$ can lead to different outcomes depending on what the other agents currently do.
Cooperative, competitive, and mixed settings
MARL problems are categorised by the reward structure:
Setting Reward structure Examples
Fully cooperativeAll agents share $r^i = r$ (same reward)Robot swarms, sensor networks, multi-robot manipulation
Zero-sum$\sum_i r^i = 0$ — one agent's gain is another's lossChess, Go, poker, StarCraft
Mixed (general-sum)Rewards are arbitrary — partially alignedTraffic, trading, social dilemmas
Nash equilibria in MARL
In single-agent RL, the target is the optimal policy $\pi^*$ that maximises $J$. In multi-agent settings, the analogous concept is the Nash equilibrium: a joint policy profile $\boldsymbol{\pi}^* = (\pi^{1*}, \ldots, \pi^{n*})$ such that no agent can improve its return by unilaterally changing its policy: $$J^i(\pi^{i*}, \pi^{-i*}) \geq J^i(\pi^i, \pi^{-i*}) \quad \forall\,\pi^i,\;\forall\, i$$
Why Nash equilibrium? If every agent is at a Nash equilibrium, no agent has an incentive to deviate. It's a fixed point of the mutual best-response process. Nash (1950) proved that every finite game has at least one Nash equilibrium (possibly in mixed strategies).
MARL as Nash finding. Many MARL algorithms can be interpreted as attempts to converge to a Nash equilibrium:
  • In zero-sum two-player games: Nash equilibria are minimax solutions; AlphaGo/AlphaZero converge to Nash for perfect-information games via self-play.
  • In cooperative games: Nash equilibrium becomes the team-optimal joint policy.
  • In general-sum games: Nash equilibria may be inefficient (Prisoner's Dilemma — the Nash equilibrium is mutual defection, which is worse than mutual cooperation for both agents).
Independent Q-Learning and its failure modes
The simplest MARL approach is Independent Q-Learning (IQL): each agent $i$ runs its own Q-learning algorithm, treating all other agents as part of the environment. From agent $i$'s perspective, this is just a standard Q-learning problem.
IQL algorithm: For each agent $i$ independently: $$Q^i(s, a^i) \leftarrow Q^i(s,a^i) + \alpha\left[r^i + \gamma\max_{a'^i}Q^i(s', a'^i) - Q^i(s,a^i)\right]$$ No communication between agents. No joint Q-function. Each agent has its own $Q^i$.
IQL is simple and scalable — the action space each agent must reason about is only $|\mathcal{A}^i|$, not the joint $|\mathcal{A}^1| \times \cdots \times |\mathcal{A}^n|$. In practice it often works well in cooperative settings with small agent counts. But it has fundamental theoretical problems:
  • Non-stationarity. As each agent updates its policy, the "environment" (from any single agent's view) is changing. The Markov property is violated. Q-learning's convergence proof requires a stationary environment — IQL provides no convergence guarantees in general.
  • Relative overgeneralisation. In cooperative settings, agent $i$ may learn a suboptimal action because it appears good when averaged over the stochastic actions of others, even though the joint optimal policy requires a different action. The agents get "stuck" at a local equilibrium that isn't the team optimum.
  • Credit assignment. With a shared reward, agent $i$ cannot tell whether its own action or another agent's action caused a good or bad outcome. This is the multi-agent credit assignment problem.
Centralised Training, Decentralised Execution (CTDE)
The dominant paradigm for cooperative MARL is Centralised Training with Decentralised Execution (CTDE). The key insight: during training, we can use global information (full state $s$, all agents' actions, all rewards) to learn better — but at deployment, each agent must act based only on its local observation $o^i$ (because agents may not be able to communicate in real time).
CTDE decouples the training phase (where we have full information) from the execution phase (where we must be decentralised):
  • Training: A centralised critic $Q(s, \mathbf{a})$ (or value function $V(s)$) is trained with access to the full global state and all agents' actions. This critic provides gradient signals that are richer than any individual agent's local perspective.
  • Execution: Each agent's policy $\pi^i(a^i|o^i)$ is decentralised — it only depends on agent $i$'s local observation. The centralised critic is discarded after training.
MADDPG (Multi-Agent Deep Deterministic Policy Gradient) applies this idea to continuous action spaces: each agent has a centralised critic $Q^i(s, a^1, \ldots, a^n)$ that takes all agents' actions as input, but each actor $\pi^i(o^i)$ is trained to produce decentralised actions.
QMIX addresses cooperative MARL with shared reward by learning a mixing network: each agent has its own Q-function $Q^i(o^i, a^i)$, but the team Q-function $Q_\text{tot}(s, \mathbf{a})$ is a monotone combination of individual Q-functions: $$Q_\text{tot}(s, \mathbf{a}) = f_\text{mix}(Q^1(o^1,a^1), \ldots, Q^n(o^n,a^n); s)$$ The monotonicity constraint $\frac{\partial Q_\text{tot}}{\partial Q^i} \geq 0$ ensures that if agent $i$'s individual greedy action maximises $Q^i$, it also contributes positively to $Q_\text{tot}$. This allows decentralised execution (each agent greedily maximises its own $Q^i$) while the centralised $Q_\text{tot}$ remains consistent — a remarkable property called Individual-Global-Max (IGM).
Self-play and competitive settings
In zero-sum two-player games, the most powerful training technique is self-play: the agent plays against a copy of itself. Self-play has two major advantages:
  • Automatic curriculum. As the agent improves, so does its opponent (since it's also the agent). The difficulty of the opponent scales automatically with the agent's ability — always providing a meaningful challenge.
  • Convergence to Nash (in two-player zero-sum games). For perfect-information two-player zero-sum games, Nash equilibrium = minimax solution, which is unique. Self-play with sufficient exploration provably converges to Nash in this case. This is why AlphaZero works: chess and Go are perfect-information zero-sum games.
In general-sum or many-player games, self-play can cycle — the agent oscillates between policies without converging to Nash. More sophisticated methods (fictitious play, policy-space response oracles) are needed.
Worked examples
Example 1 — Prisoner's Dilemma: Nash vs cooperative outcome
Two agents, each choosing Cooperate (C) or Defect (D). Payoff matrix (agent 1, agent 2):
$(C,C) \to (3,3)$; $(C,D) \to (0,4)$; $(D,C) \to (4,0)$; $(D,D) \to (1,1)$.
Find the Nash equilibrium and the Pareto-optimal outcome. Why is independent Q-learning likely to converge to the Nash equilibrium rather than Pareto-optimal?
Nash equilibrium. For agent 1: if agent 2 plays C, agent 1 gets 3 from C or 4 from D → D is better. If agent 2 plays D, agent 1 gets 0 from C or 1 from D → D is better. D strictly dominates C for agent 1 (and by symmetry, for agent 2). Nash equilibrium: $(D,D)$ with payoff $(1,1)$. Pareto-optimal outcome: $(C,C)$ with payoff $(3,3)$ — both agents do better than Nash. Why IQL converges to $(D,D)$: Each agent's Q-function will learn $Q(D) > Q(C)$ for any fixed opponent policy (D always gives more reward). As agents update independently, they both shift toward D. The cooperative $(C,C)$ equilibrium is unstable under independent Q-learning — any agent deviating to D gets a short-term gain, breaking the cooperation. Only coordination mechanisms (communication, commitment, joint training) can sustain $(C,C)$.
Example 2 — CTDE advantage over IQL
Two cooperative agents in a 2D grid. Agent 1 covers the left half; Agent 2 covers the right half. A target appears in one half; the agent covering that half must touch it (reward $+1$); the other agent does nothing (reward $0$). Agents observe only their own half. Why does IQL fail here, and how does CTDE fix it?
IQL failure: Agent 1 cannot observe the right half. From agent 1's perspective, sometimes it gets reward $+1$ (target in its half) and sometimes $0$ (target in agent 2's half), regardless of its own actions. Agent 1's Q-function is noisy — it cannot distinguish "I missed the target" from "the target was in agent 2's half." Both look identical from agent 1's local observation. CTDE fix: During training, the centralised critic $Q(s, a^1, a^2)$ has access to the full state (both halves), so it can correctly attribute reward. The critic signals agent 1 to move toward targets in its own half and agent 2 to move toward targets in its own half — the credit assignment problem is solved centrally. At execution, each agent still acts on its own local observation, which works because the policies learned during CTDE training are conditioned correctly.
Food for thought
Reflection questions
  1. QMIX's IGM property requires that the best joint action can be found by each agent independently maximising its own $Q^i$. This restricts the class of Q-functions that QMIX can represent. What class of cooperative problems is QMIX guaranteed to solve optimally, and what class of problems might it fail on?
    QMIX solves problems where the optimal joint action is factorizable — where the global best action can be decomposed into independent per-agent best actions. It fails when there are non-monotone interactions: problems where the optimal action for agent $i$ depends on what agent $j$ does in a non-monotone way (e.g., if agent 1 and agent 2 must coordinate on opposite actions — one goes left, the other goes right — the global Q-function is not monotone in either individual Q). QTRAN and QPLEX extend beyond QMIX's representational class but are harder to scale.
  2. In zero-sum games, self-play converges to Nash. In general-sum games, it can cycle. Intuitively, why does the zero-sum structure prevent cycling?
    In a two-player zero-sum game, Nash equilibrium = minimax solution, which is unique (by von Neumann's minimax theorem). Any strategy that beats the Nash equilibrium policy is itself beaten by another strategy — there are no cycles in the best-response graph because all paths lead to the unique Nash fixed point. In general-sum games, Nash equilibria can be multiple and unstable; best-response dynamics can cycle between equilibria without converging.
Common pitfalls
Mistakes worth knowing about in advance
  • Assuming IQL will converge in multi-agent settings. IQL has no convergence guarantees. It works empirically in many cooperative settings but can fail badly in competitive or mixed settings. Use it as a baseline, not as a reliable algorithm.
  • Conflating Nash equilibrium with optimal play. In general-sum games, Nash equilibria can be highly suboptimal for all agents (Prisoner's Dilemma). Nash equilibrium means "no unilateral deviation improves things" — not "this is the best outcome possible."
  • Ignoring the joint action space explosion. With $n$ agents each having $k$ actions, the joint action space has $k^n$ elements. A centralised Q-function over joint actions becomes intractable for $n > 4$–$5$ agents. CTDE methods like QMIX avoid this by factorising the joint Q-function.
  • Using self-play in general-sum games without precautions. Self-play can converge to a local Nash in zero-sum two-player games. In many-player general-sum games it typically doesn't converge — use league training (a diverse pool of opponents) or population-based methods instead.
