4 · Value Functions & Bellman Equations
Vπ · Qπ · Bellman Expectation · Optimal Value · Bellman Optimality · Backup Diagrams
Every RL algorithm — whether it solves equations directly, samples trajectories, or bootstraps one-step estimates — is ultimately trying to answer one question: how good is it to be in this state? Value functions make that question precise. This lesson defines them, derives the self-consistency equation they must satisfy, and explains why solving that equation directly is intractable for most real problems — which motivates everything in Lessons 5 through 9.
Notation used in this lesson

$q_\pi(s,a)$Action-value function under policy $\pi$ — expected return from taking $a$ in state $s$, then following $\pi$
$v_*(s)$Optimal state-value function — the best achievable expected return from $s$
$q_*(s,a)$Optimal action-value function
$\mathbb{E}_\pi[\cdot \mid S_t = s]$Expected value of a random quantity, given starting state $s$, when following $\pi$
$\pi(a \mid s)$Probability of choosing action $a$ in state $s$ under policy $\pi$
$p(s',r \mid s,a)$Dynamics function — joint probability of next state $s'$ and reward $r$ given $(s,a)$
$\sum_a, \sum_{s'}, \sum_r$Sums over all actions, next states, and possible reward values respectively
Why value functions?
An agent following a policy $\pi$ in a state $s$ will produce a return $G_t$ — but $G_t$ is a random variable, not a fixed number. The actual return depends on the stochastic transitions of the environment and the stochastic choices of $\pi$ itself. Different runs from the same state can yield wildly different totals.
The value of a state under policy $\pi$ is the expected return: the long-run average over all those possible trajectories. This is a fixed, stable number that characterises how good state $s$ is — not for one lucky run, but in a statistical sense, across all possible futures. A high value means that policy $\pi$ reliably generates good long-run reward from $s$; a low value means the opposite.
Value functions are the foundation of policy optimisation. Once you can measure how good each state (or state-action pair) is under your current policy, you can improve the policy by shifting probability toward better-valued actions. This is the core loop of almost every RL algorithm.
The state-value function $v_\pi$
The state-value function under policy $\pi$ is defined as:
$$v_\pi(s) \;=\; \mathbb{E}_\pi\!\left[G_t \;\Big|\; S_t = s\right] \;=\; \mathbb{E}_\pi\!\left[\sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \;\Bigg|\; S_t = s\right]$$
Read this as: "starting in state $s$ at time $t$ and following $\pi$ from that point, what is the expected total discounted reward?" For episodic tasks, the sum is finite; the discounted form applies to both.
By convention, the value of terminal states is always 0 — there is no future reward to collect once an episode ends. This boundary condition is what lets the Bellman equations produce a consistent solution.
Intuition for a grid world. In the 4×4 grid world from Lesson 3, the value $v_\pi(s)$ of a cell under the random policy ($\pi = \frac{1}{4}$ in each direction) is negative in every non-terminal state — and the magnitude grows with the distance to the nearest terminal. A corner far from both terminals has a very negative value; a cell adjacent to a terminal is nearly zero. The value function is a map of "how far am I from getting out?"
The action-value function $q_\pi$
The action-value function (also called the Q-function) refines the question: not just "how good is state $s$?" but "how good is it to take action $a$ specifically in state $s$, and then follow $\pi$?"
$$q_\pi(s, a) \;=\; \mathbb{E}_\pi\!\left[G_t \;\Big|\; S_t = s,\; A_t = a\right]$$
The distinction matters. Under a random policy, some actions from state $s$ may be much better than others. $v_\pi(s)$ averages over all of them according to $\pi$; $q_\pi(s, a)$ evaluates a single specific action. This is why $q_\pi$ is more useful for control: once you know $q_\pi(s, a)$ for all actions, you can improve the policy by favouring actions with higher $q$ values — without needing a model of the environment.
Relating $v_\pi$ and $q_\pi$
The two functions are two views of the same thing, and each can be expressed in terms of the other. Starting from definitions:
$$v_\pi(s) = \sum_a \pi(a \mid s)\; q_\pi(s, a)$$
The state-value is the average action-value, weighted by the probability that $\pi$ chooses each action. Conversely, the action-value is the expected immediate reward plus the discounted state-value of the next state:
$$q_\pi(s, a) = \sum_{s',\, r} p(s', r \mid s, a)\;\bigl[r + \gamma\, v_\pi(s')\bigr]$$
Read the second equation as: "take action $a$ in $s$; the environment transitions to some $(s', r)$ pair; your value is the immediate reward $r$ plus the discounted value of wherever you land." Substituting the second equation into the first produces the Bellman equation.
The Bellman expectation equation
The two identities above combine into a single self-consistency condition that $v_\pi$ must satisfy. Substituting the $q_\pi$ expression into the $v_\pi$ expression:
$$\boxed{v_\pi(s) = \sum_a \pi(a \mid s) \sum_{s',\, r} p(s', r \mid s, a) \;\bigl[r + \gamma\, v_\pi(s')\bigr]}$$
This is the Bellman expectation equation for $v_\pi$. It says that the value of state $s$ equals the expected immediate reward plus the discounted value of the successor state — where the expectation is over both the action chosen by $\pi$ and the transition randomness of the environment. It is a local consistency condition: knowing the values of neighbouring states $s'$, you can compute $v_\pi(s)$.
⟶ Worth deriving this yourself. The Bellman equation falls out of a two-step substitution that uses the recursive structure of $G_t$ and the tower property of conditional expectation. Start from $v_\pi(s) = \mathbb{E}_\pi[R_{t+1} + \gamma G_{t+1} \mid S_t = s]$, expand the expectation over the possible actions and transitions, and use the Markov property to replace $\mathbb{E}_\pi[G_{t+1} \mid S_{t+1} = s']$ with $v_\pi(s')$. Working through this once — with a concrete two-state MDP — gives much more intuition than reading the result.
An analogous equation holds for $q_\pi$:
$$q_\pi(s, a) = \sum_{s',\, r} p(s', r \mid s, a)\;\Bigl[r + \gamma \sum_{a'} \pi(a' \mid s')\; q_\pi(s', a')\Bigr]$$
Together these form a system of linear equations — one equation per state (or state-action pair), all coupled through the $v_\pi(s')$ terms on the right. For a finite MDP with $|\mathcal{S}|$ states, this is a system of $|\mathcal{S}|$ linear equations in $|\mathcal{S}|$ unknowns, solvable in principle by Gaussian elimination.
Backup diagrams
A backup diagram is a tree diagram that shows which values are "backed up" into the equation for $v_\pi(s)$ or $q_\pi(s,a)$. The top of the tree is the quantity being computed; the leaves are the values being used to compute it.
Left: backup diagram for $v_\pi(s)$ — the policy selects an action (blue edges), the environment selects a next state and reward (green edges). Right: backup for $q_\pi(s,a)$ — starts with an action choice, then fans out to next states, then the policy fans out over next actions.
The diagram makes the Bellman equation visual: to back up a value into $v_\pi(s)$, you need one level of policy averaging and one level of environment averaging. The leaves of the one-step tree are the $v_\pi(s')$ values — which are themselves backed up from their own one-step trees. The Bellman equation is a recursive definition; the consistency condition it expresses is that these definitions must all agree simultaneously across all states.
Optimal value functions
Among all possible policies for a finite MDP, there exists at least one optimal policy $\pi^*$ that achieves the highest possible expected return from every state simultaneously. The value functions under $\pi^*$ are the optimal value functions:
$$v_*(s) = \max_\pi\; v_\pi(s) \qquad \text{for all } s \in \mathcal{S}$$
$$q_*(s, a) = \max_\pi\; q_\pi(s, a) \qquad \text{for all } s \in \mathcal{S},\; a \in \mathcal{A}$$
The existence of a single policy that simultaneously maximises value in every state is not obvious — it is a theorem. The key is the partial ordering of policies: if $v_{\pi'}(s) \geq v_\pi(s)$ for all $s$, then $\pi' \geq \pi$. It can be shown that the partial order is always resolvable, so a globally optimal $\pi^*$ always exists.
The relationship between $v_*$ and $q_*$ is the same as before, but the policy averaging collapses into a max:
$$v_*(s) = \max_a\; q_*(s, a)$$
The optimal state-value is just the highest action-value available from that state. An optimal deterministic policy is to act greedily with respect to $q_*$: in every state, take the action with the highest $q_*$ value.
Bellman optimality equations
Substituting the greedy relationship into the backup for $q_*$ gives the Bellman optimality equations:
$$v_*(s) = \max_a \sum_{s',\, r} p(s', r \mid s, a)\;\bigl[r + \gamma\, v_*(s')\bigr]$$
$$q_*(s, a) = \sum_{s',\, r} p(s', r \mid s, a)\;\Bigl[r + \gamma\,\max_{a'}\; q_*(s', a')\Bigr]$$
These look similar to the Bellman expectation equations, with one crucial difference: the policy averaging $\sum_a \pi(a \mid s)$ has been replaced by $\max_a$. This makes the system nonlinear — no longer a simple set of linear equations. The max operator couples the equations in a way that cannot be resolved by matrix algebra alone.
Proof — Bellman optimality equations have a unique solution (contraction mapping)

Define the Bellman optimality operator $\mathcal{T}: \mathbb{R}^{|\mathcal{S}|} \to \mathbb{R}^{|\mathcal{S}|}$ by: $$(\mathcal{T}v)(s) = \max_a \sum_{s',r} p(s',r \mid s,a)\bigl[r + \gamma v(s')\bigr]$$ We claim $\mathcal{T}$ is a $\gamma$-contraction under the $\sup$-norm $\|v\|_\infty = \max_s |v(s)|$.

Contraction. For any two value functions $u, v$: $$|(\mathcal{T}u)(s) - (\mathcal{T}v)(s)| = \left|\max_a \mathbb{E}[\gamma u(s')] - \max_a \mathbb{E}[\gamma v(s')]\right|$$ Using $|\max_a f(a) - \max_a g(a)| \leq \max_a |f(a) - g(a)|$: $$\leq \gamma \max_a \mathbb{E}[|u(s') - v(s')|] \leq \gamma \|u - v\|_\infty$$ So $\|\mathcal{T}u - \mathcal{T}v\|_\infty \leq \gamma\|u-v\|_\infty$. Since $\gamma < 1$, $\mathcal{T}$ is a strict contraction.

Banach fixed-point theorem. Any strict contraction on a complete metric space has a unique fixed point, and repeated application $v_{k+1} = \mathcal{T}v_k$ converges to it from any starting $v_0$. The fixed point satisfies $v_* = \mathcal{T}v_*$, which is exactly the Bellman optimality equation. $\blacksquare$

This also proves that value iteration converges (Lesson 5): each application of $\mathcal{T}$ halves the error by $\gamma$, so convergence is geometric.

Optimal backup diagrams. The backup diagram for $v_*(s)$ has a flat arc (max over actions) instead of the weighted sum over $\pi(a|s)$. This is the distinguishing visual feature: expectation backups average over actions; optimality backups maximise over them.
Worked example: value functions for the recycling robot
Example 1 — Comparing two policies on the recycling robot
Using the recycling robot MDP from Lesson 3 ($\alpha = 0.7$, $\beta = 0.4$, $r_\text{search} = 2$, $r_\text{wait} = 1$, $r_\text{rescue} = -3$, $\gamma = 0.9$), compute $v_\pi$ for two specific policies and determine which dominates.
Policy A — always search from high, always recharge from low. Under this policy the Bellman equations are:
Policy A: search/high, recharge/low · γ = 0.9
From high, search always: reward is 2 regardless of outcome. Transitions: high→high with 0.7, high→low with 0.3.
$$v_\pi(\text{high}) = 2 + 0.9\bigl[0.7\cdot v_\pi(\text{high}) + 0.3\cdot v_\pi(\text{low})\bigr]$$
From low, recharge always: reward is 0, always→high.
$$v_\pi(\text{low}) = 0 + 0.9\cdot v_\pi(\text{high})$$
Solve:
Substitute $v_\pi(\text{low}) = 0.9\,v_\pi(\text{high})$:
$v_\pi(\text{high}) = 2 + 0.9[0.7 + 0.3 \times 0.9]\,v_\pi(\text{high})$
$v_\pi(\text{high}) = 2 + 0.9 \times 0.97\,v_\pi(\text{high})$
$v_\pi(\text{high})(1 - 0.873) = 2$
$v_\pi(\text{high}) = 2 / 0.127 \approx \mathbf{15.75}$
$v_\pi(\text{low}) = 0.9 \times 15.75 \approx \mathbf{14.17}$
Policy B — always search from high, always search from low (risky). Under this policy:
Policy B: search/high, search/low · γ = 0.9
From low, search: $r(\text{low,search}) = 0.6(-3) + 0.4(2) = -1$. Transitions: low→high with 0.6, low→low with 0.4.
$$v_\pi(\text{high}) = 2 + 0.9[0.7\,v_h + 0.3\,v_l]$$
$$v_\pi(\text{low}) = -1 + 0.9[0.6\,v_h + 0.4\,v_l]$$
Solve the 2×2 linear system:
$0.37\,v_h - 0.27\,v_l = 2$   ...(1)
$-0.54\,v_h + 0.64\,v_l = -1$  ...(2)

From (2): $v_l = (-1 + 0.54\,v_h)\,/\,0.64$
Sub into (1): $0.37\,v_h - 0.27(-1+0.54\,v_h)/0.64 = 2$
$0.091\,v_h = 1.01$
$v_\pi(\text{high}) \approx \mathbf{11.10}$,   $v_\pi(\text{low}) \approx \mathbf{7.80}$
Comparison. Policy A (search/recharge): $v_\pi(\text{high}) = 15.75$, $v_\pi(\text{low}) = 14.17$. Policy B (search/search): $v_\pi(\text{high}) = 11.10$, $v_\pi(\text{low}) = 7.80$. Policy A dominates Policy B in every state — the expected penalty of running flat is too high to justify searching from low battery. This is exactly the kind of comparison value functions make possible: given two policies, compute their values and pick the better one.

Why we can't just solve the Bellman equations
For small, fully-known MDPs like the recycling robot, solving the Bellman equations as a linear system is perfectly feasible. But for realistic problems, this approach breaks down on three fronts:
  • The model is usually unknown. The dynamics function $p(s',r \mid s,a)$ is the complete rulebook of the environment. For a robot navigating a physical space, a game with complex rules, or a financial market, no such rulebook exists in closed form. We can sample trajectories, but we cannot write down $p$.
  • The state space is enormous. A backgammon game has roughly $10^{20}$ states; Go has about $10^{170}$. A linear system with $10^{20}$ equations is not just computationally infeasible — it cannot even be stored. This is the curse of dimensionality: state spaces grow exponentially with the number of variables describing the system.
  • The optimality equations are nonlinear. Even if the model is known and the state space is manageable, the Bellman optimality equations involve a $\max_a$ that makes the system nonlinear, ruling out direct matrix-based solutions.
Each of these obstacles motivates a different class of algorithms. Lesson 5 (Dynamic Programming) assumes the model is known and iteratively solves the Bellman equations rather than setting up the full linear system. Lesson 6 (Monte Carlo) drops the model requirement by estimating value functions from sampled episodes. Lesson 7 (Temporal Difference) bootstraps one-step estimates, combining the model-free sampling of MC with the step-by-step updates of DP. Lesson 8 (Function Approximation) addresses the large-state-space problem by parameterising the value function.
Food for thought
Reflection questions
  1. The Bellman equations form a linear system $v = r + \gamma P v$, where $P$ is the transition matrix under policy $\pi$ and $r$ is the expected reward vector. The solution is $v = (I - \gamma P)^{-1} r$. Why is $I - \gamma P$ guaranteed to be invertible for $\gamma < 1$? What happens at $\gamma = 1$ for a continuing task with no absorbing state?
    The stochastic transition matrix $P$ has spectral radius exactly 1 (all eigenvalues satisfy $|\lambda| \leq 1$; the dominant eigenvalue is 1). Scaling by $\gamma < 1$ shrinks the spectral radius to $\gamma < 1$. By the Neumann series, $(I - M)^{-1} = \sum_{k=0}^\infty M^k$ converges whenever the spectral radius of $M$ is less than 1 — so $(I - \gamma P)$ is invertible. At $\gamma = 1$, if $P$ is irreducible (continuing task, no absorbing state), eigenvalue 1 of $P$ means $I - P$ has a zero eigenvalue and is singular. The linear system has no unique solution — undiscounted values diverge for a task that never terminates.
  2. An agent knows $v_*(s)$ for every state but does not know $q_*(s,a)$ or the dynamics $p(s',r|s,a)$. Can it still recover the optimal policy? If yes, how? If no, why not?
    No — not without the model. Acting greedily with respect to $v_*$ requires $\arg\max_a \sum_{s',r} p(s',r \mid s,a)[r + \gamma v_*(s')]$, which needs the dynamics $p(s',r \mid s,a)$ to evaluate the one-step lookahead. This is why $q_*$ is strictly more useful than $v_*$ in the model-free setting: $\pi_*(s) = \arg\max_a\, q_*(s,a)$ requires no model at all. When designing a model-free RL algorithm, prefer learning $q$ over $v$ — unless you separately learn a model of the environment.
  3. Consider an MDP where all reward is received only at terminal states (zero reward everywhere else). How does the value function behave as $\gamma \to 0$ vs $\gamma \to 1$? What does this tell you about the role of discounting in credit assignment?
    As $\gamma \to 0$: the agent becomes purely myopic — only the immediate next reward matters. Since all non-terminal rewards are 0, $v_\pi(s) \to 0$ for every non-terminal state regardless of how close $s$ is to the terminal reward. The agent cannot see the end of the task at all. As $\gamma \to 1$: the terminal reward is discounted by $\gamma^T$ where $T$ is the time-to-terminal; as $\gamma \to 1$ this approaches 1, so $v_\pi(s)$ approaches the undiscounted expected terminal reward. The whole trajectory is valued equally. This illustrates that $\gamma$ is a credit-assignment dial: low $\gamma$ collapses credit to the immediate future; high $\gamma$ distributes it over arbitrarily long horizons. Sparse-reward tasks (reward only at goal) require $\gamma$ close to 1.
Common pitfalls
Mistakes worth knowing about in advance
  • Confusing $v_\pi$ with $v_*$. $v_\pi(s)$ is the value of state $s$ under a specific policy $\pi$ — it changes if you change $\pi$. $v_*(s)$ is the maximum value achievable from $s$ across all policies. Most algorithms estimate $v_\pi$ for some $\pi$ and then improve $\pi$; don't assume your estimated value function is already optimal.
  • The Bellman equation is a characterisation, not an algorithm. The Bellman expectation equation tells you what $v_\pi$ must look like — it is a constraint. It does not tell you how to compute $v_\pi$. The algorithms in Lessons 5–7 use the Bellman equation as their update target, but the method of computing iterative updates is an additional idea on top.
  • Greedy with respect to $v_\pi \neq$ greedy with respect to $v_*$. Acting greedily with respect to the current $v_\pi$ (choosing the action that maximises $r + \gamma v_\pi(s')$) improves the policy but does not produce the optimal policy in one step. You need to re-evaluate and re-improve repeatedly; this is the policy iteration loop of Lesson 5.
  • Assuming the optimal policy is unique. Multiple policies can achieve $v_*$ simultaneously — ties in action values are common, and any tie-breaking rule yields an optimal policy. The optimal value function $v_*$ is unique; the optimal policy is not necessarily.
Connects to
  • Probability Theory — the Bellman equation derivation uses the tower property of conditional expectation: $\mathbb{E}[G_{t+1} \mid S_t = s] = \sum_{s'} \Pr\{S_{t+1}=s' \mid S_t=s\}\,\mathbb{E}[G_{t+1} \mid S_{t+1}=s']$. Understanding this step requires fluency with conditional expectations.
  • Linear Algebra — the Bellman expectation equations form the linear system $v = r + \gamma P v$, where $P$ is the policy-weighted transition matrix. The solution $(I - \gamma P)^{-1}r$ exists and is unique because $\|{\gamma P}\| < 1$ when $\gamma < 1$; this is the Neumann series argument.
  • Control Theory — the Bellman optimality equation is the discrete-time analog of the Hamilton-Jacobi-Bellman (HJB) equation in continuous optimal control. LQR (Lesson 13 of Control Theory) solves the HJB for linear dynamics with quadratic cost — it is RL's Bellman equation made tractable by algebraic structure.
