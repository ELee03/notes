6 · Monte Carlo Methods
Model-Free Learning · Episode Returns · First-Visit MC · MC Control · Importance Sampling
Dynamic programming gives you exact value functions — but only if you already know the transition dynamics $p(s',r \mid s,a)$. For most real problems that model is unavailable: you can interact with the environment and observe what happens, but you cannot enumerate all outcomes and their probabilities. Monte Carlo methods solve this by learning directly from experience. Run complete episodes, observe the returns, and average. No model required — just data from the world.
Notation used in this lesson
$G_t$ Discounted return from step $t$: $G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots$
$T$ Terminal time step — when the episode ends
$\mathcal{T}(s)$ Set of all time steps at which state $s$ was visited (across all episodes)
$N(s)$ Number of times state $s$ has been (first) visited; used as denominator in running average
$b$ Behavior policy — the policy actually used to generate episodes (may differ from $\pi$)
$\rho_{t:T-1}$ Importance sampling ratio: $\prod_{k=t}^{T-1} \dfrac{\pi(A_k \mid S_k)}{b(A_k \mid S_k)}$
The Monte Carlo idea
Recall Lesson 2. To estimate $q_*(a)$ — the expected reward of action $a$ — you simply pulled that arm many times and averaged the rewards. No model of the slot machine was needed; repeated sampling and the law of large numbers did the work.
Monte Carlo prediction is the same idea, applied to complete episodes. The value $v_\pi(s)$ is by definition the expected discounted return starting from $s$ under $\pi$: $$v_\pi(s) = \mathbb{E}_\pi[G_t \mid S_t = s]$$ If you observe many returns from episodes that visited $s$, their average converges to this expectation. That is MC prediction in one sentence.
What makes MC different from DP: DP uses the Bellman equation to bootstrap — it updates $V(s)$ from $V(s')$ without waiting for the episode to end. MC waits for the whole episode, then uses the actual return $G_t$ (not an estimated one). No model, no bootstrapping — but you must wait until the episode terminates. This means MC is only applicable to episodic tasks.
MC prediction: estimating $v_\pi$
Given a policy $\pi$, the goal is to estimate $v_\pi(s)$ for all states $s$. The algorithm is straightforward:
  1. Initialize $V(s) = 0$ and $N(s) = 0$ for all $s$.
  2. Generate an episode by following $\pi$: $S_0, A_0, R_1, S_1, A_1, R_2, \ldots, S_{T-1}, A_{T-1}, R_T$.
  3. For each state $s$ that appeared in the episode, compute the return $G_t$ from the time $t$ it was visited.
  4. Update the running average: $N(s) \leftarrow N(s)+1$, then $V(s) \leftarrow V(s) + \frac{1}{N(s)}[G_t - V(s)]$.
  5. Repeat from step 2.
The update in step 4 is the incremental sample-mean formula — exactly the same rule used to update action-value estimates in the bandit algorithm. Here the "observation" is the full return $G_t$, and the "count" $N(s)$ is the number of times $s$ has been sampled. As $N(s) \to \infty$, $V(s) \to v_\pi(s)$ by the law of large numbers.
The update can also be written with a fixed step size $\alpha$ instead of $1/N(s)$: $$V(S_t) \leftarrow V(S_t) + \alpha[G_t - V(S_t)]$$ The $1/N$ version gives equal weight to all past returns (appropriate for stationary environments). Fixed $\alpha$ gives more weight to recent returns (appropriate when the environment — or the policy — is changing).
First-visit vs every-visit
A single episode may visit the same state multiple times — for instance, a random walk through a grid world might loop back. This creates an ambiguity: how many times do you count that state?
First-visit MC counts only the first occurrence of each state in each episode. If state $s$ appears at $t = 3$ and again at $t = 9$, only the return from $t = 3$ is used. The returns across different episodes are then averaged.
Every-visit MC counts every occurrence. Both the return from $t = 3$ and from $t = 9$ are included in the average for $V(s)$.
Both converge to $v_\pi(s)$ with enough episodes. First-visit has cleaner theoretical properties (the returns used are i.i.d.), while every-visit extracts more data per episode. In practice the difference is minor. The lesson primarily uses first-visit MC.
Efficient computation of returns: Rather than computing $G_t$ forward from each step (which would be $O(T^2)$), compute backwards in a single pass. Start at $t = T-1$ with $G = 0$. At each step moving backward: $G \leftarrow \gamma G + R_{t+1}$. This gives $G_t$ at each $t$ in $O(T)$ total. Then record the first occurrence of each state as you pass through.
MC control: finding an optimal policy
Prediction estimates $v_\pi$ for a fixed policy. Control means improving the policy toward optimality. The GPI structure from Lesson 5 applies here too: alternate between evaluation and improvement.
But there is a complication. Policy improvement in DP was: $$\pi'(s) = \arg\max_a \sum_{s',r} p(s',r \mid s,a)[r + \gamma v_\pi(s')]$$ This requires the model $p$ to look one step ahead. Without a model, you cannot extract a better policy from $v_\pi(s)$ alone — you need $q_\pi(s,a)$.
Solution: estimate $Q(s,a)$ instead of $V(s)$. Apply the same MC averaging idea to action-value pairs: $$Q(s,a) \leftarrow Q(s,a) + \alpha[G_t - Q(s,a)]$$ where $G_t$ is the return following the visit to $(s,a)$. Now policy improvement is model-free: $\pi'(s) = \arg\max_a Q(s,a)$.
There is a second complication: to estimate $Q(s,a)$ accurately, every state-action pair must be visited frequently. But if the policy is deterministic, it will never take the non-greedy actions — those $Q(s,a)$ values are never updated. Two solutions exist:
Exploring starts. Mandate that every episode starts from a randomly chosen $(s_0, a_0)$ pair, with each pair having nonzero probability of being selected. This guarantees that in the limit, every pair is visited infinitely often. Unrealistic (you cannot usually choose your starting position arbitrarily), but useful as a theoretical baseline.
ε-soft policies. Instead of a deterministic policy, use an ε-soft policy: every action has probability at least $\varepsilon / |\mathcal{A}(s)|$. The greedy action gets probability $1 - \varepsilon + \varepsilon / |\mathcal{A}(s)|$. The agent always has some chance of exploring. This is covered in the next section.
ε-soft on-policy MC control
An ε-soft policy is a generalization of $\varepsilon$-greedy from Lesson 2. Formally: for all $s$ and $a$, $\pi(a \mid s) \geq \varepsilon / |\mathcal{A}(s)|$. The greedy action takes the remaining probability mass.
The on-policy MC control algorithm with ε-soft policies:
  1. Initialize $Q(s,a)$ arbitrarily, $N(s,a) = 0$. Set $\pi$ to any ε-soft policy.
  2. Generate an episode under $\pi$.
  3. For each $(s,a)$ pair in the episode (first-visit), update $Q(s,a)$ using the return $G_t$ from that visit.
  4. Update $\pi$ to be ε-greedy with respect to current $Q$: take the greedy action with probability $1-\varepsilon+\varepsilon/|\mathcal{A}(s)|$, and each other action with probability $\varepsilon/|\mathcal{A}(s)|$.
  5. Repeat from step 2.
This converges to the optimal policy among all ε-soft policies — which is the best you can do while maintaining ε-exploration. It is not quite $\pi^*$ (the hard-greedy optimal policy), but it approaches $\pi^*$ as $\varepsilon \to 0$. In practice, decaying $\varepsilon$ over time lets the algorithm converge toward the true optimum.
⟶ Worth thinking through: Why does policy improvement still work when switching from $\pi$ to its ε-greedy version? You might expect that adding random exploration could hurt. The key insight is that for ε-soft policies, the policy improvement theorem generalizes: every ε-greedy policy with respect to $Q_\pi$ is at least as good as $\pi$ itself (both are ε-soft, so the comparison is valid). Work through the one-step proof of this — it mirrors the original policy improvement theorem exactly.
Off-policy MC
On-policy methods learn the value of the policy being followed. A more powerful separation is possible: use one policy $b$ (the behavior policy) to generate data, while learning about a different policy $\pi$ (the target policy). This is off-policy learning.
Off-policy is important for several reasons:
  • Learning from prior data. If you have logs of episodes generated by some old policy (or a human expert), you can extract value estimates for a new policy $\pi$ without running any new episodes.
  • Aggressive exploration while learning a safe policy. Let $b$ be highly exploratory (even uniform random), while $\pi$ is the greedy policy you are trying to improve. You can explore freely without compromising the policy you deploy.
  • Learning multiple policies simultaneously from a single stream of experience.
The requirement for off-policy MC: $b$ must be soft enough to cover $\pi$ — that is, for any action $\pi$ would take, $b$ must sometimes take it too. Formally: $\pi(a \mid s) > 0 \Rightarrow b(a \mid s) > 0$.
Importance sampling
The challenge is that episodes are generated by $b$, but we want to estimate $v_\pi(s) = \mathbb{E}_\pi[G_t \mid S_t = s]$ — an expectation under $\pi$. We cannot just average the returns from $b$-generated episodes, because those returns were produced by a different policy and have the wrong distribution.
Importance sampling fixes this by re-weighting each return. The key identity: if you want $\mathbb{E}_\pi[X]$ but can only sample under $b$, $$\mathbb{E}_\pi[X] = \mathbb{E}_b\!\left[\frac{\pi(X)}{b(X)} \cdot X\right]$$ where $\pi(X)/b(X)$ is the ratio of how likely $X$ is under $\pi$ vs under $b$. For a trajectory starting at $S_t$ with actions $A_t, \ldots, A_{T-1}$:
$$\rho_{t:T-1} = \prod_{k=t}^{T-1} \frac{\pi(A_k \mid S_k)}{b(A_k \mid S_k)}$$
$\rho_{t:T-1}$ is the importance sampling ratio: how much more (or less) likely this specific sequence of actions was under $\pi$ than under $b$. Then:
$$\mathbb{E}_b[\rho_{t:T-1} \cdot G_t \mid S_t = s] = v_\pi(s)$$
So we can estimate $v_\pi(s)$ by collecting returns $G_t$ under $b$, multiplying each by $\rho_{t:T-1}$, and averaging.
Two variants differ in how they average:
Ordinary importance sampling: $$V(s) = \frac{\displaystyle\sum_{t \in \mathcal{T}(s)} \rho_{t:T(t)-1}\, G_t}{|\mathcal{T}(s)|}$$ This is an unbiased estimator of $v_\pi(s)$. However, $\rho$ can be very large when $b$ and $\pi$ disagree a lot, making this estimator high-variance — sometimes catastrophically so.
Weighted importance sampling: $$V(s) = \frac{\displaystyle\sum_{t \in \mathcal{T}(s)} \rho_{t:T(t)-1}\, G_t}{\displaystyle\sum_{t \in \mathcal{T}(s)} \rho_{t:T(t)-1}}$$ This normalizes the weights so they sum to 1. The estimator is biased (converges at a slightly slower rate for any finite number of episodes), but its variance is bounded — in practice, much better behaved.
The variance problem. $\rho_{t:T-1}$ is a product over many steps. If $\pi$ and $b$ disagree even slightly at each step, the product grows or shrinks exponentially with episode length. In a 50-step episode, if $\rho$ at each step is 1.1 on average, $\rho_{0:49} \approx 1.1^{50} \approx 117$. Weighted IS mitigates this; per-decision importance sampling (which we do not cover here) provides further variance reduction. For very long episodes or large policy differences, off-policy MC becomes impractical — Temporal Difference methods (Lesson 7) handle this much better.

After just two episodes, states near the terminal corners already have accurate estimates ($V(11) = V(14) = -1$, which equals the true distance to the nearest terminal). States farther in (like 10) have noisier estimates — they converge slower because they appear in fewer episode trajectories, and when they do appear, their return depends on the random walk taken from that point.
Example 2 — Importance sampling calculation
Suppose you are in a 2-action MDP. The behavior policy is $b(a=\text{left} \mid s) = 0.5$, $b(a=\text{right} \mid s) = 0.5$. The target policy is $\pi(a=\text{left} \mid s) = 1.0$ (deterministic left). A 3-step episode under $b$ takes actions (left, left, right). Compute the importance sampling ratio $\rho_{0:2}$ and explain what the result means.
$$\rho_{0:2} = \frac{\pi(\text{left})}{b(\text{left})} \cdot \frac{\pi(\text{left})}{b(\text{left})} \cdot \frac{\pi(\text{right})}{b(\text{right})} = \frac{1.0}{0.5} \cdot \frac{1.0}{0.5} \cdot \frac{0.0}{0.5} = 0$$
The ratio is zero because $\pi$ would never take the third action (right). This episode is irrelevant for estimating $v_\pi$ — it followed a path that $\pi$ would never follow, so its return carries zero weight. Only episodes where every action taken is consistent with $\pi$ contribute nonzero weight.
Now suppose all three actions were left: $\rho_{0:2} = (1/0.5)^3 = 8$. This episode's return gets multiplied by 8 — it counts heavily because $\pi$ would always take this path, but $b$ only took it by chance.

Monte Carlo vs Dynamic Programming
Property DP Monte Carlo
Model required? Yes — needs $p(s',r|s,a)$ explicitly No — learns from sampled episodes
Bootstrapping? Yes — updates $V(s)$ using $V(s')$ No — uses actual return $G_t$
Applies to Episodic and continuing tasks Episodic tasks only
Update timing After each step (sweep over all states) After each complete episode
Bias Biased (depends on $V$ initialization) until convergence Unbiased (first-visit) — true sample mean
Variance Low (bootstraps from neighbors) High (full episode return is noisy)
State coverage Visits every state every sweep Only visits states on sampled trajectories
The bias-variance tradeoff is a recurring theme: DP bootstraps, which introduces bias (the estimate depends on other potentially wrong estimates) but reduces variance. MC uses true returns, which is unbiased but noisier. Temporal Difference learning (Lesson 7) sits between these extremes, bootstrapping like DP but from incomplete episodes like MC.
Food for thought
Reflection questions
  1. First-visit MC is unbiased — the expected value of the estimate is exactly $v_\pi(s)$ regardless of how $V$ was initialized. Every-visit MC is also unbiased in the limit, but subtly biased for finite samples. Why? (Hint: think about what happens when a state appears twice in one episode and what the two returns represent relative to the definition of $v_\pi(s)$.)
    Answer / hint
    When a state $s$ appears at time $t_1$ and again at $t_2 > t_1$ in the same episode, first-visit MC uses only $G_{t_1}$ — which is the return from the first visit, an unbiased sample of $v_\pi(s)$. Every-visit MC also uses $G_{t_2}$, but this return is not an independent sample: the trajectory from $t_2$ is a subset of the trajectory from $t_1$, so $G_{t_2}$ is correlated with $G_{t_1}$ and is strictly shorter. For a finite number of episodes, averaging these correlated returns introduces a bias. As episodes → ∞, the fraction of "re-visit" returns goes to zero relative to total samples, so every-visit MC becomes asymptotically unbiased — but at finite sample sizes it overweights states that loop back to themselves.
  2. In the importance sampling derivation, the ratio $\rho_{t:T-1}$ only involves the actions taken ($\pi(A_k|S_k)/b(A_k|S_k)$), not the transition probabilities $p(s'|s,a)$. Why does the state transition probability cancel out? What assumption about the environment makes this cancellation valid?
    Answer / hint
    The IS ratio is the ratio of trajectory probabilities under $\pi$ vs. $b$: $$\rho = \frac{\prod_k \pi(A_k|S_k)\,p(S_{k+1}|S_k,A_k)}{\prod_k b(A_k|S_k)\,p(S_{k+1}|S_k,A_k)}$$ The transition probability $p(S_{k+1}|S_k,A_k)$ appears identically in numerator and denominator and cancels. The key assumption is that the transition dynamics are the same for both $\pi$ and $b$ — they are policies acting in the same environment. This always holds when both policies interact with a fixed MDP: the environment dynamics are independent of the policy.
  3. Suppose the environment has very long episodes — thousands of steps to reach a terminal. How does this affect the relative performance of MC vs. DP (assuming DP had a model)? What about very short episodes of 2–3 steps?
    Answer / hint
    Long episodes: MC suffers badly. Each episode is expensive to collect, and the variance of the return (a sum of T random variables) grows with episode length. Off-policy IS ratios become exponentially extreme over long trajectories. DP, with a known model, suffers none of this — each sweep is a deterministic matrix operation, and information propagates back efficiently. MC is practical only when DP's model assumption fails.

    Short episodes (2–3 steps): MC shines. Returns have very low variance (just 2–3 reward terms). Episodes are cheap to collect. Even without a model, you get accurate estimates quickly. Off-policy ratios are just 2–3 terms — manageable. For short tasks (e.g., simple bandit-like decision problems), MC can outperform DP in wall-clock time even when the model is known.
Common pitfalls
Mistakes worth knowing about in advance
  • Applying MC to continuing tasks. MC requires complete episodes to compute $G_t$. For continuing tasks (no natural terminal state — e.g. a robot that runs forever), there is no episode to average over. DP and TD methods handle this naturally; MC does not. Before reaching for MC, check that your task is episodic.
  • Computing returns forward instead of backward. Computing $G_t = R_{t+1} + \gamma R_{t+2} + \cdots$ forward for each $t$ costs $O(T^2)$ per episode. The backward pass ($G \leftarrow \gamma G + R_{t+1}$) gives all returns in $O(T)$. This is not a conceptual error, but failing to use the backward pass makes large episodes very slow.
  • Confusing on-policy and off-policy settings. In on-policy MC, you evaluate the policy you are following. In off-policy MC, you evaluate $\pi$ while following $b$ — and must apply importance sampling. Forgetting to weight returns by $\rho$ in an off-policy setting produces biased estimates that converge to $v_b$, not $v_\pi$.
  • Ignoring the coverage requirement. Off-policy MC requires that $b$ covers $\pi$: wherever $\pi$ chooses an action, $b$ must sometimes choose it too. If $b$ never takes some action that $\pi$ would, $\rho$ will always be zero for those trajectories and you get no signal about the value of those states under $\pi$. This is an algorithmic failure, not just high variance.
  • Expecting off-policy MC to work well with long episodes. The importance sampling ratio $\rho_{t:T-1}$ is a product over potentially many steps. Even small per-step discrepancies between $\pi$ and $b$ make this product extreme — causing estimates to have enormous variance or collapse to near-zero. Off-policy MC is practical mainly for short episodes or when $\pi \approx b$. For longer horizons, use off-policy TD (Lesson 7) instead.
Connects to
  • RL · Multi-Armed Bandits (Lesson 2) — the sample-average update $Q \leftarrow Q + \frac{1}{N}[R-Q]$ in bandits is exactly MC prediction for a single-state problem with one step per episode. Every concept in this lesson is a direct generalization of that simpler setting.
  • Probability Theory · Law of Large Numbers — the convergence guarantee for MC prediction follows directly from the LLN: the sample mean converges almost surely to the expectation as $N \to \infty$. The rate of convergence is governed by the variance of $G_t$, which is why high-variance episodes (long trajectories, large rewards) slow down convergence.
  • RL · Temporal Difference Learning (Lesson 7) — TD unifies the DP and MC approaches: it bootstraps like DP but does not need a model, and it updates every step rather than waiting for episode completion. Many of the limitations of MC (episodic-only, high variance, no bootstrapping) are directly addressed by TD.
