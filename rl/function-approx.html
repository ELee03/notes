8 · Function Approximation
Parameterized Value Functions · Semi-Gradient TD · Linear Approximation · Neural Networks · Deadly Triad · DQN
Everything in Lessons 4–7 stored one number per state — a table. For small, discrete problems this works fine. But chess has roughly $10^{44}$ states; continuous robot joint angles have infinitely many. You cannot store a separate value for each. Function approximation replaces the table with a compact parameterized function: store a weight vector $\mathbf{w}$ and evaluate $\hat{v}(s, \mathbf{w})$ on demand. This is the step that connects classical RL to deep learning — and it introduces a new set of challenges that require careful handling.
Notation used in this lesson
$\hat{v}(s, \mathbf{w})$ Parameterized approximation to $v_\pi(s)$; $\mathbf{w} \in \mathbb{R}^d$ are the weights
$\mathbf{x}(s)$ Feature vector of state $s$: $\mathbf{x}(s) \in \mathbb{R}^d$; for linear FA, $\hat{v}(s,\mathbf{w}) = \mathbf{w}^\top\mathbf{x}(s)$
$\nabla\hat{v}(s,\mathbf{w})$ Gradient of $\hat{v}$ with respect to $\mathbf{w}$; for linear FA this equals $\mathbf{x}(s)$
$\overline{VE}(\mathbf{w})$ Mean Squared Value Error: $\sum_s \mu(s)[v_\pi(s) - \hat{v}(s,\mathbf{w})]^2$
$\mu(s)$ On-policy state distribution — how often state $s$ is visited under $\pi$
$\delta_t$ TD error (same as Lesson 7): $R_{t+1} + \gamma\hat{v}(S_{t+1},\mathbf{w}) - \hat{v}(S_t,\mathbf{w})$
Why function approximation?
The tabular methods in Lessons 4–7 make one strong assumption: you can maintain a separate value estimate for each state (and each state-action pair, for $Q$). This breaks down in two ways:
  • Storage: Real problems have enormous or continuous state spaces. A robot arm with 6 joints at $10^{-3}$ rad precision has roughly $6000^6 \approx 5\times10^{22}$ states. Storing one float per state is impossible.
  • Generalization: Even if you could store $10^{22}$ values, you could never visit all states during training. A tabular agent that visits state $s$ updates only $V(s)$ — nothing else. A function approximator that visits $s$ can update $\hat{v}$ in a way that also improves estimates for nearby states that were never visited. This is the key benefit: generalization.
Function approximation is an instance of supervised learning applied to the RL problem. Instead of a fixed dataset, the "training examples" come from interactions with the environment, and the "labels" are either true values (if known), Monte Carlo returns, or TD targets. The complication — and much of this lesson — is what happens when the labels are themselves produced by the function being trained.
Parameterized value functions
Replace the table $V : \mathcal{S} \to \mathbb{R}$ with a function $\hat{v}(s, \mathbf{w})$ parameterized by a weight vector $\mathbf{w} \in \mathbb{R}^d$. The function can be:
  • Linear: $\hat{v}(s,\mathbf{w}) = \mathbf{w}^\top \mathbf{x}(s)$ where $\mathbf{x}(s)$ is a feature vector. Simple, stable, converges provably — but limited to functions expressible as linear combinations of the features.
  • Neural network: $\hat{v}(s,\mathbf{w})$ is a multi-layer network with weights $\mathbf{w}$. Universal approximation — can represent any continuous function — but non-convex optimization and stability issues arise.
  • Decision trees, radial basis functions, etc.: Other function classes are possible; the gradient-descent framework below applies whenever the function is differentiable w.r.t. $\mathbf{w}$.
The key change in the math: instead of looking up $V(s)$ in a table, you compute $\hat{v}(s,\mathbf{w})$ by evaluating the function. Instead of updating $V(s)$, you update $\mathbf{w}$ via gradient descent. The single weight vector $\mathbf{w}$ encodes value estimates for all states simultaneously.
The value error objective
What are we trying to minimize? The most natural objective is the Mean Squared Value Error (VE), weighted by the state distribution under the target policy:
$$\overline{VE}(\mathbf{w}) = \sum_{s \in \mathcal{S}} \mu(s)\bigl[v_\pi(s) - \hat{v}(s, \mathbf{w})\bigr]^2$$
The weighting $\mu(s)$ matters: we care more about getting accurate values in states we actually visit. Under on-policy training, $\mu(s)$ is the stationary distribution of the Markov chain induced by $\pi$ — states visited most often are weighted most heavily. This is exactly the right thing to optimize when the goal is good performance under $\pi$.
Note: $v_\pi(s)$ is the true value we're approximating. In practice we don't know it — we have to substitute a target. This substitution is where MC, TD, and DP diverge.
Gradient descent on value error
The gradient of $\overline{VE}$ with respect to $\mathbf{w}$ is: $$\nabla_\mathbf{w}\overline{VE} = -2\sum_s \mu(s)\bigl[v_\pi(s) - \hat{v}(s,\mathbf{w})\bigr]\nabla_\mathbf{w}\hat{v}(s,\mathbf{w})$$ Stochastic gradient descent (SGD) samples a single state $S_t \sim \mu$ and uses its gradient as an unbiased estimate of the full gradient:
$$\mathbf{w}_{t+1} = \mathbf{w}_t + \alpha\bigl[v_\pi(S_t) - \hat{v}(S_t, \mathbf{w}_t)\bigr]\nabla_\mathbf{w}\hat{v}(S_t, \mathbf{w}_t)$$
With a suitable step size schedule ($\sum_t \alpha_t = \infty$, $\sum_t \alpha_t^2 < \infty$), this converges to a local minimum of $\overline{VE}$ (global minimum for linear FA, since VE is convex in $\mathbf{w}$ for linear approximators).
The challenge: $v_\pi(S_t)$ is unknown. We substitute a target $U_t \approx v_\pi(S_t)$:
  • MC target: $U_t = G_t$ (actual return). Unbiased. SGD with MC targets converges to the VE minimum.
  • DP target: $U_t = \sum_a \pi(a|S_t) \sum_{s',r} p(s',r|S_t,a)[r + \gamma\hat{v}(s',\mathbf{w})]$. Requires a model; not truly SGD.
  • TD target: $U_t = R_{t+1} + \gamma\hat{v}(S_{t+1},\mathbf{w})$. Model-free and online — but biased, because $U_t$ depends on $\mathbf{w}$.
Semi-gradient TD
Substituting the TD target $U_t = R_{t+1} + \gamma\hat{v}(S_{t+1}, \mathbf{w}_t)$ into the gradient update gives the semi-gradient TD rule:
$$\mathbf{w}_{t+1} = \mathbf{w}_t + \alpha\underbrace{\bigl[R_{t+1} + \gamma\hat{v}(S_{t+1},\mathbf{w}_t) - \hat{v}(S_t,\mathbf{w}_t)\bigr]}_{\delta_t} \nabla_\mathbf{w}\hat{v}(S_t, \mathbf{w}_t)$$
The "semi" in semi-gradient refers to the fact that we differentiate only $\hat{v}(S_t, \mathbf{w})$ — we treat the target $R_{t+1} + \gamma\hat{v}(S_{t+1},\mathbf{w}_t)$ as a fixed constant for the purpose of differentiation, even though it also depends on $\mathbf{w}_t$. A true gradient step would require differentiating through the target too, yielding the "residual gradient" algorithm — which is unbiased but slower in practice.
Semi-gradient TD is not following the gradient of any fixed objective, which means standard convergence guarantees for SGD don't apply. For linear function approximation with on-policy data, convergence can be proved independently — it converges to a fixed point within a bounded distance of the VE minimum, called the TD fixed point. For nonlinear approximators or off-policy data, convergence is not guaranteed (see: Deadly Triad).
⟶ Worth seeing explicitly: The semi-gradient TD update looks like a supervised learning gradient step with "label" = TD target. But the label changes every time $\mathbf{w}$ changes (because the target includes $\hat{v}(S_{t+1}, \mathbf{w})$). This is the moving-target problem: you are chasing a moving bullseye. For linear FA on-policy, the proof shows the bullseye moves slowly enough that you still converge. For neural networks off-policy, the bullseye can move faster than you can follow — leading to oscillation or divergence. DQN fixes this with a frozen target network (see later).
Linear function approximation
Linear FA is the most theoretically well-understood case. Represent value as a dot product: $$\hat{v}(s,\mathbf{w}) = \mathbf{w}^\top \mathbf{x}(s) = \sum_{i=1}^d w_i x_i(s)$$ The gradient is simply $\nabla_\mathbf{w}\hat{v}(s,\mathbf{w}) = \mathbf{x}(s)$. The semi-gradient TD update becomes:
$$\mathbf{w}_{t+1} = \mathbf{w}_t + \alpha \delta_t\, \mathbf{x}(S_t)$$
Very simple — the weight update is the TD error times the feature vector. The key design question is how to construct $\mathbf{x}(s)$:
State aggregation (coarsest): divide the state space into groups. $x_i(s) = 1$ if $s$ belongs to group $i$, else 0. All states in the same group share a single value estimate. Easy to implement; poor resolution.
Tile coding (more powerful): use multiple overlapping grids ("tilings"), each offset from the others. State $s$ activates one tile in each tiling; the feature vector is the union of active tiles (a sparse binary vector). Tile coding provides local generalization with well-controlled resolution and is widely used in classical RL (e.g. mountain car).
Radial basis functions: $x_i(s) = \exp(-\|s - c_i\|^2 / 2\sigma^2)$ for center $c_i$. Smooth generalization over continuous state spaces.
Polynomial/Fourier basis: $x_k(s) = s^k$ or $x_k(s) = \cos(k\pi s)$. Useful for smooth value functions over low-dimensional continuous spaces.
Generalization is a double-edged sword. When you update $\mathbf{w}$ from state $s$, the values of all states that share features with $s$ also change. This is what enables learning from fewer samples — but it also means an update for $s$ can hurt the estimate for an unrelated state $s'$ if they happen to share features. Feature design is the art of ensuring that states which share features deserve similar values.
Neural networks as value function approximators
A neural network $\hat{v}(s,\mathbf{w})$ can, in principle, represent any continuous function (universal approximation theorem). The state $s$ — which might be a raw image, a vector of sensor readings, or any structured observation — is fed into the network, and a scalar value is produced.
The semi-gradient TD update still applies: $\mathbf{w} \leftarrow \mathbf{w} + \alpha\delta_t\nabla_\mathbf{w}\hat{v}(S_t,\mathbf{w})$. The gradient $\nabla_\mathbf{w}\hat{v}$ is now computed via backpropagation through the network layers. Conceptually, nothing changes — we're still doing gradient steps with the TD error as the learning signal.
In practice, neural networks introduce several new challenges:
  • Non-convex loss landscape. VE is not convex in $\mathbf{w}$ for nonlinear networks, so gradient descent may get stuck in local minima or saddle points.
  • Catastrophic interference. Updating $\mathbf{w}$ for one state can severely degrade estimates for other states — unlike tabular updates which are local.
  • Correlated updates. Sequential RL data is highly correlated (consecutive states share most features). SGD assumes i.i.d. data; correlated updates cause oscillation and slow convergence.
  • The deadly triad (see next section). Off-policy training + bootstrapping + function approximation can diverge.
The deadly triad
Sutton & Barto identify three ingredients that, when combined, lead to instability or divergence. Each alone, or in pairs, is manageable. All three together create fundamental problems:
Function Approx Bootstrapping Off-Policy Result
Tabular Yes (TD) Yes ✓ Converges
Linear Yes (TD) No (on-policy) ✓ Converges to TD fixed point
Any No (MC) Yes ✓ Converges (true gradient on VE)
Linear Yes (TD) Yes ⚠ May diverge (Baird counterexample)
Nonlinear (e.g. NN) Yes (TD) Yes ✗ Diverges easily; DQN tricks needed
The key insight from the table: removing any one of the three ingredients restores convergence. In practice, all three are often needed. DQN addresses the problem with engineering tricks rather than removing an ingredient.
The Baird counterexample is the canonical demonstration of divergence. It constructs a linear FA + TD + off-policy combination where the weight vector $\mathbf{w}$ grows without bound — even with a linear approximator, which is normally well-behaved. The key is that off-policy updates sample states according to a distribution $\mu$ that differs from the one the convergence proof requires. Reading the counterexample in S&B (Section 11.2) is the clearest way to understand why this happens.
Deep Q-Networks (DQN)
In 2015, DeepMind published a Q-learning algorithm that achieved human-level performance on 49 Atari games using raw pixel input — without any game-specific engineering. The key insight was not the neural network itself (neural nets had been tried in RL before), but two practical tricks that tame the deadly triad:
1. Experience replay. Instead of updating $\mathbf{w}$ from the most recent transition $(S_t, A_t, R_{t+1}, S_{t+1})$ immediately, store transitions in a large replay buffer (typically $10^6$ entries). At each training step, sample a random mini-batch from the buffer. This breaks temporal correlations (nearby transitions are highly correlated) and ensures that each transition is used for multiple gradient updates. Off-policy training is a requirement for replay — the stored transitions were collected by past policies.
2. Target network. Instead of computing the TD target $R_{t+1} + \gamma\max_{a'}Q(S_{t+1}, a'; \mathbf{w})$ using the current weights, use a separate "target network" with weights $\mathbf{w}^-$: $$\text{TD target} = R_{t+1} + \gamma \max_{a'} Q(S_{t+1}, a'; \mathbf{w}^-)$$ The target network is a copy of the main network, updated only every $C$ steps (e.g. $C = 1000$). This makes the target stable during those $C$ steps, preventing the oscillation caused by chasing a moving bullseye.
Together, these two tricks transform the notoriously unstable combination of deep RL into something that trains reliably. DQN is Q-learning with:
  • A convolutional neural network for $Q(s, a; \mathbf{w})$ (raw pixel input)
  • Experience replay buffer to break correlations
  • A frozen target network for stable TD targets
  • Standard SGD (RMSProp) to minimize the squared TD error
DQN does not "solve" the deadly triad — all three ingredients are still present. It works empirically because the tricks make the instability manageable in practice, and because neural networks are surprisingly robust to mild violations of convergence conditions. More theoretically principled alternatives (LSTD, GTD) exist but are less widely used.
Worked examples
Example 1 — Linear FA with state aggregation: three update steps
1D random walk, states $\{1,\ldots,7\}$, terminals 0 (reward 0) and 8 (reward +1). True value: $v_\pi(s) = s/8$. State aggregation: Group A = $\{1,2,3,4\}$, Group B = $\{5,6,7\}$. Features: $\mathbf{x}(s)=[1,0]^\top$ for A, $[0,1]^\top$ for B. Initialize $\mathbf{w}=[0,0]^\top$, $\alpha=0.1$, $\gamma=1$. Walk through three transitions and determine which produce updates.
Step 1: Agent at $s=2$ (Group A). Goes right to $s=3$ (Group A), $R=0$. TD error: $\delta = 0 + w_A - w_A = 0$. No update — same group, same estimate, no information gained.
Step 2: Agent at $s=4$ (Group A). Goes right to $s=5$ (Group B), $R=0$. TD error: $\delta = 0 + w_B - w_A = 0 - 0 = 0$. Still no update — both weights at 0.
Step 3: Agent at $s=7$ (Group B). Goes right to terminal $s=8$, $R=+1$. TD target: $1 + 0 = 1$. TD error: $\delta = 1 - w_B = 1$. Update: $\mathbf{w} \leftarrow [0,0] + 0.1 \times 1 \times [0,1]^\top = [0,\; 0.1]$.
After Step 3, $\hat{v}(5) = \hat{v}(6) = \hat{v}(7) = 0.1$ — all Group B states immediately benefit from the single observation at $s=7$. This is generalization: updating $\hat{v}(7)$ also improves estimates for states 5 and 6 that haven't been visited yet.
Example 2 — Semi-gradient TD vs true gradient descent
$\hat{v}(s,w) = w \cdot s$ (linear in one scalar parameter $w$). Transition: $(s=3, R=0, s'=4)$. Show what the semi-gradient TD update does to $w$, and contrast with what a full gradient step would do. Explain why they differ.
TD target: $0 + 1 \cdot w \cdot 4 = 4w$. TD error: $\delta = 4w - 3w = w$.
Semi-gradient: treat $4w$ as a constant. Gradient of $\hat{v}(3,w) = 3w$ w.r.t. $w$ is 3. Update: $w \leftarrow w + \alpha \cdot w \cdot 3 = w(1 + 3\alpha)$. If $\alpha > 0$, $w$ grows on every step — diverges. (This $\hat{v} = ws$ is a pathological FA; standard linear FA with proper feature vectors converges on-policy.)
True gradient: differentiate through the target too. $\nabla_w[\hat{v}(3,w) - (R + \hat{v}(4,w))]^2 = 2(3w - 4w)(-3 + 4) = 2(-w)(1) = -2w$. This pushes $w$ toward zero — stable. The full gradient corrects for the fact that increasing $w$ also increases the target, which semi-gradient ignores.

Food for thought
Reflection questions
  1. Experience replay requires storing transitions and sampling them randomly — it turns an online method into something batch-like. What happens if your environment changes over time (non-stationarity)? How would you modify the replay buffer to handle a slowly drifting environment?
    With a fixed-size FIFO buffer, old transitions are evicted as new ones arrive — this naturally handles slow drift, since the buffer always reflects the most recent $N$ transitions. For more control: use a sliding window (small $N$ for fast drift, large $N$ for slow drift) or exponentially weight recent transitions more heavily during sampling. If the environment changes abruptly (e.g., a regime switch), detecting the change and flushing the buffer is the cleanest solution — stale data from before the switch will actively mislead learning. This tension between data efficiency (larger buffer) and adaptability (smaller buffer) is a fundamental trade-off in non-stationary RL.
  2. The target network in DQN is updated every $C$ steps. If $C$ is very large, the target is very stable but potentially very wrong (stale). If $C = 1$, you're back to the unstable semi-gradient TD. How would you choose $C$ in practice? Is there a principled argument for a particular value?
    Choosing $C$ is largely empirical — the original DQN paper used $C \approx 1000$–$10000$ steps, chosen by sweep. A semi-principled heuristic: $C$ should be long enough that $Q$ has made meaningful progress between updates (otherwise the target network provides no additional stability), but short enough that the target hasn't drifted so far from truth that updates correct in the wrong direction.

    An alternative that sidesteps the discrete choice: Polyak (soft) target updates $\mathbf{w}_\text{target} \leftarrow \tau\,\mathbf{w} + (1-\tau)\,\mathbf{w}_\text{target}$ applied every step with $\tau \approx 0.005$. This is equivalent to a continuous, exponentially-weighted blend — more principled and removes the cliff-edge instability at $C=1$. Used in most modern deep RL (SAC, TD3, DDPG).
  3. State aggregation groups nearby states and assigns them the same value. What implicit assumption does this make about the value function? When does this assumption hold, and when does it fail badly?
    State aggregation assumes the value function is approximately constant within each group — i.e., the true $v_\pi$ varies slowly relative to the group boundaries.

    Holds well: tasks with smooth, monotone value functions (e.g., 1D random walk, where value increases linearly from left to right). Nearby states really do have similar values.

    Fails badly: (a) when the optimal policy changes action at a boundary that falls inside a group — the two halves of the group need different values but share one; (b) when the environment has irregular geometry, so "nearby by index" ≠ "nearby by dynamics"; (c) in high-dimensional spaces where coarse grouping is the only option but groups contain very diverse states. In these cases, the approximation error floor is large no matter how many episodes you run — a representation limitation, not a data limitation.
Common pitfalls
Mistakes worth knowing about in advance
  • Treating semi-gradient TD like supervised learning. In supervised learning, the labels are fixed. In semi-gradient TD, the TD target $R + \gamma\hat{v}(S',\mathbf{w})$ changes every time $\mathbf{w}$ changes — it is a moving target. This is not a minor detail: it fundamentally changes convergence behavior. Don't reason about stability using supervised learning intuitions.
  • Thinking function approximation always generalizes correctly. FA generalizes from visited to unvisited states based on shared features. If your feature design groups states that deserve different values into the same group, the approximation will be systematically wrong for some states no matter how much data you collect. Feature design (or representation learning) is often the bottleneck in practical RL systems.
  • Forgetting that VE is weighted by the state distribution. Minimizing $\overline{VE}(\mathbf{w})$ gives you the best approximation on average, weighted by how often states are visited. If the policy changes dramatically, the distribution $\mu$ changes, and a previously good $\mathbf{w}$ may become poor. This is why on-policy training and off-policy training produce different approximations.
  • Assuming DQN is theoretically principled. Experience replay and target networks are engineering heuristics that work empirically. They do not restore the theoretical convergence guarantees of tabular Q-learning. DQN can and does diverge on some problems — it is not a general solution to the deadly triad, just a practical mitigation.
Connects to
  • Deep Learning · Neural Networks — DQN uses standard deep learning components (convolutional layers for pixel input, batch updates via SGD/RMSProp, experience replay as a form of data shuffling). Understanding backpropagation, batch normalization, and optimizer choices is directly applicable here.
  • Optimization · Gradient Descent — Semi-gradient TD is a variant of SGD with a moving target. Understanding why SGD convergence requires Lipschitz gradients and convexity helps clarify why semi-gradient methods are so much harder to analyze than supervised learning optimization.
  • RL · Policy Gradient Methods (Lesson 9) — Policy gradients take a different approach: instead of approximating the value function and deriving a policy, directly parameterize and optimize the policy. Actor-critic methods combine both: a parameterized value function (critic) and a parameterized policy (actor), with the semi-gradient TD error used to update both.
