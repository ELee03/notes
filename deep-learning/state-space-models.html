10 · State Space Models
Classical SSM · HiPPO · S4 · Mamba · Selective Scan · SSM–Transformer Duality
Transformers are the dominant sequence model, but they have a fundamental bottleneck: self-attention is quadratic in sequence length. Processing a sequence of $S$ tokens requires $O(S^2)$ memory and $O(S^2 d)$ computation. Flash Attention reduces the constant factor, but the quadratic scaling remains. For genomics (sequences of millions of base pairs), audio (16 kHz for seconds = tens of thousands of timesteps), and long-context language modeling, quadratic attention becomes impractical. State space models (SSMs) are a class of sequence models derived from classical linear systems theory that achieve linear-time, constant-memory autoregressive inference. The S4 model (Gu et al. 2022) showed that SSMs could match transformers on long-range benchmarks; Mamba (Gu & Dao 2023) added input-dependent (selective) state spaces that outperform transformers on language modeling at comparable parameter counts. This lesson derives the SSM from first principles, explains the computational challenges, and develops the key innovations in S4 and Mamba.
The problem with quadratic attention
At sequence length $S$, the attention weight matrix $A \in \mathbb{R}^{S \times S}$ requires $O(S^2)$ memory and $O(S^2 d)$ compute. For $S = 1{,}000{,}000$ (human genome context), the matrix alone takes $10^{12} \times 2\text{ bytes} = 2\text{ TB}$. Flash Attention reduces the HBM footprint to $O(S)$ by tiling, but the underlying algorithm still performs $O(S^2)$ floating-point operations. The wall is algorithmic, not just engineering.
Recurrent models (LSTMs, GRUs) are $O(1)$ in memory and $O(S)$ in total compute for autoregressive generation. But they have a different problem: sequential computation. Each step depends on the previous hidden state, so you cannot parallelise the forward pass over sequence positions during training. This makes training LSTMs slow on modern GPU hardware, which thrives on parallel operations. The ideal sequence model would be: parallelisable for training (like transformers) and $O(1)$ memory for inference (like RNNs). State space models achieve both.
Classical linear state space model
A continuous-time linear state space model relates an input signal $u(t)$ to an output $y(t)$ through a hidden state $\mathbf{h}(t) \in \mathbb{R}^N$: $$\dot{\mathbf{h}}(t) = \mathbf{A}\mathbf{h}(t) + \mathbf{B}u(t)$$ $$y(t) = \mathbf{C}\mathbf{h}(t) + Du(t)$$ The matrices $\mathbf{A} \in \mathbb{R}^{N \times N}$, $\mathbf{B} \in \mathbb{R}^{N \times 1}$, $\mathbf{C} \in \mathbb{R}^{1 \times N}$, and $D \in \mathbb{R}$ are the system parameters. (In control theory, this is the standard SISO state-space form from DL6 prerequisite material; $D$ is the feedthrough term and is usually set to 0 in SSM sequence models.) The state $\mathbf{h}(t)$ compresses the entire history of the input $u(\tau)$ for $\tau \leq t$ — it is the "memory" of the system. The key question is: what structure of $\mathbf{A}$ makes this a useful sequence model?
Discretisation
Discretisation via the bilinear (Tustin) transform

Neural networks process discrete sequences $\{u_1, u_2, \ldots, u_S\}$, not continuous signals. We discretise the continuous SSM with step size $\Delta$ (the sampling period).

Zero-order hold (ZOH): Assume $u(t)$ is constant over each interval $[k\Delta, (k+1)\Delta)$. The exact discretisation is: $$\bar{\mathbf{A}} = e^{\mathbf{A}\Delta}, \quad \bar{\mathbf{B}} = (\mathbf{A}^{-1}(e^{\mathbf{A}\Delta} - I))\mathbf{B}$$ The matrix exponential $e^{\mathbf{A}\Delta}$ is expensive to compute for large $N$.

Bilinear (Tustin) approximation: Approximate $e^{\mathbf{A}\Delta} \approx (I - \frac{\Delta}{2}\mathbf{A})^{-1}(I + \frac{\Delta}{2}\mathbf{A})$: $$\bar{\mathbf{A}} = (I - \frac{\Delta}{2}\mathbf{A})^{-1}(I + \frac{\Delta}{2}\mathbf{A}), \quad \bar{\mathbf{B}} = (I - \frac{\Delta}{2}\mathbf{A})^{-1}\Delta\mathbf{B}$$ The discrete-time recurrence is then: $$\mathbf{h}_k = \bar{\mathbf{A}}\mathbf{h}_{k-1} + \bar{\mathbf{B}}u_k$$ $$y_k = \mathbf{C}\mathbf{h}_k$$

For inference, this is a linear RNN: at each step, multiply the state by $\bar{\mathbf{A}}$ and add $\bar{\mathbf{B}}u_k$. This is $O(N^2)$ per step (matrix-vector product) or $O(N)$ per step if $\bar{\mathbf{A}}$ is diagonal. For training, we need a smarter approach.

The global convolution view. Because the discrete SSM is a linear time-invariant (LTI) system, it can be expressed as a convolution. The output is: $$y_k = \sum_{j=0}^{k} \mathbf{C}\bar{\mathbf{A}}^{k-j}\bar{\mathbf{B}}\, u_j$$ Define the convolution kernel $\bar{\mathbf{K}} \in \mathbb{R}^S$: $$\bar{K}_i = \mathbf{C}\bar{\mathbf{A}}^i\bar{\mathbf{B}}, \quad i = 0, 1, \ldots, S-1$$ Then the full output sequence $\mathbf{y} = \bar{\mathbf{K}} * \mathbf{u}$, computable in $O(S \log S)$ using the fast Fourier transform (FFT). This gives the training parallelism we need: the entire sequence can be processed in one FFT convolution. For inference, we use the recurrent form (one step at a time, $O(1)$ memory). One model, two execution modes.
HiPPO — memorising history optimally
HiPPO: High-order Polynomial Projection Operators (Gu et al. 2020)

The matrix $\mathbf{A}$ determines which aspects of the past input the state $\mathbf{h}(t)$ retains. A random $\mathbf{A}$ has no reason to preserve long-range information. HiPPO provides a principled construction based on orthogonal polynomials.

Idea: At time $t$, maintain the best approximation of the history $u(\tau)$ for $\tau \leq t$ as a polynomial of degree $\leq N-1$ with respect to a measure $\omega_t$ on $(-\infty, t]$. As $t$ increases, update the polynomial coefficients incrementally.

LegT (Legendre, translated) measure: Uniform measure on $[t-\theta, t]$ (sliding window). The HiPPO matrix is: $$A_{nk} = -\begin{cases}(2n+1)^{1/2}(2k+1)^{1/2} & k < n \\ n+1 & k = n \\ 0 & k > n \end{cases}$$ $$B_n = (2n+1)^{1/2}$$ This lower-triangular structure ensures that $\mathbf{h}(t)$ encodes the $N$ Legendre polynomial coefficients of the windowed history. Higher-order coefficients capture fine-grained temporal structure; lower-order capture coarser trends.

LegS (Legendre, scaled) measure: Scale by $1/t$ (all history equally weighted). The HiPPO-LegS matrix is: $$A_{nk} = -\begin{cases}\sqrt{(2n+1)(2k+1)} & k < n \\ n+1 & k = n \\ 0 & k > n \end{cases}, \quad B_n = \sqrt{2n+1}$$ This variant is the most commonly used in S4.

Initialising $\mathbf{A}$ with the HiPPO matrix — rather than randomly — gives the SSM a strong inductive bias toward remembering long-range history. This was the key empirical finding of the HiPPO paper: randomly-initialised RNNs forget history quickly; HiPPO-initialised models remember it polynomially.

S4 — Structured State Space Sequence Model
S4 computational challenge and solution (Gu et al. 2022)

The problem with the convolution view: Computing the kernel $\bar{K}_i = \mathbf{C}\bar{\mathbf{A}}^i\bar{\mathbf{B}}$ for $i = 0, \ldots, S-1$ requires $S$ matrix-vector products with $\bar{\mathbf{A}} \in \mathbb{R}^{N \times N}$ — a total of $O(SN^2)$ operations. For $N = 64$ and $S = 16{,}000$: $65{,}536 \times 16{,}000 \approx 10^9$ operations, just for the kernel. Then the FFT convolution costs $O(SN\log S)$. The bottleneck is computing the kernel.

S4 idea: If $\mathbf{A}$ is diagonal (or normal — unitarily diagonalisable), $\mathbf{A} = \text{diag}(\lambda_1, \ldots, \lambda_N)$, then $\bar{K}_i = \mathbf{C}\bar{\mathbf{A}}^i\bar{\mathbf{B}} = \sum_n C_n \bar{A}_n^i \bar{B}_n$ — a sum of $N$ geometric sequences. This is the formula for a (generalised) Cauchy/Vandermonde matrix-vector product, computable in $O(N + S)$ via Horner's method or $O((N+S)\log^2 N)$ via dedicated algorithms.

DPLR (Diagonal Plus Low-Rank) structure: The HiPPO-LegS matrix is not diagonal but can be written as $\mathbf{A} = \mathbf{\Lambda} - \mathbf{P}\mathbf{Q}^*$ where $\mathbf{\Lambda}$ is diagonal and $\mathbf{P}, \mathbf{Q} \in \mathbb{C}^{N \times r}$ is a rank-$r$ correction. S4 uses this DPLR structure: the diagonal part is handled efficiently, and the rank-1 correction uses the Woodbury matrix identity. Total kernel computation: $O(N + S\log S)$ — independent of $N$ in the dominant term for long sequences.

What S4 learns: $\mathbf{A}$ is initialised to HiPPO-LegS but the diagonal part $\mathbf{\Lambda}$ (complex numbers) is trainable, as are $\mathbf{B}$, $\mathbf{C}$, and $\Delta$. The model learns per-sequence-position input–output mappings; stacking multiple SSM layers with non-linearities and skip connections gives the full S4 network.

Diagonal-S4 / DSS: Later work showed that simply initialising a diagonal $\mathbf{A}$ with HiPPO-inspired eigenvalues (complex-valued, on or near the unit circle in discrete time) matches S4 performance while being much simpler to implement. The key insight: the DPLR structure was largely solving the problem created by the dense HiPPO matrix; if you diagonalise from the start, the whole apparatus is unnecessary.

Mamba — selective state spaces
S4 and all prior SSMs are linear time-invariant (LTI): the matrices $\mathbf{A}$, $\mathbf{B}$, $\mathbf{C}$ are fixed (learned once) and applied uniformly to every position in every sequence. This is exactly the property that enables the global convolution view and $O(S \log S)$ training. But it is also a fundamental limitation. A fixed $\mathbf{A}$ means the model transitions the state the same way regardless of the content of the current input — the state cannot selectively remember or forget based on what it has seen. Contrast this with the attention mechanism, which computes input-dependent weights over all previous positions. Mamba (Gu & Dao 2023) introduces selective state spaces: $\mathbf{B}$, $\mathbf{C}$, and $\Delta$ are functions of the input $u_k$.
Selective SSM — the Mamba mechanism

In Mamba, the parameters that vary per input token are: $$\mathbf{B}_k = \text{Linear}(u_k) \in \mathbb{R}^N$$ $$\mathbf{C}_k = \text{Linear}(u_k) \in \mathbb{R}^N$$ $$\Delta_k = \text{softplus}(\text{Linear}(u_k)) \in \mathbb{R}^{d_\text{inner}}$$ (broadcast to $N$ via a learned projection) while $\mathbf{A}$ remains fixed (shared, initialised as a diagonal negative-real matrix so that $e^{\mathbf{A}\Delta}$ is stable for any $\Delta > 0$). The discrete recurrence for each channel: $$h_k = \bar{A}_k h_{k-1} + \bar{B}_k u_k, \quad \bar{A}_k = e^{\Delta_k A}, \quad \bar{B}_k = (\Delta_k A)^{-1}(e^{\Delta_k A}-1)\Delta_k B_k$$ $$y_k = C_k h_k$$

Selective forgetting via $\Delta_k$: When $\Delta_k \to 0$ (small step), $\bar{A}_k \to 1$ and $\bar{B}_k \to 0$ — the state is preserved unchanged, ignoring the new input ($u_k$ has no effect on $h_k$). When $\Delta_k \to \infty$ (large step), $\bar{A}_k \to 0$ and $\bar{B}_k \to B_k$ — the state resets to the new input, forgetting the past. By making $\Delta_k$ input-dependent, the model learns to selectively decide, token by token, whether to absorb new information or preserve the existing state. This is the discrete analogue of the forget/input gate mechanism in LSTMs, but derived from a principled continuous-time dynamical systems framework.

Loss of LTI ⟹ Loss of global convolution. Since $\mathbf{B}$, $\mathbf{C}$, and $\Delta$ vary with position, the system is no longer time-invariant, and the convolution trick no longer applies. Mamba cannot be trained as a global FFT convolution. The training algorithm instead uses a parallel scan (also called prefix scan or scan operation): a classical parallel algorithms primitive that computes all prefix products $h_1, h_2, \ldots, h_S$ in $O(\log S)$ parallel steps (vs. $O(S)$ sequential steps). On GPU with thousands of cores, the parallel scan achieves near-maximal hardware utilisation. The parallel scan over the recurrence is implemented as a custom CUDA kernel (hardware-aware). Training speed is comparable to Transformer training despite the non-LTI recurrence.

The Mamba block. A Mamba layer is more than just the SSM. Each Mamba block applies: (1) layer normalisation, (2) two parallel projections of the input to dimension $d_\text{inner} = 2d$, (3) one branch goes through the selective SSM; the other goes through a SiLU-activated gate, (4) the branches are multiplied (gating), (5) projected back to $d$, and added to the residual. This gated structure is borrowed from gated MLPs (Dauphin et al. 2017, "Gating" / GLU). The full Mamba model stacks these blocks with alternating layer normalisation, similar to a transformer stack without attention.
Mamba-2 and the state space duality
Mamba-2 (Dao & Gu 2024) unifies selective SSMs and transformers under the Structured State Space Duality (SSD): a restricted class of selective SSMs (where the state matrix $\mathbf{A}$ is a scalar times identity, equivalent to uniform decay) can be computed equivalently as a masked attention over a structured attention matrix. Specifically, the SSD layer with scalar $\mathbf{A} = a_k I$ (one scalar per token) corresponds to attention with an attention bias equal to $\sum_{j=k}^{t-1} \log a_j$ — a "causal decay" bias. This bidirectional view means: (1) SSMs can inherit attention-optimisation tricks (block decomposition, tensor cores, Flash Attention variants). (2) Transformers can be enhanced with SSM-style gating and decay. Mamba-2 trains 2–8× faster than Mamba-1 due to better hardware utilisation via the SSD matrix form, while matching or exceeding Mamba-1's language modelling performance.

Worked examples
Example 1 — Discretising a 1D SSM by ZOH

Scalar SSM: $\dot{h}(t) = -0.5 h(t) + u(t)$, $y(t) = h(t)$. So $A = -0.5$, $B = 1$, $C = 1$, $D = 0$. Discretise with $\Delta = 0.1$ using exact ZOH. Given $h_0 = 0$ and inputs $u_1 = 2$, $u_2 = 0$, compute $h_1$ and $h_2$.

ZOH discretisation: $\bar{A} = e^{A\Delta} = e^{-0.5 \times 0.1} = e^{-0.05} \approx 0.9512$. $\bar{B} = A^{-1}(e^{A\Delta}-1)B = \frac{1}{-0.5}(0.9512 - 1)(1) = \frac{-0.0488}{-0.5} = 0.0976$.

Step 1: $h_1 = \bar{A} h_0 + \bar{B} u_1 = 0.9512 \times 0 + 0.0976 \times 2 = 0.1952$ $y_1 = C h_1 = 0.1952$

Step 2: $h_2 = \bar{A} h_1 + \bar{B} u_2 = 0.9512 \times 0.1952 + 0.0976 \times 0 = 0.1857$ $y_2 = C h_2 = 0.1857$

The state decays exponentially when no input is present (time constant = $1/0.5 = 2$ continuous-time units, or $2/0.1 = 20$ discrete steps to lose $e^{-1} \approx 63\%$ of the state). This is the fundamental "memory" of the SSM — controlled by $A$. In S4/Mamba, the learnable $\Delta$ adjusts how quickly the state responds: small $\Delta$ = slow response (long memory), large $\Delta$ = fast response (short memory).

Example 2 — The convolution kernel and long-range dependency

A scalar SSM has $\bar{A} = 0.99$, $\bar{B} = 1$, $C = 1$. Compute the convolution kernel $\bar{K}_i$ for $i = 0, 1, \ldots, 4$. What is $\bar{K}_{100}$? What does this tell you about the model's long-range memory?

$\bar{K}_i = C \bar{A}^i \bar{B} = 1 \times 0.99^i \times 1 = 0.99^i$

$\bar{K}_0 = 1$, $\bar{K}_1 = 0.99$, $\bar{K}_2 = 0.980$, $\bar{K}_3 = 0.970$, $\bar{K}_4 = 0.961$.

$\bar{K}_{100} = 0.99^{100} = e^{100\ln 0.99} = e^{-1.005} \approx 0.366$.

After 100 steps, the output still has a 37% contribution from that position's input. Compare to an LSTM: if the forget gate outputs $f = 0.99$ constantly, the cell state also decays at rate $0.99^k$. The difference is that the SSM's $\bar{A} = 0.99$ is a global parameter fixed after training, so every input at every position contributes exactly $0.99^{k-j}$ to output $k$. The LSTM's forget gate is input-dependent, so it can choose to forget or remember based on content. This is the core limitation of LTI SSMs (S4) that Mamba addresses with selective state spaces: the decay rate is fixed for S4, but input-dependent for Mamba.

Example 3 — Mamba's selective forgetting

A Mamba model is processing a sequence. At position $k=5$, the network computes $\Delta_5 = 0.01$ (very small, near-zero step). At position $k=10$, it computes $\Delta_{10} = 100$ (very large step). Explain intuitively what happens to the hidden state at each of these positions.

At position $k=5$ ($\Delta_5 = 0.01$): $\bar{A}_5 = e^{A \times 0.01} \approx e^{-0.01} \approx 0.99 \approx 1$ (state barely changes). $\bar{B}_5 \approx 0$ (very small, since $\bar{B} \approx (A\Delta)^{-1}(e^{A\Delta}-1)\Delta B \approx \Delta B \approx 0.01B$ for small $\Delta$). Effect: $h_5 \approx 1 \times h_4 + 0 \times u_5 \approx h_4$. The state is preserved, and the new input $u_5$ is largely ignored. The model is saying: "this token is not important; carry forward what I had."

At position $k=10$ ($\Delta_{10} = 100$): $\bar{A}_{10} = e^{A \times 100} \approx 0$ (very close to zero for $A < 0$). $\bar{B}_{10} = (A\Delta)^{-1}(e^{A\Delta}-1)\Delta B \approx -A^{-1}B$ (saturates as $\Delta \to \infty$). Effect: $h_{10} \approx 0 \times h_9 + \bar{B}_{10} u_{10}$. The state is essentially reset to the new input, forgetting all previous history. The model is saying: "this token is critical; discard the past and start fresh from here."

This is input-dependent selectivity: the model learns, via gradient descent on $\Delta$, which positions in a sequence deserve to reset the state vs. which should be ignored. For language modeling, a sentence boundary or topic shift might trigger $\Delta \to \infty$ (reset), while a function word like "the" might trigger $\Delta \to 0$ (ignore).

Food for thought
  • S4 achieves parallelisable training via the global convolution (FFT) trick, but Mamba cannot use this because its parameters are input-dependent (non-LTI). Mamba instead uses a parallel scan. Why is a parallel scan fundamentally different from the FFT convolution? What mathematical property does LTI give you that is lost with selective state spaces?
    The FFT convolution works because a LTI system's output is the convolution of its impulse response with the input: $y = K * u$. Convolution is defined in terms of a fixed kernel $K$ applied to all positions uniformly — it's the mathematical operation that exploits time-invariance. The DFT transforms a convolution to a pointwise product in frequency space, enabling $O(S\log S)$ computation. In Mamba, the "kernel" at each position is different (because $B_k$, $C_k$, $\Delta_k$ depend on $u_k$). This is a state-varying linear system — mathematically, it computes a product of matrices that vary with the input: $h_k = \prod_{j=1}^k \bar{A}_j \cdot h_0 + \ldots$ This is no longer a convolution (which requires identical kernels) — it's an associative prefix product of varying matrices. The parallel scan works because matrix (or scalar) multiplication is associative, which is all that's needed: you can compute prefix products in a tree structure with $O(\log S)$ parallel depth. LTI gives you convolution (FFT-able, very fast for large $S$). Selectivity breaks LTI, giving you a prefix product (parallel-scan-able, still efficiently parallelisable but without the FFT's multiplicative speedup from the DFT).
  • Transformers can attend directly to any position in the context with a single operation (the attention mechanism retrieves a specific key-value pair exactly). Mamba compresses the entire history into a fixed-size state vector $\mathbf{h} \in \mathbb{R}^N$. What tasks would you expect Mamba to struggle with relative to transformers? What tasks might it excel at?
    Mamba struggles with: Tasks requiring exact retrieval from specific past positions. If a document contains a specific fact at position 5000 that must be verbatim retrieved at position 10000, Mamba must have compressed that fact into its $N$-dimensional state at every intermediate step. The compression is lossy; transformers with explicit attention can retrieve it exactly (assuming the key-value cache holds the relevant token). Tasks with arbitrary random-access lookups (e.g., "what was word 3 of this document?") or in-context few-shot tasks that require the model to scan the context for specific examples. Benchmarks like MQAR (Multi-Query Associative Recall) were specifically designed to test this and Mamba-1 struggled there; Mamba-2 improved via the SSD formulation. Mamba excels at: Tasks where the relevant signal is local or recent — language modeling, audio waveforms, genomic sequences where local structure dominates. Tasks where constant-memory inference matters: streaming audio, real-time sensor processing, auto-regressive generation with very long contexts (Mamba's per-step inference cost is $O(Nd)$ regardless of the context length, whereas a transformer's cost grows as $O(Sd)$ where $S$ is the total context). Domains where quadratic attention is prohibitive: million-base-pair genomics, long-form audio. The current (2025) practical answer: hybrid architectures (e.g., Jamba — interleaving Mamba and transformer layers) outperform either alone on most tasks, suggesting complementary strengths rather than a clear winner.
  • The HiPPO initialisation for $\mathbf{A}$ is based on polynomial approximation theory — it ensures the state $\mathbf{h}(t)$ maintains the optimal polynomial approximation of the input history. Why is this a better initialisation than random Gaussian or zeros, even if the matrix is later fine-tuned?
    All neural network parameters are fine-tuned from an initial value. The question is whether the gradient signal during fine-tuning is strong enough to reach a good solution from a given initialisation. For SSMs, the gradient through a sequence of $S$ recurrent steps is a product of $S$ Jacobians — exactly the exploding/vanishing gradient problem. A random $\mathbf{A}$ has eigenvalues scattered arbitrarily: eigenvalues with magnitude $> 1$ explode, $< 1$ vanish, and the network converges slowly or not at all on long-range tasks. HiPPO constructs $\mathbf{A}$ to have a specific eigenstructure: (1) Eigenvalues are designed so that the polynomial coefficients update stably — neither exploding nor vanishing — as the input evolves. (2) The orthogonal polynomial basis is an optimal compression of the history: it minimises the approximation error for any function that can be represented with $N$ coefficients. (3) This means the gradient signal for long-range dependencies is non-vanishing from the start. Empirically, S4 models trained with HiPPO initialisation converge much faster and reach better performance on long-range benchmarks than random-init models, even with the same final fine-tuning. Initialisation matters because it sets the geometry of the loss landscape the optimiser must navigate — a good initialisation starts in a basin of attraction for a good solution.
Connects to
LSTMs & GRUs (DL4) — LSTMs and Mamba are both gated recurrences, both solvable with a sequential recurrence at inference. Mamba's selective $\Delta$ plays the role of the LSTM's forget gate, but derives from a principled SSM framework rather than an ad hoc gating intuition. The parallel scan that trains Mamba efficiently has no analogue for LSTMs (whose gates are multiplicative non-linear functions, not associative products).
The Transformer (DL6) — S4 and Mamba were designed as alternatives to transformer self-attention. The state space duality (Mamba-2) reveals a formal equivalence between a restricted class of SSMs and structured linear attention, suggesting these architectures are not as different as they appear. Hybrid architectures (Jamba, Zamba) mix attention and SSM layers.
Control Theory — The SSM equations $\dot{h} = Ah + Bu$, $y = Ch + Du$ are exactly the continuous-time state space form used in classical control theory. The discretisation methods (ZOH, bilinear/Tustin transform) are standard in digital control. The HiPPO framework draws on approximation theory and operator semigroups. The two fields have been largely separate, but SSM sequence models have brought them into direct contact.
