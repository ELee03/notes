7 · ADC & DAC
SAR · Quantization · Antialiasing · EFR32 ADC · DMA Acquisition Pipeline
The ADC (Analog-to-Digital Converter) is where the physical world enters the microcontroller. Pressure, temperature, bioelectric potential, light intensity — every real-world measurand arrives at an MCU pin as a continuous voltage. The ADC's job is to map that voltage onto a binary integer at a fixed instant in time, producing a discrete sample. Understanding how that mapping works — and how it can go wrong — is essential for any measurement system.
Quantization and resolution
An N-bit ADC divides the input range [0, Vref] into 2N equal steps. The smallest distinguishable voltage change is one LSB (Least Significant Bit):
$$\text{LSB} = \frac{V_\text{ref}}{2^N}, \qquad V_\text{in} \approx \text{code} \times \text{LSB}$$
Resolution Codes LSB at Vref=2.5 V Typical use
8-bit2569.8 mVTemperature, rough sensing
10-bit10242.4 mVGeneral-purpose sensing
12-bit4096610 µVEFR32 default — biosignals, precision sensors
16-bit6553638 µVAudio, precision measurement (delta-sigma)
24-bit16.7 M150 nVInstrumentation, weighing scales
Quantization error is unavoidable — every conversion rounds the true voltage to the nearest code, introducing ±½ LSB of uncertainty. At 12 bits with a 2.5 V reference this is ±305 µV. This is a fundamental noise floor that cannot be reduced without increasing resolution or narrowing the input range. In practice, other error sources (input noise, reference noise, offset, gain error) often dominate.
SAR ADC architecture
Nearly every MCU-integrated ADC uses the Successive Approximation Register (SAR) architecture. A SAR ADC performs a binary search to find the digital code closest to the sampled voltage in exactly N clock cycles.
4-bit SAR ADC converting ≈10.6: each clock cycle tests one bit from MSB down. 4 bits → 4 cycles → code 1010 = 10.
The internal DAC starts at mid-scale (MSB = 1). If its output exceeds Vin, the MSB is cleared; if not, it stays set. Then the next bit is set and compared. In this way each comparison eliminates half the remaining uncertainty — a binary search. After N comparisons the result is exact to within ±½ LSB. An N-bit SAR ADC always takes exactly N clock cycles per conversion, regardless of the input, which makes timing predictable.
The SAR architecture dominates MCU-integrated ADCs because it is power-efficient, fast enough for most sensor applications, and area-efficient. The main alternatives are delta-sigma (higher resolution through oversampling and noise shaping, slower throughput, used in audio codecs and precision instruments) and flash ADCs (one comparator per code, 2N−1 comparators for N bits, extremely fast but impractical above 8–10 bits due to size and power).
Key ADC parameters
Four parameters define how an ADC performs in a real system beyond the ideal binary search model:
Reference voltage
The ADC codes the input voltage relative to Vref. Noise or drift in the reference voltage directly adds to measurement error — a 1 mV shift in a 2.5 V reference introduces a 0.04% full-scale error, which at 12 bits is nearly 2 LSB. For precision work, use an external precision reference (e.g., a band-gap reference IC with 10 ppm/°C drift) rather than the internal reference. For general-purpose sensing, the EFR32 internal 2.5 V reference (typ. 1% initial accuracy) is adequate. Never reference an ADC to VDD unless you have no choice; VDD carries load transients from the radio and digital logic.
Acquisition (sampling) time
Before the conversion begins, the ADC must charge its internal sampling capacitor (typically 2–10 pF) to the input voltage. If the source impedance is high (e.g., a biosensor electrode through a 1 MΩ input resistor), the RC time constant may be long. The acquisition time register must be set long enough for the capacitor to settle to within ½ LSB — otherwise the conversion result reflects a partially-charged capacitor, not the true input.
tacq ≥ (Rsource + Rswitch) × Csample × ln(2N+1)
At 12 bits, ln(213) ≈ 9. With a 10 kΩ source and 5 pF sampling cap, tacq ≥ 10k × 5p × 9 = 450 ns minimum. Add margin for parasitic capacitance: use 1 µs or more.
Sample rate and throughput
Total conversion time = acquisition time + N × (1 / f_ADC). The EFR32 ADC can run at up to 13 MHz with a maximum sample rate of ~1 Msps at 12-bit resolution. For most biosignal applications (ECG, EMG: 0.05–3 kHz bandwidth) a few kHz sample rate is more than adequate, and you can use a longer acquisition time to tolerate higher source impedance.
ENOB — Effective Number of Bits
A real ADC is noisier than the ideal. $\text{ENOB} = (\text{SINAD} - 1.76) / 6.02$, where SINAD is the signal-to-(noise + distortion) ratio in dB. A "12-bit" ADC might achieve 10–11 ENOB in practice — the lower bits are dominated by thermal noise. Improving ENOB requires better supply decoupling, a quieter reference, and sometimes hardware averaging (oversampling: take $M$ samples and average → gain $\tfrac{1}{2}\log_2 M$ bits). The EFR32 ADC supports hardware oversampling up to 256× for up to 16 effective bits.
Aliasing and antialiasing filters
The Nyquist theorem requires sampling at at least twice the highest signal frequency: fsample ≥ 2 × fsignal. Any frequency component above fsample/2 (the Nyquist frequency) that reaches the ADC will alias — it will appear as a spurious component at a lower frequency in the sampled signal. Aliasing is irreversible; once aliased, you cannot distinguish the alias from a real signal component.
A 350 Hz component sampled at 500 Hz aliases to fs−350 = 150 Hz — indistinguishable from a real 150 Hz signal. The antialiasing (AA) low-pass filter attenuates components above fs/2 before they reach the ADC.
The antialiasing filter is an analog low-pass filter placed before the ADC input. A first-order RC filter is often sufficient: choose R and C so the −3 dB cutoff is at or below fsample/2. For a 1 kHz sample rate, cutoff ≤ 500 Hz; a 10 kΩ resistor and 33 nF capacitor gives fc = 1/(2π × 10k × 33n) ≈ 482 Hz. The RC filter also reduces high-frequency noise from nearby digital switching.
RC filter doubles as a source impedance buffer issue. The series resistor R adds to the source impedance. Ensure the ADC acquisition time is long enough for the total (source || R) to charge the sampling capacitor. For biosensor applications, the dominant source impedance is usually the electrode impedance (kΩ–MΩ), making an instrumentation amplifier front-end essential before the ADC.
EFR32 ADC API
EFR32 Series 1 uses the ADC0 peripheral with EMLIB's em_adc.h driver. Series 2 replaces it with IADC0 (Improved ADC) and em_iadc.h. The concepts are identical; the API differs in structure names and channel selection syntax. 
#include "em_adc.h"
#include "em_cmu.h"

// ── ADC0 initialization (Series 1 — em_adc.h) ─────────────────

void adc_init(void) {
    CMU_ClockEnable(cmuClock_ADC0, true);

    // Top-level ADC init: clock prescaler and timebase
    // ADC_PrescaleCalc targets a given ADC clock frequency (≤13 MHz)
    ADC_Init_TypeDef init = ADC_INIT_DEFAULT;
    init.prescale = ADC_PrescaleCalc(7000000, 0);  // 7 MHz ADC clock
    init.timebase = ADC_TimebaseCalc(0);            // min timebase for warm-up
    ADC_Init(ADC0, &init);

    // Single-ended conversion on APORT channel
    ADC_InitSingle_TypeDef single = ADC_INITSINGLE_DEFAULT;
    single.reference = adcRef2V5;                // internal 2.5 V reference
    single.resolution = adcRes12Bit;             // 12-bit result
    single.posSel = adcPosSelAPORT1XCH0;         // input on APORT1X CH0 pin
    single.negSel = adcNegSelVSS;               // single-ended vs GND
    single.acqTime = adcAcqTime16;               // 16 ADC clocks ≈ 2.3 µs
    single.fifoOverwrite = true;
    ADC_InitSingle(ADC0, &single);
}

// Blocking single conversion (polling)
uint32_t adc_read_single(void) {
    ADC_Start(ADC0, adcStartSingle);
    while (!(ADC0->IF & ADC_IF_SINGLE)) { }
    return ADC_DataSingleGet(ADC0);  // clears the IF flag
}

// Convert ADC code to millivolts (V_ref = 2500 mV, 12-bit)
uint32_t code_to_mv(uint32_t code) {
    return (code * 2500ul) / 4096ul;
}
Scan mode for multiple channels
Scan mode sequences through multiple channels in one triggered conversion. Each channel produces one 16-bit result word (with channel ID in the upper bits) stored to the SCANDATAX register. This is what you feed to DMA. 
#define N_CH  4   // scan 4 channels per trigger

void adc_scan_init(void) {
    ADC_InitScan_TypeDef scan = ADC_INITSCAN_DEFAULT;
    scan.reference  = adcRef2V5;
    scan.resolution = adcRes12Bit;
    scan.acqTime    = adcAcqTime16;

    // Build channel mask: APORT1X channels 0–3
    ADC_ScanMask_TypeDef mask = { { 0 } };
    ADC_ScanSingleEndedInputAdd(&mask, adcScanInputGroup0,
                                adcPosSelAPORT1XCH0);
    ADC_ScanSingleEndedInputAdd(&mask, adcScanInputGroup1,
                                adcPosSelAPORT1XCH2);
    ADC_ScanSingleEndedInputAdd(&mask, adcScanInputGroup2,
                                adcPosSelAPORT1XCH4);
    ADC_ScanSingleEndedInputAdd(&mask, adcScanInputGroup3,
                                adcPosSelAPORT1XCH6);

    scan.scanDmaEm2Wu = true;  // ADC can wake from EM2 and signal DMA
    ADC_InitScan(ADC0, &scan, &mask);
}

// Start one scan manually; or use PRS trigger for regular rate
void adc_trigger_scan(void) {
    ADC_Start(ADC0, adcStartScan);
}
DMA acquisition pipeline
For continuous biosignal acquisition — ECG, EMG, accelerometry — you want a fixed sample rate with zero CPU involvement per sample. The three building blocks from earlier lessons combine into a complete hardware pipeline:
The PRS (Peripheral Reflex System) is a hardware crossbar that routes signals between peripherals without CPU involvement. Configure LETIMER to output an underflow pulse on a PRS channel, then configure ADC0 to use that PRS channel as its scan trigger. The ADC starts a scan every LETIMER period — no software involvement. 
#include "em_prs.h"
#include "em_letimer.h"

// 1. Route LETIMER0 UF (underflow) to PRS channel 0
PRS_SourceSignalSet(0,                        // PRS channel 0
                    PRS_CH_CTRL_SOURCESEL_LETIMER0,
                    PRS_CH_CTRL_SIGSEL_LETIMER0UF,
                    prsEdgePos);

// 2. Configure ADC0 scan to start on PRS ch0
// In ADC_InitScan_TypeDef:
scan.prsEnable = true;
scan.prsSel    = adcPRSSELCh0;

// 3. DMA: signal = ldmaPeripheralSignal_ADC0_SCAN
//    Source = &ADC0->SCANDATAX (16-bit result + channel ID)
//    Descriptor: P2M_HALFWORD, HALF count, LINKREL ping-pong

// 4. LETIMER configured for 1 ms period (1 kHz)
// → ADC scans 4 channels every 1 ms
// → DMA moves each 8-byte frame to buffer
// → CPU wakes every 256 ms to process 256 frames of 4-channel data
SCANDATAX vs SCANDATA. ADC0→SCANDATAX contains both the 16-bit result and the 4-bit channel ID in bits [19:16]. Reading SCANDATAX after each DMA beat lets you demultiplex channels in the buffer. ADC0→SCANDATA contains only the result and can be used when channels are always scanned in a fixed order (simpler but loses channel context if any scan is missed).
DAC and PWM-as-DAC
A DAC (Digital-to-Analog Converter) is the inverse of the ADC: it turns a digital code into a proportional voltage. Some EFR32 variants include a VDAC0 (12-bit, up to 500 ksps), but many BLE-focused EFR32 variants (including BGM13/BGM12 BLE modules) do not include a VDAC.
PWM + low-pass filter as a DAC is a universal fallback. A PWM signal with duty cycle D produces an average voltage of D × VDD. Pass the PWM through an RC low-pass filter and the result is a DC voltage proportional to duty cycle. Resolution is determined by the PWM counter TOP value: a 12-bit TOP gives 12-bit DAC resolution. Settling time is limited by the RC time constant (typically 1–10 ms for audio-rate updates). 
// PWM DAC: TIMER0 CC0 → R=10kΩ, C=100nF → f_c ≈ 159 Hz output filter
// Set output voltage to ~1.0 V on a 3.3 V supply:
// D = 1.0 / 3.3 = 0.303 → CC = 0.303 × (TOP + 1)

void dac_set_voltage_mv(uint32_t mv) {
    uint32_t top = TIMER_TopGet(TIMER0);
    // CC = mv / VDD_mV * (top + 1)
    uint32_t cc = (mv * (top + 1)) / 3300ul;
    TIMER_CompareSet(TIMER0, 0, cc);
}

// For where VDAC0 exists (e.g., EFR32FG12):
// VDAC_Init_TypeDef dacInit = VDAC_INIT_DEFAULT;
// VDAC_Init(VDAC0, &dacInit);
// VDAC_ChannelOutputSet(VDAC0, 0, code);  // 12-bit code → voltage
For audio-rate output (e.g., driving a speaker with synthesized waveforms), the PWM frequency should be much higher than the filter cutoff to keep ripple low — 100 kHz PWM with a 1 kHz filter cutoff gives 40 dB of ripple attenuation. For static or slowly-changing voltages (bias generation, sensor excitation), even a 10 kHz PWM is fine.
