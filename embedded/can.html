11 · CAN Bus
Differential Signalling · Arbitration · Frame Format · Bit Stuffing · Error Confinement
CAN — Controller Area Network — was designed in the 1980s for automotive wiring harnesses and is now the dominant embedded bus in vehicles, industrial machinery, and medical devices. Its defining property is robustness: CAN survives short circuits on the bus, ground offsets between nodes, and faulty nodes that continuously transmit errors. A misbehaving node is automatically removed from the bus, leaving all other nodes unaffected.
CAN is a multi-master broadcast bus. Any node can transmit at any time; if two nodes transmit simultaneously, a deterministic arbitration mechanism decides which message wins — without destroying either transmission. The winning node continues uninterrupted; the losing node retries automatically. Priority is encoded directly in the message ID: a lower ID number always wins arbitration.
Differential signalling (CAN-H / CAN-L)
CAN uses a two-wire differential bus. Rather than signalling logic levels relative to ground, a CAN transceiver drives a voltage difference between two lines: CAN-H (high) and CAN-L (low). The receiver responds to the differential voltage Vdiff = VCAN-H − VCAN-L, not to either line alone.
Recessive: both lines float to 2.5 V; Vdiff ≈ 0 V. Dominant: transceiver drives CAN-H high and CAN-L low; Vdiff ≈ 2 V. The differential signal rejects common-mode noise that would affect a single-ended bus.
The advantage of differential signalling is noise rejection. Any interference (motor switching, ignition noise, ground bounce) couples equally onto both CAN-H and CAN-L as common-mode noise, which the differential receiver subtracts out. This is why CAN is reliable in the harshest electrical environments — the same noise that would corrupt a single-ended UART or I2C signal has almost no effect on a differential bus.
Dominant and recessive bits
CAN bits are not called "0" and "1" — they are called dominant (logic 0) and recessive (logic 1). This distinction is crucial:
  • Dominant (0) — the transceiver actively drives CAN-H HIGH and CAN-L LOW. Dominant overwrites recessive: if any node drives dominant, the bus is dominant, regardless of what other nodes are doing.
  • Recessive (1) — the transceiver is passive; both lines float to 2.5 V via the termination resistors. Recessive is only seen if no node is driving dominant simultaneously.
Think of it like an open-drain wired-AND: if any node pulls LOW (dominant), the bus is LOW regardless. The bus is HIGH (recessive) only when everyone releases it. This is the physical basis of non-destructive arbitration.
Non-destructive arbitration
When two or more nodes begin transmitting simultaneously, they both start by transmitting their message ID, one bit at a time, MSB first. After each bit, every transmitting node reads back the bus to see what it actually sees.
  • If a node transmitted a recessive bit (1) but reads back dominant (0), it knows another node with a lower-numbered (higher-priority) ID is also transmitting. It immediately stops transmitting and switches to receiver mode. No bytes were lost — the bus looks like a normal transmission.
  • The winning node (which transmitted all dominant bits where others backed off) never even knew there was a conflict. It continues transmitting its complete frame without interruption.
  • The losing node waits for the bus to go idle, then retransmits automatically.
Node A (ID 0x16 = 0b0010110) vs Node B (ID 0x19 = 0b0011001). Both nodes transmit together until bit 3: A drives dominant, B drives recessive. The bus shows dominant. B reads dominant when it expected recessive — it stops and lets A win. A's lower ID = higher priority.
Frame format
CAN defines several frame types. The most important is the Data Frame, which carries up to 8 bytes of payload. The CAN 2.0A standard uses 11-bit identifiers; CAN 2.0B (extended) uses 29-bit identifiers by adding an 18-bit extension field.
Field Bits Purpose
SOF 1 Start of Frame — dominant bit that synchronises all nodes
Identifier 11 (or 29) Message ID; lower number = higher priority. Transmitted MSB first during arbitration.
RTR 1 Remote Transmission Request. Dominant = data frame; recessive = requests data from a node
IDE 1 Identifier Extension. Dominant = standard (11-bit); recessive = extended (29-bit)
DLC 4 Data Length Code — number of data bytes (0–8)
Data 0–64 0–8 bytes of payload
CRC 15 + 1 15-bit CRC over all preceding fields; followed by a recessive delimiter bit
ACK slot 1 + 1 Transmitter sends recessive; any receiver that got a valid CRC overwrites it with dominant
EOF 7 7 recessive bits marking end of frame
The ACK mechanism is a key difference from other protocols. The transmitter does not know who, if anyone, received its message. All it knows is whether any node on the bus acknowledged it. If the ACK slot stays recessive (nobody acknowledged), the transmitter raises an error. This gives CAN a broadcast delivery guarantee: if the CRC is valid, at least one receiver confirmed receipt.
Bit stuffing
CAN has no separate clock line. Nodes synchronise on bit edges — specifically, on transitions from recessive to dominant or dominant to recessive. Long runs of identical bits would give no edges to synchronise on. To prevent this, CAN uses bit stuffing: after every five consecutive bits of the same polarity, the transmitter inserts one bit of the opposite polarity. Receivers remove these stuff bits automatically.
Rule: after 5 consecutive bits of same value, insert 1 bit of opposite value
Example: 111110 → 1111100   (0 is the stuff bit; receiver discards it)
Example: 000001 → 00000 + 1 + 1 = 000001 with transparent stuff at position 6
Bit stuffing inflates the frame slightly — in the worst case a data field has 20% overhead. More importantly, a stuff error — seeing six consecutive bits of the same polarity — is always a protocol error. This is one of the five error detection mechanisms in CAN (the others are CRC check, frame format check, ACK check, and bit monitoring).
Error detection and error confinement
CAN's robustness comes from its layered error system. Every node maintains two internal counters: the Transmit Error Counter (TEC) and the Receive Error Counter (REC). Errors increment the counters; successful transmissions and receptions decrement them.
Based on these counters, each node is in one of three error states:

CAN error confinement state machine. A faulty node escalates from Error Active → Error Passive → Bus Off, automatically disconnecting itself without taking down the bus.
Error Active nodes transmit active error flags (6 dominant bits) when they detect an error, which immediately aborts the current frame for all nodes and triggers retransmission. Error Passive nodes — ones that have already been flagged as unreliable — can only send passive error flags (6 recessive bits), which are invisible on the bus and do not disturb other nodes. A node in Bus Off state cannot transmit or receive at all.
This graduated response is intentional. A node that detects occasional errors due to noise will increment its error counters but stay in Error Active. A node with a hardware fault that continuously generates errors will escalate to Bus Off and disconnect itself, leaving all other nodes communicating normally. CAN is designed so one faulty node cannot permanently corrupt bus traffic.
Termination resistors
CAN is a transmission line. At the bit rates used in real systems (125 kHz to 1 MHz, sometimes higher), the bus wires act as a transmission line, and signal reflections from unterminated ends will corrupt frames. The fix is to place a 120 Ω resistor (matched to the CAN cable's characteristic impedance) at each end of the bus — exactly two resistors, one at each physical end.
Exactly two 120 Ω resistors, one at each physical end of the bus. Nodes tap in along the middle. The parallel combination of two 120 Ω resistors = 60 Ω, which is the intended bus impedance seen by the transceivers.
Common termination mistakes: adding termination at every node (too low impedance, distorts the signal), putting termination in the middle of the bus (unterminated stubs cause reflections from both ends), or forgetting termination entirely (signal rings and causes bit errors). On a bench setup with a short cable, missing termination may work by luck at low bit rates — it will fail in production at the full intended bit rate or over a long cable.
Split termination. Some boards use two 60 Ω resistors with a capacitor to ground at the midpoint instead of one 120 Ω. This forms an RC low-pass filter that reduces high-frequency common-mode noise — useful in very noisy environments such as engine bays.
