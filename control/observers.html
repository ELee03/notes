Observers & the Separation Principle
Luenberger Observer · Error Dynamics · Duality · Output Feedback · Kalman Filter
The state-feedback designs of Lessons 12 and 13 require measuring every state at every instant. In practice, only a few signals are directly accessible — perhaps position but not velocity, voltage but not current. The observer (or state estimator) closes this gap: it maintains a running model of the system and corrects it continuously using the available measurements. The separation principle then says the most remarkable thing: you can design the state feedback and the observer completely independently, and the combined system inherits the eigenvalues of both designs.
Notation recap

$e = x - \hat{x}$Estimation error
$L$Observer gain matrix ($n \times p$ for $p$ outputs)
$A - LC$Observer error dynamics matrix (analogous to $A - BK$)

The Observer Problem
We have a plant whose state evolves according to: $$\dot{x} = Ax + Bu, \qquad y = Cx$$ We know the matrices $A$, $B$, $C$ and we observe $y(t)$ and $u(t)$. The state $x(t)$ is not directly available. The question is: can we recover $x(t)$ from the history of $y$ and $u$?
The answer is yes — provided the system is observable (Lesson 11). Observability guarantees that distinct initial conditions produce distinct output trajectories, so the state can in principle be recovered from output measurements. But how to do this in real time, continuously updating as new measurements arrive?
A naive approach: simulate the model from a guessed initial condition. But any initial error in the guess never corrects itself — the simulated trajectory diverges from the true state as model errors accumulate. We need feedback.
The Luenberger Observer
The Luenberger observer is a copy of the plant with a feedback correction term proportional to the output error — the difference between what the observer predicts the output should be ($C\hat{x}$) and what is actually measured ($y$):
$$\dot{\hat{x}} = A\hat{x} + Bu + L(y - C\hat{x})$$
Expanding: $\dot{\hat{x}} = (A - LC)\hat{x} + Bu + Ly$. The observer takes two inputs: the plant input $u$ (which we know since we choose it) and the plant output $y$ (which we measure). It produces a running estimate $\hat{x}$ of the true state.
The gain $L$ (an $n \times p$ matrix for a plant with $p$ outputs and $n$ states) controls how aggressively the observer corrects its estimate based on the output prediction error. The structure is the continuous-time analogue of a feedback controller: the output residual $y - C\hat{x}$ is the "error signal" that drives the correction.
Error Dynamics
Define the estimation error $e(t) = x(t) - \hat{x}(t)$. Differentiating and substituting: $$\dot{e} = \dot{x} - \dot{\hat{x}} = (Ax + Bu) - (A\hat{x} + Bu + L(Cx - C\hat{x}))$$ $$= A(x - \hat{x}) - LC(x - \hat{x}) = (A - LC)\,e$$
This is a homogeneous linear ODE in the error:
$$\dot{e} = (A - LC)\,e \qquad \Rightarrow \qquad e(t) = e^{(A-LC)t}\,e(0)$$
The error decays to zero if and only if all eigenvalues of $A - LC$ have negative real parts. Since $e(0) = x(0) - \hat{x}(0)$ is the initial estimation error (which we can't control — we don't know $x(0)$), the observer must be designed so that $A - LC$ is stable regardless of initial conditions.
The key insight. The observer error is completely independent of the control input $u$. The error dynamics $\dot{e} = (A-LC)e$ depend only on the observer gain $L$ and the plant matrices $A$, $C$. You can design $L$ purely to make the error decay fast, without worrying about the control law at all.
Observer Gain Design
The observer design problem is: choose $L$ such that $A - LC$ has eigenvalues at desired locations in the left half-plane. Comparing with Lesson 12:
State feedback
Choose $K$ so $A - BK$ has desired eigenvalues.
Possible iff $(A,B)$ is controllable.
⟷
Observer design
Choose $L$ so $A - LC$ has desired eigenvalues.
Possible iff $(A,C)$ is observable.
The two problems are formally identical after transposing: noting that $A - LC = (A^T - C^T L^T)^T$, the eigenvalues of $A-LC$ are the same as those of $A^T - C^T L^T$. So the observer gain problem is the pole-placement problem for the system $(A^T,\, C^T)$: $$L^T = K_{\text{obs}} \quad \text{where} \quad K_{\text{obs}} \text{ places poles of } A^T - C^T K_{\text{obs}}$$ Apply Ackermann's formula to $(A^T, C^T)$ to get $K_{\text{obs}}$, then $L = K_{\text{obs}}^T$.
How fast should the observer poles be? Observer poles must be stable (in the left half-plane), but their exact placement is a design choice. The common rule of thumb: place observer poles roughly $2$–$5\times$ faster (further left) than the closed-loop poles. This ensures the estimation error decays before it significantly degrades the closed-loop performance. Going too fast amplifies measurement noise; going too slow leaves large estimation errors that corrupt the feedback signal.
Kalman Duality
The symmetry between controllability/observability and between the state-feedback/ observer problems is not coincidental — it is the Kalman duality principle:
Duality theorem. The pair $(A, B)$ is controllable if and only if $(A^T, B^T)$ is observable. Equivalently, $(A, C)$ is observable if and only if $(A^T, C^T)$ is controllable.

As a consequence: every result about controllability and state feedback has a dual result about observability and observers. The controllability matrix $\mathcal{C} = [B,\,AB,\ldots,A^{n-1}B]$ dualises to the observability matrix $\mathcal{O} = [C^T,\,A^TC^T,\ldots,(A^T)^{n-1}C^T]^T$. Ackermann's formula dualises from state feedback to observer gain design.
Duality means you never need separate theory for observers — every observer result is a free corollary of the corresponding controllability result, obtained by transposing the system. This is one of the most elegant structural observations in linear systems theory.
Output Feedback via Estimated State
We now have:
  • State feedback gain $K$ designed so $A - BK$ is stable with desired eigenvalues (Lessons 12–13)
  • Observer gain $L$ designed so $A - LC$ is stable with desired eigenvalues (this lesson)
The practical control law uses the estimated state instead of the true state: $$u = -K\hat{x}$$ The combined closed-loop system has state $[x,\,e]^T$ where $e = x - \hat{x}$: $$\frac{d}{dt}\begin{bmatrix}x \\ e\end{bmatrix} = \underbrace{\begin{bmatrix}A - BK & BK \\ 0 & A - LC\end{bmatrix}}_{A_{cl}}\begin{bmatrix}x \\ e\end{bmatrix}$$
The closed-loop matrix $A_{cl}$ is upper-triangular, so its eigenvalues are precisely the union of the eigenvalues of $A - BK$ and $A - LC$: $$\text{eig}(A_{cl}) = \text{eig}(A - BK) \cup \text{eig}(A - LC)$$
The Separation Principle
Separation Principle
For a linear time-invariant system, the output-feedback controller $u = -K\hat{x}$ (where $\hat{x}$ is produced by a Luenberger observer) has the following property:

The closed-loop eigenvalues are the union of the state-feedback eigenvalues (eigenvalues of $A - BK$) and the observer eigenvalues (eigenvalues of $A - LC$).

Therefore, $K$ and $L$ can be designed completely independently: design $K$ to achieve desired state-feedback performance, then design $L$ to achieve desired observer convergence rate. The combination is always stable if both individual designs are stable.
This is profoundly useful in practice. Without the separation principle, designing an output-feedback controller would require solving a joint optimisation over $K$ and $L$ simultaneously — an intractable problem in general. The separation principle splits it into two independent, tractable design problems.
Proof sketch. The upper-triangular structure of $A_{cl}$ is the key. Substituting $\hat{x} = x - e$ into $u = -K\hat{x}$: $$\dot{x} = Ax + Bu = Ax - BK(x - e) = (A-BK)x + BKe$$ The $x$-dynamics are driven by $e$, but the $e$-dynamics are independent of $x$. The block structure isolates the two subsystems' eigenvalues.
Kalman Filter Preview
In practice, measurements are noisy and the system is subject to disturbances. The Kalman filter is the optimal observer when noise is present: $$\dot{x} = Ax + Bu + Gw, \qquad y = Cx + v$$ where $w \sim \mathcal{N}(0, W)$ is process noise and $v \sim \mathcal{N}(0, V)$ is measurement noise (both zero-mean Gaussian with known covariance matrices $W$ and $V$).
The Kalman filter has exactly the same form as the Luenberger observer, but the gain $L$ is now time-varying (or, at steady state, constant) and determined by solving a dual ARE:
$$\text{(Filter ARE):} \quad A\Sigma + \Sigma A^T - \Sigma C^T V^{-1} C\Sigma + GWG^T = 0$$ $$L_{\text{KF}} = \Sigma C^T V^{-1}$$
The matrix $\Sigma$ is the steady-state estimation-error covariance. Comparing with the LQR Riccati equation $(A^TP + PA - PBR^{-1}B^TP + Q = 0)$:
Duality of LQR and Kalman filter. The Kalman filter ARE is the LQR ARE with the substitution: $A \to A^T$, $B \to C^T$, $Q \to GWG^T$, $R \to V$, $P \to \Sigma$. The Kalman gain is $L = \Sigma C^T V^{-1}$, dual to the LQR gain $K^* = R^{-1}B^TP$.

This means: every result about LQR — including the guaranteed stability margins and Bryson's tuning rule — has a dual result for the Kalman filter. Large $V$ (trust the model more than the measurements) → smaller $L$ → slower corrections. Large $W$ (model is unreliable) → larger $\Sigma$ → larger $L$ → faster corrections.
LQG control. Combining the LQR state-feedback gain $K^*$ with the Kalman filter observer $L_{\text{KF}}$ yields the Linear Quadratic Gaussian (LQG) controller — the optimal output-feedback controller for linear systems with Gaussian noise. The separation principle applies: LQR and Kalman filter are designed independently and combined. LQG is the foundation of modern control and the starting point for $H_\infty$ and robust control theory.

Worked Examples
Example 1 — Observer design by duality
Design a Luenberger observer for: $$A = \begin{bmatrix}0&1\\-2&-3\end{bmatrix},\quad C = \begin{bmatrix}1&0\end{bmatrix}$$ with observer poles at $-5 \pm 5j$. Use the duality/pole-placement approach.
Step 1 — Check observability. $\mathcal{O} = \begin{bmatrix}C \\ CA\end{bmatrix} = \begin{bmatrix}1&0\\0&1\end{bmatrix}$, $\det\mathcal{O} = 1 \neq 0$. ✓

Step 2 — Transpose the problem. Apply pole placement to $(A^T,\, C^T)$ with gain $\tilde{K} = [\tilde k_1,\, \tilde k_2]$: $$A^T - C^T\tilde{K} = \begin{bmatrix}-\tilde k_1 & -2-\tilde k_2 \\ 1 & -3\end{bmatrix}$$ Characteristic polynomial: $s^2 + (\tilde k_1 + 3)s + (3\tilde k_1 - \tilde k_2 - 2 + 2)$.

Step 3 — Match desired polynomial. Desired poles $-5 \pm 5j$: $(s+5)^2 + 25 = s^2 + 10s + 50$. $$\tilde k_1 + 3 = 10 \;\Rightarrow\; \tilde k_1 = 7$$ $$3(7) - \tilde k_2 = 50 \;\Rightarrow\; \tilde k_2 = 21 - 50 = -29$$ Wait — recheck the $(2,2)$ entry: $\det(A^T - C^T\tilde K) = (-\tilde k_1)(-3) - (1)(-2-\tilde k_2) = 3\tilde k_1 + 2 + \tilde k_2$. For the char poly constant term: $3(7) + 2 + \tilde k_2 = 50 \Rightarrow \tilde k_2 = 27$. So $\tilde K = [7,\, 27]$ and the observer gain is: $$L = \tilde K^T = \begin{bmatrix}7\\27\end{bmatrix}$$ Verify: $A - LC = \begin{bmatrix}0-7&1\\-2-27&-3\end{bmatrix} = \begin{bmatrix}-7&1\\-29&-3\end{bmatrix}$. Char poly: $s^2 + 10s + (21+29) = s^2 + 10s + 50$. ✓
Example 2 — Verifying the separation principle
For the double integrator ($A = \begin{bmatrix}0&1\\0&0\end{bmatrix}$, $B = \begin{bmatrix}0\\1\end{bmatrix}$, $C=[1,0]$) with state-feedback gain $K = [1,\,\sqrt{3}]$ and observer poles at $-3 \pm 3j$, write the combined closed-loop matrix and verify the separation principle by computing its eigenvalues.
Step 1 — Find observer gain. Observer desired poly: $(s+3)^2 + 9 = s^2 + 6s + 18$. $A - LC = \begin{bmatrix}-l_1 & 1 \\ -l_2 & 0\end{bmatrix}$, char poly $s^2 + l_1 s + l_2$. So $l_1 = 6$, $l_2 = 18$. $L = [6,\, 18]^T$.

Step 2 — Form $A_{cl}$. $$A_{cl} = \begin{bmatrix}A-BK & BK \\ 0 & A-LC\end{bmatrix} = \begin{bmatrix}0&1&0&0\\-1&-\sqrt{3}&1&\sqrt{3}\\0&0&-6&1\\0&0&-18&0\end{bmatrix}$$ Step 3 — Eigenvalues. Since $A_{cl}$ is block upper-triangular, eigenvalues are $\text{eig}(A-BK) \cup \text{eig}(A-LC)$. $A-BK$: char poly $s^2 + \sqrt{3}s + 1$, roots $-\sqrt{3}/2 \pm j/2 \approx -0.866 \pm 0.5j$. $A-LC$: char poly $s^2 + 6s + 18$, roots $-3 \pm 3j$. Combined eigenvalues: $\{-0.866 \pm 0.5j,\ -3 \pm 3j\}$ — the two designs are completely decoupled. ✓
Example 3 — Kalman filter gain for a noisy double integrator
The double integrator has process noise entering through acceleration ($G = [0;\,1]$, $W = 0.1$) and position measurement noise ($V = 0.01$). Describe how the Kalman filter gain is determined and what happens at the noise extremes $W \to 0$ and $V \to 0$.
The filter ARE is $A\Sigma + \Sigma A^T - \Sigma C^T V^{-1} C\Sigma + GWG^T = 0$ where $GWG^T = \begin{bmatrix}0&0\\0&0.1\end{bmatrix}$ and $C^T V^{-1} C = 100\begin{bmatrix}1&0\\0&0\end{bmatrix}$.

The $(1,1)$ entry of the ARE gives $-100\sigma_{11}^2 = 0 \Rightarrow \sigma_{11} = 0$, meaning the steady-state position estimation error covariance vanishes (the position measurement is accurate enough to be trusted exactly). The Kalman gain $L = \Sigma C^T V^{-1}$ concentrates on correcting the velocity estimate from position measurements.

Limiting behaviour:
  • $W \to 0$ (model perfect): $\Sigma \to 0$, $L \to 0$ — trust the model completely, ignore measurements.
  • $V \to 0$ (sensor perfect): $\Sigma \to 0$ but $L = \Sigma C^T V^{-1}$ remains finite as the ratio $\Sigma/V$ stays bounded — follow measurements closely.
  • $W/V$ large (noisy model, good sensors): large $L$, aggressive correction each measurement step.
Food for Thought
  • The separation principle says closed-loop eigenvalues are the union of feedback and observer eigenvalues. But the LQR guarantee of $\geq 60°$ phase margin does not carry over to the combined LQG controller. Why not — what changes when the observer is in the loop?
    Answer
    The LQR margin proof relies on the return-difference inequality $\|I + K^*(j\omega I-A)^{-1}B\| \geq 1$, which holds because the feedback path is the direct state $x$. When an observer is inserted, the actual closed-loop transfer function from plant input to feedback becomes $K^*(sI - A + BK^* + LC)^{-1}L \cdot C(sI-A)^{-1}B$ — it involves the observer's poles and zeros and introduces additional phase lag. The return-difference is no longer bounded away from zero in general, so margins can shrink to near zero even when both $K^*$ and $L$ are individually well-designed. This failure mode is known as the "LQG fragility" problem; loop transfer recovery (LTR) is the standard fix.
  • In the observer error dynamics $\dot{e} = (A-LC)e$, the input $u$ does not appear. Does this mean the control design has no effect on how fast the observer converges? What would happen if you used a very bad state-feedback gain $K$?
    Answer
    Correct — the error $e = x - \hat x$ converges at the rate of $A-LC$ eigenvalues, independent of $K$. A bad $K$ doesn't slow the observer. However, a destabilising $K$ causes the true state $x(t)$ to grow without bound. The observer faithfully estimates this diverging state (if $A-LC$ is stable), so $\hat x \to x$ while $x \to \infty$. You get a very accurate estimate of a runaway system — the control signal $u = -K\hat x$ also grows, but in the wrong direction. The separation principle guarantees stability only when both $A-BK$ and $A-LC$ are independently stable.
  • The Kalman filter is optimal for Gaussian noise. If the noise is heavy-tailed (occasional large outliers), would a Kalman filter still work well? What might you use instead?
    Answer
    No — the Kalman filter is optimal for minimum mean-squared error under Gaussian assumptions. Heavy-tailed noise (e.g. Cauchy, Student-$t$) means occasional very large measurement residuals. A standard Kalman filter treats these outliers as high-probability events and overcorrects, producing large transient estimation errors. Alternatives: (1) Huber-based robust Kalman filter — replaces the quadratic cost with a Huber loss that caps the influence of large residuals; (2) unscented Kalman filter (UKF) with heavy-tailed noise models; (3) particle filter — represents the posterior distribution with a set of weighted samples, making no Gaussian assumption and handling arbitrary noise distributions.
Common Pitfalls
  • Placing observer poles too slow. Observer poles slower than or at the same speed as the closed-loop poles means the estimation error is still significant when the state reaches steady state. This degrades performance — the state feedback is working on a bad estimate. A typical guideline: observer poles 2–5× faster (further left) than closed-loop poles.
  • Placing observer poles too fast. Very fast observer poles amplify high-frequency measurement noise. In practice, the "noise floor" of the sensors determines an upper limit on how fast you can place observer poles. This is precisely what the Kalman filter optimises.
  • Misapplying the separation principle to nonlinear systems. The separation principle is a linear result. For nonlinear systems, combining a state feedback designed for the linearisation with a nonlinear observer does not in general give a stable closed-loop. Nonlinear observers (extended Kalman filter, unscented Kalman filter, particle filter) must account for the coupling.
  • Forgetting the observer initial condition. The observer starts with an estimate $\hat{x}(0)$ that may be far from the true $x(0)$. The initial estimation error $e(0) = x(0) - \hat{x}(0)$ decays exponentially, but before it decays, the control signal may be very different from what full-state feedback would give. Large initial transients or actuator saturation can result.
  • Using LQG and expecting LQR stability margins. The LQR state-feedback design has guaranteed $\geq 60°$ phase margin. Adding the Kalman filter (to form LQG) removes this guarantee. The LQG loop can be arbitrarily close to instability even when LQR and Kalman filter are individually well-designed. If robustness is critical, use LQG/LTR (loop transfer recovery) or $H_\infty$ methods to restore margins.
