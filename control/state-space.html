Beyond transfer functions
Transfer functions are a complete input-output description of a linear system. Given $G(s)$, you can compute the output for any input, draw Bode plots, find stability margins, and design controllers. But transfer functions hide everything that happens inside the system — the internal signals, the energy stored in capacitors and springs, the velocities of masses. For many control problems, you need access to those internal quantities:
  • State feedback (Lesson 12) — the controller reads every internal state and feeds it back. This requires knowing the states, not just the output.
  • Observers (Lesson 14) — when states can't be directly measured, you estimate them from the output. This requires a model of the internal dynamics.
  • Multi-input multi-output (MIMO) systems — a system with $m$ inputs and $p$ outputs has an $m \times p$ transfer function matrix. The state-space framework handles this naturally; working with TF matrices is cumbersome.
  • Analysis and optimal control (Lesson 13) — questions of controllability, observability, and optimal gain selection all live in the state-space.
The state-space representation is the language of modern (post-1960) control theory. The classical frequency-domain tools you've seen so far are powerful for SISO design; the state-space is the foundation for everything beyond.
The state equations
A linear time-invariant system is described by the pair of matrix equations:
$$\dot{x}(t) = Ax(t) + Bu(t) \qquad \text{[state equation]}$$ $$y(t) = Cx(t) + Du(t) \qquad \text{[output equation]}$$
The first equation says: the rate of change of the state depends linearly on the current state and the current input. The second says: the measured output is a linear combination of the state and the input (the $D$ term is the direct feedthrough — zero in most physical systems).
Together, the four matrices $(A, B, C, D)$ are a complete description of the system. Given any initial state $x(0) = x_0$ and any input history $u(t)$, the output $y(t)$ is uniquely determined. That's the same information the transfer function contains — but organized differently, with the internal dynamics explicit.
The key distinction between $A, B, C, D$:
  • $A$ governs how states evolve due to the current state ("internal dynamics").
  • $B$ governs how the input drives the state ("how inputs enter the system").
  • $C$ selects which linear combinations of state are measured ("what sensors see").
  • $D$ is the direct feedthrough from input to output — physically, a system where input energy skips the dynamics entirely. For most mechanical and electrical systems, $D = 0$.
Choosing state variables
The state of a system is the minimal information needed to completely predict its future behavior, given the future inputs. This is a physical concept before it is a mathematical one — the right choice of state variables is usually suggested by physics:
System type State variables Physical reason
Mechanical (spring-mass) position $q$, velocity $\dot{q}$ Stores kinetic energy ($\frac{1}{2}m\dot{q}^2$) and potential energy ($\frac{1}{2}kq^2$) — both needed to predict future
Electrical (RLC) capacitor voltages $v_C$, inductor currents $i_L$ Capacitors and inductors are energy-storage elements; resistors dissipate, no memory
Thermal temperatures $T_i$ Thermal capacitance stores energy; knowing temperatures determines future heat flow
Fluid (tank) fluid heights $h_i$ Height determines potential energy and outflow rate
Rule of thumb: For a system with $n$ independent energy-storage elements (springs, masses, capacitors, inductors — counted by their degrees of freedom), you need $n$ state variables. This matches the order of the differential equation describing the system.
The choice is not unique — any invertible transformation of a valid state vector gives another valid state vector. Different choices lead to different $(A, B, C, D)$ matrices but the same input-output behavior. This is the basis of the similarity transformation and the modal decomposition discussed below.
Physical examples
Example A: Spring-mass-damper. Newton's second law for a mass $m$ with spring constant $k$, damping $b$, and applied force input $u(t)$:
$$m\ddot{q} + b\dot{q} + kq = u$$
Choose $x_1 = q$ (position) and $x_2 = \dot{q}$ (velocity). Then:
$$\dot{x}_1 = x_2, \qquad \dot{x}_2 = -\frac{k}{m}x_1 - \frac{b}{m}x_2 + \frac{1}{m}u$$
Writing in matrix form with output $y = q = x_1$:
$$\dot{x} = \underbrace{\begin{bmatrix}0 & 1 \\ -k/m & -b/m\end{bmatrix}}_{A}x + \underbrace{\begin{bmatrix}0 \\ 1/m\end{bmatrix}}_{B}u, \qquad y = \underbrace{\begin{bmatrix}1 & 0\end{bmatrix}}_{C}x$$
The $A$ matrix captures everything about the internal dynamics. The first row ($\dot{x}_1 = x_2$) just says "velocity is the derivative of position." The second row encodes Newton's law for the acceleration: restoring force $-kx_1/m$ and damping $-bx_2/m$.
Example B: Series RLC circuit. A series circuit with resistor $R$, inductor $L$, capacitor $C$, and voltage source $u(t)$. The KVL loop equation is $L\dot{i} + Ri + v_C = u$, and capacitor dynamics give $\dot{v}_C = i/C$.
Choose $x_1 = v_C$, $x_2 = i_L = i$. Output: voltage across the capacitor, $y = v_C$.
$$\dot{x} = \begin{bmatrix}0 & 1/C \\ -1/L & -R/L\end{bmatrix}x + \begin{bmatrix}0 \\ 1/L\end{bmatrix}u, \qquad y = \begin{bmatrix}1 & 0\end{bmatrix}x$$
Compare with the spring-mass-damper: the matrices are structurally identical, with the correspondence $m \leftrightarrow L$, $k \leftrightarrow 1/C$, $b \leftrightarrow R$. This is not a coincidence — both systems are second-order with one energy-storage element in each "quadrature" (position/voltage, velocity/current). The state-space formulation makes the analogy between mechanics and circuits explicit.
Converting an ODE to state-space
Given an $n$-th order ODE with no input derivatives:
$$y^{(n)} + a_{n-1}y^{(n-1)} + \cdots + a_1\dot{y} + a_0 y = b_0 u$$
The phase variable (or companion) realization is the most direct approach. Let $x_1 = y$, $x_2 = \dot{y}$, …, $x_n = y^{(n-1)}$. Then:
$$\dot{x} = \underbrace{\begin{bmatrix} 0 & 1 & 0 & \cdots & 0 \\ 0 & 0 & 1 & \cdots & 0 \\ \vdots & & & \ddots & \vdots \\ 0 & 0 & 0 & \cdots & 1 \\ -a_0 & -a_1 & -a_2 & \cdots & -a_{n-1} \end{bmatrix}}_{A_c} x + \underbrace{\begin{bmatrix}0\\0\\\vdots\\0\\b_0\end{bmatrix}}_{B_c} u, \qquad y = \underbrace{\begin{bmatrix}1 & 0 & \cdots & 0\end{bmatrix}}_{C_c} x$$
The matrix $A_c$ is the companion matrix of the characteristic polynomial. Its bottom row is the negated coefficients of the ODE; the rest is a shift structure (identity matrix shifted right). This form is useful theoretically, though not always numerically well-conditioned.
What about input derivatives? When the ODE has terms like $\dot{u}$ or $\ddot{u}$ on the right-hand side, the phase-variable trick breaks down (you can't set $x_1 = y$ and differentiate without bringing derivatives of $u$ into the state equation). The fix is a different realization: start from the transfer function $G(s)$ and use the observable canonical form or controllable canonical form. The companion matrix above is already the controllable canonical form for the no-input-derivative case.
From state-space to transfer function
Taking the Laplace transform of both state equations (zero initial conditions):
$$sX(s) = AX(s) + BU(s) \implies X(s) = (sI - A)^{-1}BU(s)$$ $$Y(s) = CX(s) + DU(s) = \bigl[C(sI-A)^{-1}B + D\bigr]U(s)$$
So the transfer function is:
$$G(s) = C(sI - A)^{-1}B + D$$
For a MIMO system, $G(s)$ is a $p \times m$ matrix of rational functions — one entry for each input-output pair. For a SISO system, it reduces to a scalar.
The matrix $(sI - A)^{-1}$ is computed via the adjugate: $(sI-A)^{-1} = \text{adj}(sI-A)/\text{det}(sI-A)$. The denominator $\text{det}(sI-A)$ is the characteristic polynomial of $A$ — a polynomial of degree $n$ in $s$. This is where the poles come from.
Eigenvalues of $A$ are the poles
The poles of the transfer function $G(s)$ are (generically) the roots of $\text{det}(sI - A) = 0$ — which are exactly the eigenvalues of $A$.
$$\text{poles of } G(s) \subseteq \{\lambda \,:\, \det(\lambda I - A) = 0\} = \text{eigenvalues of } A$$
The "subset" qualification matters: if there are pole-zero cancellations in $G(s)$, some eigenvalues of $A$ disappear from the transfer function. This happens when the realization is non-minimal — there are hidden modes that the input can't reach or the output can't see. A minimal realization has no such cancellations, and the eigenvalues of $A$ are exactly the poles of $G(s)$.
Stability from eigenvalues: A system is (asymptotically) stable if and only if all eigenvalues of $A$ have strictly negative real parts:
$$\text{system stable} \iff \text{Re}(\lambda_i) < 0 \text{ for all } i = 1, \ldots, n$$
This is the state-space analog of the Routh-Hurwitz criterion: both answer the same stability question, but from different representations. In state-space, computing the eigenvalues of $A$ directly gives you the answer.
Complex eigenvalues come in conjugate pairs (since $A$ is real-valued), $\lambda_{1,2} = \sigma \pm j\omega_d$. The real part $\sigma$ determines the growth/decay rate; the imaginary part $\omega_d$ determines the oscillation frequency. This is exactly the same $\sigma + j\omega_d$ you've seen on the s-plane in root locus design — the eigenvalues of the closed-loop $A$ matrix are the closed-loop poles.
Modal decomposition
The most revealing way to understand a linear system is to decompose it into independent modes, each with its own time constant and frequency. This is done by diagonalizing $A$.
If $A$ has $n$ linearly independent eigenvectors $v_1, \ldots, v_n$ (true for most matrices in practice — exceptions involve repeated eigenvalues with insufficient eigenvectors, which require Jordan form), then:
$$A = T\Lambda T^{-1}, \qquad \Lambda = \text{diag}(\lambda_1, \ldots, \lambda_n), \qquad T = \begin{bmatrix}v_1 & v_2 & \cdots & v_n\end{bmatrix}$$
Define a change of coordinates $z = T^{-1}x$ (the modal coordinates). Then the state equation becomes:
$$\dot{z} = T^{-1}AT\, z + T^{-1}Bu = \Lambda z + \tilde{B}u$$
Since $\Lambda$ is diagonal, the modal state equation decouples into $n$ independent first-order equations:
$$\dot{z}_i = \lambda_i z_i + \tilde{B}_i u, \qquad i = 1, \ldots, n$$
Each mode $z_i(t)$ evolves as $e^{\lambda_i t}$ (plus a driven term). The original state is $x(t) = Tz(t) = \sum_i z_i(t) v_i$ — a superposition of independent exponential modes, each with its own decay rate and oscillation frequency given by $\lambda_i$.
Modes and physical intuition: For the spring-mass-damper with underdamping ($b^2 < 4mk$), the eigenvalues are $\lambda_{1,2} = -\zeta\omega_n \pm j\omega_n\sqrt{1-\zeta^2}$, giving a single oscillatory mode with exponential envelope $e^{-\zeta\omega_n t}$. This is exactly the step-response behavior from Lesson 3 — the modal decomposition explains why that behavior arises: there is only one mode, and it is a damped sinusoid.
For a system with two real eigenvalues $\lambda_1 < \lambda_2 < 0$, the response has two modes: a fast-decaying component $e^{\lambda_1 t}$ and a slow-decaying component $e^{\lambda_2 t}$. The long-time behavior is dominated by the slowest mode (eigenvalue closest to the imaginary axis).
Interactive: phase plane explorer
For a second-order autonomous system $\dot{x} = Ax$, the phase plane shows how the state $[x_1, x_2]$ evolves from various initial conditions. Each curve is a trajectory starting from a different $x_0$. The character of the trajectories — spiraling inward, spiraling outward, converging along a line, or diverging — is entirely determined by the eigenvalues of $A$.
Adjust the four entries of $A$ and watch the eigenvalues and phase portrait update. Try to reproduce the six canonical phase-plane portraits: stable node, unstable node, saddle point, stable spiral, unstable spiral, and center.

Worked examples
Example 1 — State-space of a double integrator
Derive a state-space realization for the double integrator $G(s) = 1/s^2$ (a point mass $\ddot{q} = u$ with no damping) and verify the transfer function.
Choose $x_1 = q$ (position), $x_2 = \dot{q}$ (velocity). Then $\dot{x}_1 = x_2$ and $\dot{x}_2 = u$: $$\dot{x} = \underbrace{\begin{bmatrix}0 & 1 \\ 0 & 0\end{bmatrix}}_{A}x + \underbrace{\begin{bmatrix}0\\1\end{bmatrix}}_{B}u, \qquad y = \underbrace{\begin{bmatrix}1 & 0\end{bmatrix}}_{C}x$$ Eigenvalues of $A$: $\lambda_{1,2} = 0$ (repeated). The double integrator sits at the stability boundary — perturbations persist without growing or decaying. State feedback (Lesson 12) is needed to make it stable. Verify: $(sI-A)^{-1} = \dfrac{1}{s^2}\begin{bmatrix}s & 1 \\ 0 & s\end{bmatrix}$, so $G(s) = [1\;0]\cdot\dfrac{1}{s^2}\begin{bmatrix}1 \\ s\end{bmatrix} = \dfrac{1}{s^2}$ ✓
Example 2 — Eigenvalues and transfer function from $(A,B,C)$
Given $A = \begin{bmatrix}-3 & 1 \\ -2 & 0\end{bmatrix}$, $B = \begin{bmatrix}0 \\ 1\end{bmatrix}$, $C = \begin{bmatrix}1 & 0\end{bmatrix}$. Find the eigenvalues and the transfer function.
Characteristic polynomial: $$\det(sI - A) = \det\begin{bmatrix}s+3 & -1 \\ 2 & s\end{bmatrix} = s(s+3) + 2 = s^2 + 3s + 2 = (s+1)(s+2)$$ Eigenvalues: $\lambda_1 = -1$, $\lambda_2 = -2$. Both in LHP → asymptotically stable. Transfer function: $$G(s) = C(sI-A)^{-1}B = [1\;0]\cdot\frac{1}{s^2+3s+2}\begin{bmatrix}s & 1 \\ -2 & s+3\end{bmatrix}\begin{bmatrix}0\\1\end{bmatrix} = \frac{1}{s^2+3s+2}$$ Poles at $s = -1, -2$ match the eigenvalues of $A$ — this is a minimal realization ✓.
Example 3 — Minimal vs non-minimal realization
Show that $A = \begin{bmatrix}-2&0\\0&-5\end{bmatrix}$, $B = \begin{bmatrix}1\\1\end{bmatrix}$, $C = \begin{bmatrix}1&0\end{bmatrix}$ is a non-minimal realization of $G(s) = \dfrac{1}{s+2}$ and state the minimal realization.
Verify the TF: With $A$ diagonal, $G(s) = \dfrac{C_1 B_1}{s+2} + \dfrac{C_2 B_2}{s+5} = \dfrac{1}{s+2} + \dfrac{0}{s+5} = \dfrac{1}{s+2}$ ✓ Why non-minimal? The system has 2 states ($n=2$) but $G(s)$ has degree 1. Eigenvalue $\lambda = -5$ is present in $A$ but invisible in $G(s)$: $C = [1\;0]$ only reads $x_1$, so $x_2$ (the $-5$ mode) never appears in the output. It is an unobservable mode. Because it can be driven by the input ($B_2 = 1 \neq 0$), it is controllable but unobservable — a C$\bar{\text{O}}$ mode. Minimal realization: $A = [-2]$, $B = [1]$, $C = [1]$, order $n = 1$.
Food for thought
  1. Two different physical systems — a spring-mass-damper and a series RLC circuit — have the same state-space structure with different parameter names. What does this tell you about the relationship between the mathematics of a system and its physical implementation? What other systems share this structure?
    Answer
    The same differential equation can describe many physically different systems. A spring-mass-damper $m\ddot{q} + c\dot{q} + kq = u$ and an RLC circuit $L\ddot{q} + R\dot{q} + \frac{1}{C}q = u$ (where $q$ is charge) are governed by identical second-order ODEs with $m \leftrightarrow L$, $c \leftrightarrow R$, $k \leftrightarrow 1/C$. The mathematics is universal; only the physical meanings of the states and parameters differ. Other systems with this second-order structure include: a pendulum (linearized near the upright), a torsional oscillator, a water tank with a restoring valve, and any second-order PDE mode after spatial discretization. This universality is a core reason control theory is so broadly applicable — you design for the abstract $(A,B,C)$ structure, and the result applies to any physical realization of that structure.
  2. The state vector $x$ is not unique — any invertible transformation $z = Tx$ gives an equivalent representation with matrices $TAT^{-1}, TB, CT^{-1}, D$. Since the transfer function $G(s) = C(sI-A)^{-1}B$ doesn't change under this transformation, what properties of the system are "coordinate-invariant" (same regardless of which state basis you choose)?
    Answer
    Coordinate-invariant properties — sometimes called similarity invariants — include: the eigenvalues of $A$ (hence the pole locations and stability), the characteristic polynomial of $A$, the transfer function $G(s)$, the rank of the controllability and observability matrices, and whether the system is controllable or observable. These are intrinsic properties of the dynamics, not of the coordinate system used to describe them. Coordinate-dependent quantities include: the individual entries of $A$, $B$, $C$, the eigenvectors of $A$, and the specific state trajectories $x(t)$ (the trajectory of $z(t) = Tx(t)$ looks different from $x(t)$, even though they describe the same physical evolution). This distinction is important in practice: when comparing two realizations of the same system, check eigenvalues and the TF — not the $A$ matrix entries.
  3. The phase variable realization always produces an $A$ matrix in companion form. This is numerically convenient for small systems but problematic for large ones — why might the companion matrix have poor numerical conditioning, and how does this matter for simulation or control design?
    Answer
    The companion matrix has the form with 1s on the superdiagonal and the negated polynomial coefficients in the bottom row. Its condition number grows rapidly with system order $n$ — the entries $a_0, a_1, \ldots, a_{n-1}$ of the characteristic polynomial can span many decades for high-order systems, leading to a very ill-conditioned matrix. Computing eigenvalues from the companion matrix in floating point becomes inaccurate once $n$ is moderate (roughly $n > 10$–20). For simulation, this means numerical integration of the state ODE can be inaccurate — the poorly conditioned $A$ matrix amplifies floating-point rounding at each step. For control design (e.g., pole placement using Ackermann's formula, which uses the controllability matrix $\mathcal{C}$), the companion form makes $\mathcal{C}$ poorly conditioned, causing large errors in the computed gain $K$. In practice, one transforms to a balanced or Schur form before doing numerical work. MATLAB's ss() and place() use numerically robust formulations rather than the raw companion form.
Common pitfalls
  • Confusing eigenvalues of $A$ with poles of $G(s)$. They are the same for a minimal realization, but the eigenvalues of $A$ include all modes — including hidden ones that cancel in the TF. The eigenvalues are the right object for stability analysis of the full state trajectory; the poles are the right object for input-output analysis.
  • Forgetting that the state choice affects $A, B, C, D$ but not $G(s)$. Different valid state choices give the same transfer function — they are equivalent representations. But they give different $A$ matrices, different eigenspaces, and (most importantly) different interpretations of what $x$ means physically.
  • Applying the phase-variable realization when there are input derivatives. If the right-hand side contains $\dot{u}$ or higher derivatives, setting $x_1 = y$ and differentiating introduces derivatives of $u$ into the state equation, which is not allowed. Use the controllable or observable canonical form derived from the transfer function instead.
  • Assuming $D = 0$ always. For most physical plant models $D = 0$ (no direct feedthrough). But when you cascade a controller with the plant, the controller transfer function may have $D \neq 0$ (e.g., a PD controller has $D \neq 0$ because the derivative term passes high-frequency input directly to output). Always check.
  • Dimension mismatches. The most common algebra error: $A$ must be $n \times n$, $B$ must be $n \times m$, $C$ must be $p \times n$, and $D$ must be $p \times m$. When writing state equations for a new system, always check dimensions before computing anything.
